# Abnormal Client Behavior Detection in Federated Learning

Suyi Li, Yong Cheng, Yang Liu, Wei Wang, Tianjian Chen (2019)

## 🧩 Problem to Solve

분산 머신러닝 시스템, 특히 연합 학습(Federated Learning, FL) 환경에서 클라이언트들은 자체적으로 행동하며 서버가 그들의 행동을 완전히 제어할 수 없다. 이러한 자율성으로 인해 클라이언트는 연합 모델 훈련의 정해진 과정을 의도적으로 또는 비의도적으로 벗어날 수 있으며, 이는 악의적인 공격자 또는 오작동하는 클라이언트와 같은 "비정상적인 클라이언트 행동(Abnormal Client Behavior)"으로 이어진다.

이러한 비정상적인 클라이언트 행동은 다음과 같은 심각한 문제를 야기한다.

1. **모델 성능 저하**: 비정상적인 업데이트는 전역 모델의 정확도를 크게 떨어뜨릴 수 있다.
2. **데이터 유출 위험**: 의도하지 않은 클라이언트에게 모델 정보가 유출될 위험이 있다.
3. **자원 낭비**: 비정상적인 클라이언트에 보상이나 인센티브를 할당하는 것은 자원의 낭비이다.
4. **신뢰성 저하**: 연합 학습 시스템의 전반적인 신뢰성에 악영향을 미친다.

따라서 이러한 비정상적인 클라이언트들을 적시에 탐지하고 그들의 악영향을 최소화하는 것이 매우 중요하다. 기존의 방어 기반(defense-based) 접근 방식들은 비정상 클라이언트를 탐지하기보다는 모델 집계 과정에서 잠재적인 공격에 대응하는 데 초점을 맞추며, 종종 정직한 클라이언트의 성능까지 저해하여 정확도가 저하되는 한계가 있다.

본 논문의 목표는 서버 측에서 비정상적인 클라이언트를 효과적으로 탐지하여 그들의 부정적인 영향을 제거하고, 연합 학습 모델의 견고성과 정확도를 향상시키는 새로운 탐지 기반(detection-based) 접근 방식을 제안하는 것이다.

## ✨ Key Contributions

본 논문의 핵심 아이디어는 연합 학습 시스템의 서버 측에서 클라이언트의 지역 모델 가중치 업데이트에 대한 비정상 탐지(Anomaly Detection) 모델을 활용하여 비정상적인 클라이언트 행동을 식별하고 그 영향을 최소화하는 것이다. 이는 기존의 방어 기반(defense-based) 방법론이 모든 클라이언트에 대해 일괄적으로 방어 메커니즘을 적용하여 발생하는 성능 저하 문제를 해결하기 위한 "탐지 기반(detection-based)" 접근 방식이다.

구체적인 중심 직관 및 설계 아이디어는 다음과 같다.

1. **저차원 대리 벡터(Low-dimensional Surrogates)**: 딥러닝 모델의 가중치 벡터는 차원이 매우 크기 때문에, 서버는 차원 축소(dimensionality reduction) 기법을 사용하여 이 가중치 벡터의 저차원 대리(surrogate)를 생성한다. 이를 통해 비정상 탐지 모델의 계산 복잡성을 줄인다.
2. **오토인코더(Autoencoder) 기반 비정상 탐지**: 서버는 정상적인 클라이언트들의 모델 가중치 업데이트 패턴을 학습하도록 미리 훈련된 오토인코더 모델을 사용하여, 각 클라이언트의 업데이트에 대한 재구성 오류(reconstruction error)를 기반으로 비정상 점수(anomaly score)를 계산한다.
3. **신용 점수(Credit Score) 메커니즘**: 각 클라이언트에게는 비정상 점수를 고려하여 계산된 "신용 점수"가 할당된다. 이 신용 점수는 FedAvg 알고리즘의 모델 가중치 집계 과정에서 각 클라이언트 업데이트에 할당되는 가중치($\alpha_k^{t+1}$)로 사용되어, 비정상 클라이언트의 영향을 선택적으로 줄이거나 제거한다.
4. **선택적 방어**: 비정상 클라이언트를 직접 탐지하고 그들의 영향을 줄이거나 제거함으로써, 정상 클라이언트들의 업데이트에는 불필요한 변경을 가하지 않아 모델 정확도를 더 효과적으로 유지할 수 있다. 이는 기존 방어 기반 방법의 "비표적 방어(untargeted defense)" 한계를 극복하는 차별점이다.

요약하자면, 본 연구는 서버에서 오토인코더 기반의 비정상 탐지 모델을 활용하여 각 클라이언트의 업데이트에 신용 점수를 부여함으로써, 비정상 클라이언트를 식별하고 그들의 모델 집계 기여도를 조절하는 새로운 "탐지 기반" 방식을 제안한다.

## 📎 Related Works

본 논문은 주로 분산 머신러닝 시스템, 특히 연합 학습 환경에서의 악의적인 공격자 방어에 대한 기존 연구들을 관련 연구로 소개한다.

**기존 연구 및 한계:**

* **방어 기반(Defense-based) 방법론**:
  * 분산 머신러닝 분야에서 악의적인 공격자로부터 방어하는 연구가 광범위하게 이루어졌다. 대표적인 방어 기반 방법으로는 GeoMed [16], Krum [17], Trimmed Mean [18] 등이 있다.
  * 연합 학습을 위한 방어 기반 방식도 존재하며, [19]의 연구가 이에 해당한다.
  * **한계**: 본 논문은 이러한 전통적인 방어 기반 알고리즘들이 연합 학습 환경에서 때때로 부족한 성능을 보이며, 실험 결과에서 정확도 저하를 야기한다고 지적한다. 성능 저하의 핵심 원인은 기존 방법들이 "비표적 방어(untargeted defense)"를 채택하여 정직한 클라이언트에 대한 비용을 지불하면서 공격자를 방어하기 때문이라고 설명한다. 즉, 비정상 클라이언트를 명확히 식별하기보다 모든 업데이트에 대해 방어 메커니즘을 적용하여 정상적인 업데이트까지 왜곡시킬 수 있다.

* **탐지 기반(Detection-based) 접근 방식**:
  * [20]의 저자들은 협력적 머신러닝(collaborative machine learning)을 위한 탐지 기반 접근 방식을 제안했다.
  * **한계**: [20]의 연구는 훈련 데이터의 마스킹된(masked) 특징 분포가 훈련 데이터의 실제 분포를 보존한다는 가정에 기반하지만, 이러한 가정은 연합 학습에는 적용할 수 없다. 이는 연합 학습이 클라이언트 데이터에 직접 접근하지 않고, 지역 업데이트만 처리하기 때문이다.

**기존 접근 방식과의 차별점:**

본 논문은 기존의 방어 기반 방법들이 가지고 있는 비효율성(정직한 클라이언트에도 영향을 미침)과 비표적 방어의 한계를 지적하며, 비정상 클라이언트를 직접 "탐지"하고 그들의 영향을 "제거"하는 새로운 탐지 기반 접근 방식을 제안한다. 특히, 본 연구는 연합 학습 프레임워크에 탐지 기반 접근 방식을 적용하는 "최초의 시도"라고 명시적으로 언급하며, 이 점에서 기존 연구들과 차별화된다.

## 🛠️ Methodology

본 논문은 서버 측에서 비정상적인 클라이언트 행동을 탐지하기 위한 탐지 기반 접근 방식을 제안한다. 전체 파이프라인은 크게 두 단계로 구성된다: 오토인코더 기반 비정상 점수 계산과 신용 점수를 활용한 모델 가중치 집계.

### 3.1 문제 정의 (Problem Definition)

본 연구는 하나의 서버와 $K$개의 클라이언트로 구성된 수평적 연합 학습(horizontal federated learning) 시스템을 고려한다. 이 시스템은 FedAvg 알고리즘 [2]을 사용하여 협력적으로 모델을 훈련한다. 본 논문은 공간 제약으로 인해 각 클라이언트가 암호화 없이 지역 모델 가중치 업데이트를 서버로 전송하는 경우만을 다룬다. 목표는 서버에 미리 훈련된 비정상 탐지 모델을 사용하여 비정상적인 클라이언트를 탐지하고, 연합 모델 훈련에 미치는 영향을 제거하는 것이다.

### 3.2 탐지 기반 접근 방식 (The Detection-Based Approach)

제안하는 탐지 기반 접근 방식의 핵심 아이디어는 연합 학습 시스템 내의 각 클라이언트에 "신용 점수(credit score)"를 할당하는 것이다. 이 신용 점수는 비정상 탐지 모델이 생성한 "비정상 점수(anomaly score)"를 기반으로 계산된다.

$K$개의 클라이언트가 연합 학습에 참여하고, 각 클라이언트 $k$는 $n_k$개의 훈련 데이터 포인트와 $(t+1)$번째 전역 반복(라운드)에서 지역 모델 가중치 $w_k^{t+1}$을 가진다고 가정한다.

**기존 FedAvg 집계:**
FedAvg 프레임워크 [2]에서의 모델 가중치 집계는 다음과 같다.
$$ w_{t+1} = \sum_{k=1}^{K} \frac{n_k}{n} w_k^{t+1} \quad (1) $$
여기서 $w_{t+1}$은 전역 모델 가중치 업데이트(즉, 집계된 모델 가중치 업데이트)를 나타내며, $n = \sum_{k=1}^{K} n_k$는 $K$개 클라이언트 전체의 총 데이터 포인트 수이다.

**제안하는 신용 점수 기반 집계:**
본 논문에서는 Equation (1)의 $\frac{n_k}{n}$ 항을 각 라운드마다 다를 수 있는 $\alpha_k^{t+1}$로 대체한다.
$$ w_{t+1} = \sum_{k=1}^{K} \alpha_k^{t+1} w_k^{t+1} \quad (2) $$
클라이언트 $k$에 $(t+1)$번째 라운드에서 할당된 비정상 점수 $A_k^{t+1}$가 주어졌을 때, 클라이언트 $k$의 신용 점수 $\alpha_k^{t+1}$는 다음과 같이 정의된다.
$$ \alpha_k^{t+1} = \frac{n_k (A_k^{t+1})^{-L}}{\sum_{j=1}^{K} n_j (A_j^{t+1})^{-L}}, \quad \forall j=1,2,\dots,K \quad (3) $$
이 신용 점수 $\alpha_k^{t+1}$는 클라이언트가 소유한 훈련 데이터 포인트의 비율($n_k/n$)뿐만 아니라 비정상 점수 $A_k^{t+1}$도 고려하여, 비정상 클라이언트에 대한 표적화된 방어를 가능하게 한다. 상수 $L$은 $A_k^{t+1}$이 $\alpha_k^{t+1}$ 계산에 미치는 영향을 조절하는 하이퍼파라미터이다. 만약 특정 클라이언트가 데이터의 큰 부분을 소유하고 있다면 $L$ 값은 크게 설정해야 한다.

$\sum_{k=1}^{K} \alpha_k^{t+1} = 1$이므로, Equation (2)의 제안된 반복 모델 평균화 절차는 Equation (1)의 FedAvg 알고리즘의 수렴이 보장되는 한 수렴이 보장된다. 이는 본질적으로 학습률을 스케일 다운하기 때문이다.

### 3.3 오토인코더 기반 비정상 탐지 (Autoencoder-Based Anomaly Detection)

본 접근 방식에서는 미리 훈련된(pre-trained) 오토인코더 모델을 서버에서 사용하여 클라이언트로부터 오는 비정상적인 모델 가중치 업데이트를 감지하고, 이를 통해 비정상 클라이언트를 식별한다. 오토인코더 모델은 특히 고차원 데이터의 비정상 탐지에 효과적인 것으로 알려져 있다 [23, 24].

**오토인코더 훈련:**
오토인코더 모델 훈련을 위한 모델 가중치 세트를 $D = \{w_{-1}^1, w_{-1}^2, \dots, w_{-1}^N\}$로 표시한다. 여기서 아래첨자 $-1$은 이 모델 가중치들이 비정상 탐지를 수행하기 전 시점에 서버에 축적된 것임을 나타낸다. 오토인코더 모델은 이 데이터셋 $D$를 사용하여 미리 훈련된다. 훈련 과정에서 데이터 포인트 $w_i^{-1}$는 먼저 인코더 네트워크에 의해 저차원 잠재 벡터로 압축된 다음, 디코더 네트워크에 의해 $\tilde{w}_i^{-1}$로 재구성된다 [24].

**재구성 오류(Reconstruction Error):**
$i$번째 데이터 포인트의 재구성 오류(평균 제곱 오차, Mean Squared Error, MSE)는 다음과 같다.
$$ Err(w_i^{-1}) = \|w_i^{-1} - \tilde{w}_i^{-1}\|^2 \quad (4) $$

**비정상 점수(Anomaly Score) 계산:**
$(t+1)$번째 라운드에서 클라이언트 $k$의 비정상 점수 $A_k^{t+1}$는 다음과 같이 정의된다.
$$ A_k^{t+1} = \frac{1 + Err(w_k^{t+1})}{1 + \sigma_{t+1}} \quad (5) $$
여기서 $\sigma_{t+1} = \min_j \{Err(w_j^{t+1}), j=1,2,\dots,K\}$는 현재 라운드의 모든 클라이언트 업데이트 중 최소 재구성 오류이다. 이 정규화 과정을 통해 비정상 점수는 항상 1보다 크거나 같게 된다.

**차원 축소(Dimensionality Reduction):**
딥러닝 모델의 모델 가중치 $w_k^{t+1}$의 차원은 매우 클 수 있으며, 이는 오토인코더 모델 훈련 및 비정상 탐지에 엄청난 계산 복잡성을 초래할 수 있다. 이러한 계산 복잡성을 줄이기 위해, 모델 가중치 $w_k^{t+1}$에 차원 축소 기법을 적용하여 저차원 대리 벡터(low-dimensional surrogates)를 생성하고 이를 오토인코더 모델의 입력으로 사용한다.

* **랜덤 샘플링**: 모델 가중치 $w_k^{t+1}$의 차원이 $M$일 때, $M$개 요소 중 $M'$개($M' \ll M$)를 무작위로 추출하여 대리 벡터를 형성할 수 있다 [26].
* **특정 계층 가중치 추출**: 특정 신경망 모델의 경우, 모델 가중치 $w_k^{t+1}$에서 특정 계층(예: 마지막 컨볼루션 계층의 가중치 [27])의 가중치를 추출하여 사용할 수 있다.

**강성 결정 (Hard Decisions) - 임계값 설정 (Thresholding):**
클라이언트의 비정상 점수를 얻은 후, Equation (3)에 따라 신용 점수를 계산하고 집계 작업을 실행할 수 있다. 또한 이진 분류와 유사한 절차를 통해 어떤 클라이언트가 비정상적인지에 대한 강성 결정(hard decision)을 내릴 수 있다. 만약 비정상 점수 $A_k^{t+1}$가 임계값 $A_{th}^{t+1}$보다 엄격히 크다면, 해당 클라이언트의 신용 점수 $\alpha_k^{t+1}$는 0으로 설정된다. 그렇지 않은 경우에는 $\alpha_k^{t+1}$는 $n_k/n$으로 설정된다. 임계값 $A_{th}^{t+1}$는 예를 들어, 모든 클라이언트의 비정상 점수 $\{A_k^{t+1}\}$의 평균값 또는 중앙값으로 선택할 수 있다.

## 📊 Results

본 논문은 제안된 비정상 클라이언트 탐지 기법의 성능을 평가하기 위해 여러 실험을 수행하였다.

### 4.1 실험 설정 (Experimental Settings)

* **데이터셋**: Federated Extended MNIST (FEMNIST) 데이터셋 [28]을 사용하여 62개 클래스 이미지 분류 작업을 수행한다. FEMNIST는 총 801,074개의 샘플을 3,500명의 필기자(writer)에게 불균등하게 분배하며, 각 필기자는 평균 229개의 샘플을 가진다 (표준편차 89.6).
* **클라이언트 설정**: [8, 28]과 유사하게 각 필기자가 클라이언트를 대표하여 이질적인(heterogeneous) 연합 환경을 조성한다. 각 전역 라운드마다 $K=20$명의 필기자가 선택되어 배치 크기 16, 학습률 0.06으로 지역 미니배치 경사 하강법(mini-batch gradient descent)을 20 에폭 동안 수행한다.
* **비정상 클라이언트 비율**: 매 전역 라운드에서 비정상 클라이언트의 비율은 30%로 설정된다.
* **탐지 및 신용 점수**: 비정상 점수는 Equation (5)로, 신용 점수는 Equation (3)으로 정의된다. "Credit Score" 접근 방식에서는 Equation (2)로 클라이언트 모델 가중치를 집계한다. "Thresholding" 접근 방식에서는 비정상 점수의 평균값을 임계값으로 설정한다.
* **연합 학습 모델**: LeNet-5 [29]와 유사한 아키텍처의 CNN 모델을 사용한다. 이 모델은 두 개의 컨볼루션 계층(3x3 커널, 32 채널, 2x2 최대 풀링)과 1024개 유닛의 완전 연결 계층, 최종 62차원 소프트맥스 출력 계층으로 구성된다. 은닉 계층에는 ReLu 활성화 함수가 사용된다.
* **오토인코더 모델**: 서버에서 비정상 탐지에 사용되는 오토인코더는 네 개의 은닉 계층을 가진다. 입력은 마지막 컨볼루션 계층에서 무작위로 선택된 3000차원 벡터이며, 각 은닉 계층의 유닛 수는 각각 64, 32, 32, 64이다. 은닉 계층에는 ReLu 활성화 함수가 사용된다. 오토인코더는 서버에 축적된 지역 모델 가중치 업데이트로 훈련되며, 손실 함수는 재구성 오류(MSE), 배치 크기는 32, 드롭아웃 비율은 0.2이다. 신용 점수 계산을 위한 하이퍼파라미터 $L$은 2로 설정된다.

### 4.2 실험 결과 (Experimental Results)

본 논문은 세 가지 adversarial 공격 모델, 즉 Sign-flipping, Additive Noise, Gradient Ascent를 고려하여 성능을 평가한다.

* **Sign-flipping 공격**: 모델 가중치의 부호를 뒤집는다.
* **Additive noise 공격**: 모델 가중치에 가우시안 노이즈를 추가한다.
* **Gradient ascent 공격**: 비정상 클라이언트가 경사 하강 대신 경사 상승을 지역적으로 실행한다.

**주요 정량적 및 정성적 결과:**

* **우수한 성능**: Figure 1에 제시된 바와 같이, 제안된 방법("Thresholding" 및 "Credit Score")은 모든 공격 시나리오에서 기준선(baseline) 방식보다 모델 정확도 측면에서 뛰어난 성능을 보인다.
* **이상적인 성능 달성**: "Thresholding" 방식은 비정상적인 업데이트를 제거함으로써, 공격자가 없는 FedAvg 알고리즘과 거의 동일한 성능을 달성한다. 이는 비정상 클라이언트의 영향을 효과적으로 제거할 수 있음을 시사한다.
* **기존 방어 기반 방식 대비 우위**:
  * **GeoMed**: 방어 기반 방법 중 가장 좋은 성능을 보이지만, Sign-flipping 공격 하에서 제안된 탐지 기반 접근 방식은 GeoMed보다 모델 정확도에서 10% 이상의 향상을 달성한다 (Fig. 1(a)). 또한, [17]에서 논의된 바와 같이 GeoMed는 비정상 클라이언트가 보낸 큰 값을 방어하지 못하는 한계가 있다.
  * **Krum**: 이질적인(non-iid) 연합 학습 환경에서는 Krum이 잘 작동하지 않는다. Krum은 모든 클라이언트로부터 받은 업데이트 중 가장 적합한 하나의 지역 모델 가중치 업데이트만을 선택하는데, 이는 비-iid 연합 환경에서 모델 집계에 높은 편향(bias)을 초래한다.
  * **Trimmed Mean**: Krum과 유사하게 비-iid 연합 환경에서 고려된 공격에 대해 효과적이지 않다.
  * **정보 요구 사항**: Trimmed Mean과 Krum은 공격자 비율에 대한 사전 지식을 요구하는데, 이는 연합 학습에서 미리 알 수 없는 정보이다. 제안된 탐지 기반 방식은 이러한 사전 지식 없이 작동한다.

결과적으로, 제안된 탐지 기반 접근 방식은 기존의 방어 기반 방법들이 가지는 한계(정확도 저하, 특정 공격에 대한 취약성, 사전 정보 요구)를 극복하고, 비정상 클라이언트의 영향을 효과적으로 완화하여 연합 학습 모델의 견고성과 정확도를 크게 향상시킨다.

## 🧠 Insights & Discussion

본 연구는 연합 학습 환경에서 비정상적인 클라이언트 행동 탐지에 대한 중요성과 효과적인 솔루션을 제시한다.

**논문에서 뒷받침되는 강점:**

* **표적화된 방어**: 기존의 방어 기반 방법론이 모든 클라이언트에 대해 일괄적으로 방어 메커니즘을 적용하여 발생하는 비표적 방어의 한계를 극복한다. 본 연구는 비정상 클라이언트를 직접 식별하고 그들의 모델 기여도를 조절하는 신용 점수 메커니즘을 통해 표적화된 방어를 실현한다. 이는 정상 클라이언트의 업데이트를 불필요하게 왜곡시키지 않아 모델 정확도를 더 효과적으로 유지한다.
* **뛰어난 성능**: 실험 결과는 제안된 탐지 기반 접근 방식이 다양한 공격 시나리오(Sign-flipping, Additive Noise, Gradient Ascent) 하에서 기존의 GeoMed, Krum, Trimmed Mean과 같은 방어 기반 방법론보다 훨씬 우수한 모델 정확도를 달성함을 명확히 보여준다. 특히, 공격자가 없는 이상적인 FedAvg 성능에 근접하는 "Thresholding" 방식의 결과는 제안 방법의 효과를 강력하게 뒷받침한다.
* **실용적인 적용성**: 딥러닝 모델 가중치의 고차원 문제에 대응하기 위해 차원 축소 기법(랜덤 샘플링, 특정 계층 가중치 추출)을 제안함으로써, 오토인코더 기반 비정상 탐지의 계산 복잡성을 관리할 수 있는 실용적인 방안을 제시한다.
* **유연성**: 제안된 프레임워크는 암호화되지 않은 업데이트 외에 부가적 마스킹(additive masking), 차등 프라이버시(differential privacy), 신뢰 실행 환경(TEE) 등 다양한 시나리오에 적용될 수 있는 잠재력을 가진다. (다만 본 논문에서는 암호화되지 않은 경우만 다룸)

**한계, 가정 또는 미해결 질문:**

* **사전 훈련된 오토인코더의 필요성**: 오토인코더 모델은 서버에 축적된 '정상' 업데이트 데이터로 미리 훈련되어야 한다. 이 "정상" 업데이트를 어떻게 수집하고 훈련하는지에 대한 전제와 과정이 중요하다. 초기 단계에서 비정상적인 클라이언트가 없다는 가정이 필요할 수 있으며, 시간이 지남에 따라 정상 클라이언트의 행동 패턴이 변화할 경우 오토인코더를 주기적으로 재훈련해야 할 수도 있다.
* **하이퍼파라미터 $L$의 민감도**: 신용 점수 계산에 사용되는 하이퍼파라미터 $L$은 클라이언트의 데이터 비율에 따라 중요하게 작용한다. 특히, 한 클라이언트가 데이터의 큰 부분을 소유할 경우 $L$을 크게 설정해야 한다고 언급되었는데, 이 값의 최적 선택은 중요한 과제이며, 잘못된 설정은 탐지 성능에 영향을 미칠 수 있다.
* **차원 축소의 영향**: 모델 가중치의 저차원 대리 벡터를 사용하는 것이 비정상적인 패턴을 포착하는 데 충분한 정보를 유지하는지에 대한 면밀한 분석이 필요할 수 있다. 너무 공격적인 차원 축소는 중요한 이상 징후를 놓칠 수 있다.
* **다양한 공격 시나리오에 대한 확장성**: 본 논문은 몇 가지 일반적인 공격(Sign-flipping, Additive Noise, Gradient Ascent)에 대한 성능을 보여주었지만, 표적 모델 포이즈닝(targeted model poisoning)과 같은 더 정교하고 은밀한 공격에 대한 강건성은 추가적인 연구가 필요하다. (본 논문에서는 이들 공격 시나리오를 전체 버전에서 다룰 것이라고 명시함).
* **수렴 보장의 조건**: 제안된 집계 방식의 수렴이 FedAvg의 수렴이 보장되는 한 보장된다고 언급하지만, 비정상 클라이언트의 지속적인 존재나 매우 높은 $L$ 값 등이 학습률에 어떤 영향을 미치고 실제 수렴 속도나 품질에 어떤 영향을 미칠지에 대한 심층적인 분석은 부족하다.

**논문에 근거한 간략한 비판적 해석 및 논의사항:**

본 연구는 연합 학습의 보안 측면에서 중요한 진전을 이루었다. 기존 방어 기반 방식의 근본적인 한계였던 비표적 방어 문제를 해결하고, 비정상 클라이언트를 명확히 식별하여 그 영향을 최소화하는 탐지 기반 접근 방식의 유효성을 성공적으로 입증하였다. 특히, 오토인코더와 신용 점수 시스템을 결합하여 실용적인 솔루션을 제시한 점은 높이 평가될 만하다.

그러나 사전 훈련된 오토인코더의 필요성, 하이퍼파라미터 $L$의 민감한 설정, 그리고 차원 축소의 최적화 등은 실제 시스템에 적용할 때 고려해야 할 과제이다. 향후 연구에서는 오토인코더의 적응적 훈련 방법, $L$ 값 자동 조정 메커니즘, 그리고 다양한 고급 공격에 대한 강건성 검증이 이루어져야 할 것이다. 그럼에도 불구하고, 비정상 클라이언트 탐지라는 새로운 패러다임을 연합 학습에 성공적으로 도입하여 기존의 패러다임과 비교했을 때 획기적인 성능 향상을 보여주었다는 점에서 이 연구는 연합 학습의 보안 및 신뢰성 향상에 중요한 기여를 한다.

## 📌 TL;DR

이 논문은 연합 학습 시스템에서 클라이언트의 **비정상 행동(Abnormal Client Behavior)**을 탐지하여 모델 훈련에 미치는 악영향을 최소화하는 새로운 **탐지 기반(detection-based)** 접근 방식을 제안한다. 핵심 아이디어는 서버 측에서 미리 훈련된 **오토인코더(Autoencoder)**를 사용하여 클라이언트의 지역 모델 가중치 업데이트에 대한 **비정상 점수(Anomaly Score)**를 계산하는 것이다. 딥러닝 모델 가중치의 고차원 문제를 해결하기 위해 **차원 축소(Dimensionality Reduction)** 기법을 적용하여 저차원 대리 벡터를 오토인코더 입력으로 사용한다. 이후, 이 비정상 점수를 기반으로 각 클라이언트에 **신용 점수(Credit Score)**를 할당하고, 이 신용 점수를 사용하여 연합 평균(FedAvg) 알고리즘의 모델 가중치 집계 가중치를 조절하여 비정상 클라이언트의 영향을 선택적으로 줄이거나 제거한다. 실험 결과, 제안된 탐지 기반 방법("Thresholding" 및 "Credit Score")은 Sign-flipping, Additive Noise, Gradient Ascent와 같은 다양한 공격 시나리오에서 기존의 방어 기반(defense-based) 방법론(GeoMed, Krum, Trimmed Mean)보다 **10% 이상의 모델 정확도 향상**을 포함하여 훨씬 우수한 성능을 보인다. 이 연구는 비정상 클라이언트를 직접 탐지하고 그들의 영향을 표적화하여 제거함으로써 연합 학습 모델의 견고성과 정확도를 크게 향상시킬 잠재력을 가진다.
