# Federated Learning with GAN-based Data Synthesis for Non-IID Clients

Zijian Li, Jiawei Shao, Yuyi Mao, Jessie Hui Wang, Jun Zhang (2022)

## 🧩 Problem to Solve

연합 학습(Federated Learning, FL)은 사용자 데이터를 중앙 서버에 직접 공유하지 않으면서도 분산된 클라이언트들이 협력하여 전역 모델을 학습하는 프라이버시 보호 학습 패러다임이다. 그러나 FL은 클라이언트 간 데이터가 독립적이고 동일하게 분포되지 않은(non-independent and identically distributed, non-IID) 상황에서 심각한 성능 저하를 겪는다는 문제가 있다. Non-IID 데이터 분포는 각 클라이언트가 서로 다른 데이터 분포로부터 학습하게 하여, 로컬 모델 간의 불일치를 증가시키고 전역 모델 통합의 효율성을 저하시킨다.

이러한 문제의 중요성은 실제 연합 학습 환경에서 데이터의 이질성이 흔히 발생하며, 이를 해결하지 못하면 연합 학습의 실용적인 적용이 제한된다는 점에 있다. 특히 극심한 non-IID 상황에서는 기존의 로컬 모델 편향을 줄이는 방식으로는 유의미한 개선을 달성하기 어렵고, 데이터 증강 기법 역시 프라이버시 보호 메커니즘이 없으면 데이터 유출에 취약하다는 한계가 있다.

따라서 이 논문의 목표는 차등 프라이버시(Differentially Private)를 보장하는 합성 데이터를 공유하여 non-IID 문제를 해결하고, 전역 모델의 성능을 향상시키는 새로운 연합 학습 프레임워크를 제안하는 것이다.

## ✨ Key Contributions

이 논문의 중심적인 직관 및 설계 아이디어는 각 클라이언트가 민감한 원시 데이터를 공유하는 대신, 차등 프라이버시를 만족하는 생성 모델(Generative Adversarial Network, GAN)을 사용하여 합성 데이터를 생성하고 이를 매개변수 서버(Parameter Server, PS)와 공유함으로써 클라이언트 간의 데이터 분포 이질성 문제를 완화하는 것이다. 이 합성 데이터는 전역 공유 합성 데이터셋을 구성하며, 여기에 반복적인 의사 레이블(pseudo label) 부여 메커니즘을 적용하여 신뢰도 높은 의사 레이블을 생성한다. 이렇게 생성된 합성 데이터셋은 로컬 개인 데이터셋과 함께 클라이언트의 로컬 모델 훈련에 사용되어 로컬 모델 간의 일관성을 높이고 전역 집계에 기여한다.

주요 기여는 다음과 같다.

* **GAN 기반 차등 프라이버시 합성 데이터 공유**: 각 클라이언트가 로컬 데이터로 GAN을 사전 학습하여 차등 프라이버시를 갖는 합성 데이터를 생성하고, 이를 PS에 업로드하여 전역 공유 합성 데이터셋을 구축한다. 이는 원시 데이터 공유 없이 non-IID 문제를 해결하는 동시에 프라이버시를 보존한다.
* **반복적인 의사 레이블 부여 메커니즘**: PS가 수신된 로컬 모델을 활용하여 합성 데이터에 대한 의사 레이블을 반복적으로 업데이트하는 메커니즘을 제안한다. FL 학습 과정에서 로컬 모델이 점진적으로 개선됨에 따라 의사 레이블의 신뢰도가 향상된다.
* **합성 데이터 기반 모델 훈련 및 상호작용**: 로컬 클라이언트는 로컬 개인 데이터와 의사 레이블이 부여된 합성 데이터를 Mixup 기법으로 결합하여 모델을 훈련하고, PS는 공유 합성 데이터셋으로 전역 모델을 업데이트한다. 이 과정에서 모델 훈련과 합성 데이터셋 업데이트가 상호작용하며 전역 모델의 성능을 지속적으로 향상시킨다.
* **지도 및 준지도 학습 설정에서의 적용 가능성**: 제안된 프레임워크인 SDA-FL은 지도 학습(supervised learning)뿐만 아니라 준지도 학습(semi-supervised learning) 환경에서도 레이블 없는 실제 데이터를 요구하지 않고 적용 가능하다.

## 📎 Related Works

논문은 연합 학습(FL)의 non-IID 문제와 데이터 증강 및 프라이버시 보호 관련 연구들을 소개한다.

### Non-IID Challenges in Federated Learning

Non-IID 데이터 분포는 FL의 근본적인 장애물로 인식되어 왔다. 이는 데이터 분포의 심한 왜곡이 로컬 모델의 발산을 크게 증가시켜 집계된 모델의 성능을 저하시키기 때문이다.

* **로컬 목적 함수 수정**: 많은 연구에서 전역 모델 및 다른 클라이언트의 로컬 모델로부터 얻은 추가 지식을 사용하여 로컬 목적 함수를 수정함으로써 클라이언트 드리프트(client drift)를 완화하려고 시도했다. 그러나 이러한 방법들은 극심한 non-IID 시나리오에서는 만족스러운 성능을 달성하지 못한다.
* **로컬 모델 구조 조정**: 일부 연구는 개별 클라이언트의 로컬 모델 구조를 조정하여 non-IID 데이터의 부정적인 영향을 해결하려고 했다.
* **PS에서의 작업 최적화**: 또 다른 연구 흐름은 모델 집계(model aggregation), 클라이언트 선택(client selection), 클라이언트 클러스터링(client clustering), 분류기 보정(classifier calibration) 등 PS에서의 연산을 최적화하여 데이터 이질성 문제를 다루었다.

이러한 기존 접근 방식들은 주로 모델 훈련 과정이나 집계 전략을 수정하는 데 초점을 맞추었으며, 데이터 자체의 이질성을 직접적으로 해결하기보다는 간접적인 방식으로 모델 편향을 줄이는 데 한계가 있었다.

### Data Augmentation and Privacy Preserving

최근에는 데이터 공유 형태에 기반한 FL 방법들이 주목받고 있으며, 이는 Mixup과 같은 데이터 증강 기술을 활용한다.

* **Mixup 기반 데이터 증강**: 클라이언트들이 혼합된 로컬 데이터를 공유하고 협력적으로 새로운 전역 데이터셋을 구축하여 non-IID 문제를 해결하는 방식이 제안되었다. 그러나 잦은 데이터 교환은 프라이버시 공격에 취약할 수 있다.
* **GAN 기반 데이터 증강**: GAN 기반 데이터 증강은 FL에서 로컬 데이터 불균형을 줄이는 데 효과적인 것으로 나타났다. 일반적인 아이디어는 클라이언트가 업로드한 소수의 시드(seed) 데이터 샘플을 기반으로 서버에서 좋은 생성 모델을 훈련시킨 다음, 이 잘 훈련된 생성기를 모든 클라이언트가 다운로드하여 로컬 모델을 업데이트하는 것이다. 하지만 로컬 데이터 샘플을 서버로 보내는 것은 데이터 프라이버시 요구 사항을 위반한다.
* **FedDPGAN**: FedDPGAN은 모든 클라이언트가 FL 프레임워크를 기반으로 전역 생성 모델을 공동으로 훈련하여 부족한 로컬 데이터를 보완하는 방식을 제안했다. 그러나 GAN 훈련 과정 또한 잦은 생성 모델 교환을 필요로 하여 통신 비용이 매우 높고 적대적 공격의 위험이 있다.

기존 데이터 증강 방법들은 프라이버시 침해의 가능성이 있거나, 전역 GAN 훈련 방식은 높은 통신 비용과 공격 위험을 수반한다는 한계가 있었다. 본 논문에서 제안하는 SDA-FL은 각 클라이언트가 독립적으로 차등 프라이버시를 보장하는 GAN을 훈련하여 합성 데이터를 생성하고, 이를 PS가 공유 데이터셋으로 활용함으로써 기존 방법들의 프라이버시 및 통신 비용 문제를 해결하고, non-IID 문제를 더욱 효과적으로 완화한다. 특히, 로컬 GAN을 개별적으로 훈련하여 합성 데이터를 생성하는 방식은 FedDPGAN과 같이 전역 GAN을 협력적으로 훈련하는 방식보다 더 높은 품질의 합성 데이터를 생성할 수 있음을 실험적으로 보여준다.

## 🛠️ Methodology

### 전체 파이프라인 또는 시스템 구조

SDA-FL(Synthetic Data Aided Federated Learning) 프레임워크는 그림 1에 제시된 바와 같이 두 가지 주요 단계로 구성된다.

1. **초기 전역 합성 데이터셋 구축**: FL 프로세스 시작 전에, 각 클라이언트는 로컬 데이터를 기반으로 로컬 GAN 모델을 사전 학습하여 합성 데이터 샘플을 생성한다. 이 합성 샘플들은 PS로 전송되어 전역 공유 합성 데이터셋을 구성한다. 이 데이터셋에는 초기에는 레이블이 없거나 신뢰도가 낮은 의사 레이블이 부여될 수 있다.
2. **반복적인 FL 훈련 및 합성 데이터셋 업데이트**: 각 통신 라운드(round) $t$에서 다음과 같은 과정이 반복된다.
    * **클라이언트 다운로드 및 로컬 훈련**: 모든 클라이언트는 전역 모델 $w_t$를 다운로드한다. 각 클라이언트는 로컬 개인 데이터셋 $D_k$와 PS로부터 제공된 전역 공유 합성 데이터셋 $\hat{D}_s$ (신뢰도 높은 의사 레이블 $\hat{Y}_t$ 포함)를 Mixup 기법으로 결합하여 로컬 모델을 업데이트한다.
    * **클라이언트 업로드 및 PS 집계**: 업데이트된 로컬 모델 $w_k^{t+1}$은 PS로 업로드된다.
    * **PS에서의 의사 레이블 및 전역 모델 업데이트**: PS는 수신된 로컬 모델들을 사용하여 전역 공유 합성 데이터셋의 의사 레이블을 업데이트한다. 이와 동시에, PS는 업데이트된 합성 데이터셋을 이용하여 전역 모델 $w_{t+1}$을 추가로 훈련하고 업데이트한다.

이러한 모델 훈련과 합성 데이터셋 업데이트 간의 상호작용은 매 훈련 라운드마다 전역 모델 성능을 향상시키는 데 중요하다.

### 각 주요 구성 요소 및 역할

* **클라이언트(Client)**:
  * **로컬 GAN 사전 학습**: FL 시작 전, 각 클라이언트는 자신의 로컬 데이터 $D_k = (X_k, Y_k)$를 사용하여 차등 프라이버시(Differentially Private) WGAN-GP(Wasserstein GAN with Gradient Penalty)를 훈련한다. 이 GAN은 프라이버시를 보호하면서 합성 데이터를 생성한다.
  * **합성 데이터 생성 및 업로드**: 훈련된 로컬 GAN으로 합성 데이터 샘플을 생성하여 PS에 업로드한다.
  * **로컬 모델 훈련**: 전역 모델 $w_t$를 다운로드하고, 로컬 개인 데이터 $D_k$와 PS에서 제공받은 전역 합성 데이터셋 $\hat{D}_s$을 결합하여 로컬 모델 $w_k^{t+1}$을 업데이트한다. Mixup 기법을 사용하여 데이터 증강을 수행한다.
  * **로컬 모델 업로드**: 업데이트된 로컬 모델을 PS로 다시 전송한다.
* **매개변수 서버(Parameter Server, PS)**:
  * **전역 합성 데이터셋 구축**: 각 클라이언트로부터 전송받은 합성 데이터 샘플들을 모아 전역 공유 합성 데이터셋 $\hat{D}_s$을 구축한다.
  * **반복적인 의사 레이블 업데이트**: 각 통신 라운드에서 클라이언트로부터 수신된 로컬 모델 $w_k^{t+1}$을 사용하여 전역 공유 합성 데이터셋의 의사 레이블 $\hat{Y}_t$를 업데이트한다. 이 과정은 신뢰도 임계값 $\tau$를 기반으로 한다.
  * **전역 모델 집계**: 클라이언트로부터 받은 로컬 모델 업데이트를 가중 평균하여 전역 모델 $w_{t+1}$을 집계한다.
  * **전역 모델 추가 훈련**: 전역 공유 합성 데이터셋 $\hat{D}_s$ (업데이트된 의사 레이블 포함)을 사용하여 전역 모델을 추가로 훈련하여 성능을 향상시킨다.

### 관련성이 있는 경우 훈련 목표, 손실 함수, 추론 절차 또는 알고리즘 흐름

#### 1. 차등 프라이버시 WGAN-GP

각 클라이언트는 WGAN-GP를 사용하여 합성 데이터를 생성한다. WGAN-GP의 목적 함수는 다음과 같다.
$$
\min_{G} \max_{D \in \mathcal{D}} E_{x \sim p_r(x)}[D(x)] - E_{z \sim p_z(z)}[D(G(z))] + \gamma \|\nabla_{\hat{x}} D(\hat{x}) - 1\|_2^2
$$
여기서 $G$는 생성자(generator), $D$는 판별자(discriminator) 또는 비평가(critic)이다. $p_r(x)$는 실제 데이터 분포, $p_z(z)$는 입력 노이즈 분포이다. $\mathcal{D}$는 모든 $L$-립시츠 함수 집합을 나타낸다. $\hat{x}$는 실제 샘플 $x$와 가짜 샘플 $G(z)$의 혼합이며, $\gamma$는 하이퍼파라미터이다. WGAN-GP는 판별자의 기울기 노름에 페널티를 부여하여 훈련 안정화를 도모한다.

**차등 프라이버시(Differential Privacy, DP) 적용**:
( $\epsilon$, $\delta$)-차등 프라이버시 정의는 다음과 같다.
$$
P(A_p(M) \in S) \le e^{\epsilon} \cdot P(A_p(M') \in S) + \delta
$$
여기서 $A_p$는 랜덤 메커니즘, $M, M'$는 인접한 데이터셋, $S$는 출력의 부분집합이다. $\epsilon$은 프라이버시 예산으로, 작을수록 강한 프라이버시 보호를 의미한다. 논문은 판별자 훈련 반복마다 업데이트되는 기울기에 가우시안 노이즈(Gaussian noise)를 주입하여 DP를 만족시킨다. 노이즈 분산 $\sigma_n$과 DP의 관계는 다음과 같다.
$$
\sigma_n = \frac{2q}{\epsilon} \sqrt{n_d \log(\frac{1}{\delta})}
$$
여기서 $q$는 각 인스턴스의 샘플 확률, $n_d$는 로컬 데이터셋의 총 배치 수이다. DP의 후처리(post-processing) 속성에 따라, 노이즈가 주입된 판별자 출력으로부터 얻어진 생성자 기울기 또한 동일한 수준의 DP를 만족한다.

#### 2. 의사 레이블링(Pseudo Labeling)

합성 데이터에 대한 신뢰도 높은 의사 레이블을 생성하기 위해, 미리 정의된 임계값 $\tau$를 사용한다. 모델 $w$에 의해 예측된 특정 클래스 $c$의 확률 $f_c(w;x)$가 모든 클래스 중 가장 높으면서 동시에 임계값 $\tau$보다 크면, 해당 클래스 $c$를 샘플 $x$의 의사 레이블 $\hat{y}$로 할당한다. 그렇지 않으면 레이블을 부여하지 않는다(None).
$$
\hat{y} = \begin{cases} c & \text{if } \max_c f_c(w;x) > \tau, \\ \text{None} & \text{otherwise}. \end{cases}
$$
이러한 방식으로 고품질 합성 데이터만이 높은 예측 확률을 통해 의사 레이블을 얻게 되며, 그렇지 않은 샘플은 필터링된다. FL 과정에서 로컬 모델이 개선됨에 따라 의사 레이블의 신뢰도도 지속적으로 향상된다.

#### 3. 합성 데이터 기반 모델 훈련

**로컬 모델 업데이트**:
각 클라이언트의 로컬 훈련은 로컬 실제 데이터 $(X_i^t, Y_i^t)$와 공유 합성 데이터셋의 샘플 $(\hat{X}_e, \hat{Y}_e^t)$을 Mixup 기법으로 혼합하여 수행된다. Mixup은 다음과 같이 데이터를 증강한다.
$$
\bar{X}_{i,e}^t = \lambda_1 \hat{X}_e + (1-\lambda_1) X_{i,e}^t \\
\bar{Y}_{i,e}^t = \lambda_1 \hat{Y}_e^t + (1-\lambda_1) Y_{i,e}^t
$$
여기서 $\lambda_1$은 $\text{Beta}(\alpha, \alpha)$ 분포를 따른다.
로컬 모델 업데이트를 위한 Mixup 손실 함수는 교차 엔트로피 손실 $\mathcal{L}(\cdot)$을 사용하여 다음과 같이 정의된다.
$$
\mathcal{L}_1 = \lambda_1 \mathcal{L}(f(\bar{X}_{i,e}^t; w_t), \hat{Y}_e^t) + (1-\lambda_1) \mathcal{L}(f(\bar{X}_{i,e}^t; w_t), Y_{i,e}^t)
$$
초기 FL 과정에서 의사 레이블의 신뢰도가 낮아 $\mathcal{L}_1$이 불안정할 수 있으므로, 실제 배치 샘플에 대한 추가적인 교차 엔트로피 손실 항 $\mathcal{L}_2$가 도입되어 훈련을 안정화한다.
$$
\mathcal{L}_2 = \mathcal{L}(f(X_{i,e}^t; w_t), Y_{i,e}^t)
$$
최종적으로, 로컬 모델은 SGD를 사용하여 다음 손실 함수를 최소화하는 방향으로 업데이트된다.
$$
w_k^{t+1} \leftarrow w_k^t - \eta_t \nabla (\mathcal{L}_1 + \lambda_2 \mathcal{L}_2)
$$
여기서 $\lambda_2$는 로컬 데이터 유지 정도를 제어하는 하이퍼파라미터이다.

**전역 모델 업데이트 (PS)**:
전통적인 FL과 달리 SDA-FL에서 PS는 전역 공유 합성 데이터셋 $\hat{D}_s$ 전체에 접근할 수 있다. PS는 이 데이터셋을 사용하여 전역 모델을 추가로 훈련한다. PS에는 실제 데이터가 없으므로, 두 개의 합성 샘플 배치만을 사용하여 각 반복에서 위 (8)번 식과 유사한 방식으로 전역 모델을 업데이트한다.

### 훈련 목표, 손실 함수, 추론 절차 또는 알고리즘 흐름 요약

* **훈련 목표**: 클라이언트 간 non-IID 데이터 분포로 인한 FL 성능 저하를 완화하고, 프라이버시를 보호하면서 전역 모델의 정확도를 극대화하는 것이다.
* **손실 함수**:
  * **GAN 훈련**: WGAN-GP 목적 함수를 사용하여 실제 같은 합성 데이터를 생성한다 (상세 내용은 위 참조).
  * **로컬 모델 훈련**: Mixup 손실 $\mathcal{L}_1$과 실제 데이터 교차 엔트로피 손실 $\mathcal{L}_2$의 가중 합을 사용한다 (상세 내용은 위 참조).
* **학습 절차**:
    1. **전처리 단계**: 각 클라이언트가 로컬 데이터로 차등 프라이버시 GAN을 사전 훈련하고 합성 데이터를 생성하여 PS에 업로드한다. PS는 이를 모아 전역 공유 합성 데이터셋을 구축한다.
    2. **FL 라운드 반복**:
        * **클라이언트 측**: PS로부터 전역 모델을 다운로드하고, 로컬 데이터와 (업데이트된 의사 레이블을 가진) 합성 데이터를 Mixup하여 로컬 모델을 훈련한다. 훈련된 로컬 모델을 PS에 업로드한다.
        * **PS 측**: 클라이언트로부터 로컬 모델을 수신하여 전역 모델을 집계한다. 동시에, 수신된 로컬 모델들을 사용하여 전역 합성 데이터셋의 의사 레이블을 업데이트한다. 마지막으로, 업데이트된 의사 레이블을 가진 합성 데이터셋으로 전역 모델을 추가로 훈련하여 성능을 향상시킨다.
    3. 이 과정은 미리 정해진 훈련 라운드 수까지 반복된다.

## 📊 Results

### 데이터셋, 작업, 기준선, 지표

**데이터셋**:

* **벤치마크 데이터셋**: MNIST, FashionMNIST, CIFAR-10, SVHN.
* **실제 애플리케이션 데이터셋**: COVID-19 데이터셋 (Wang et al., 2020c).
  * COVID-19 데이터셋은 정상(Normal), COVID-19, 폐렴(Pneumonia) 클래스로 구성되며, 6개 클라이언트에 불균등하게 분포된다 (Table 1 참조).
**작업**:
* **연합 지도 학습(Federated Supervised Learning)**: 다양한 벤치마크 데이터셋 및 COVID-19 데이터셋에 대한 분류 작업.
* **연합 준지도 학습(Federated Semi-Supervised Learning)**: MNIST, FashionMNIST, CIFAR-10 데이터셋에 대한 분류 작업.
**Non-IID 설정**:
* 훈련 샘플을 클라이언트에 균등하게 분할하지만, 각 클라이언트가 갖는 클래스 수를 $C$로 제한하여 non-IID 시나리오를 시뮬레이션한다. 전체 훈련 데이터셋을 $K \times C$개의 서브셋으로 분할하고 각 서브셋은 단일 클래스만 갖도록 한 다음, 이 서브셋들을 무작위로 섞어 클라이언트들에게 분배한다.
**기준선(Baselines)**:
* **연합 지도 학습**: FedAvg, FedProx, SCAFFOLD, Naivemix, FedMix. COVID-19 데이터셋에서는 FedDPGAN도 추가 비교된다. FedAvg (IID)는 이상적인 IID 분포 시나리오를 나타낸다.
* **연합 준지도 학습**: SemiFL, Local Fixmatch, Local Mixup.
**지표(Metrics)**:
* **정확도(Accuracy)**: 모델의 분류 성능을 측정하는 주요 지표.
* **프레셰 인셉션 거리(Fréchet Inception Distance, FID)**: 생성된 합성 샘플의 품질을 측정하며, FID 점수가 낮을수록 이미지 품질이 우수함을 의미한다.

**실험 설정**:

* 클라이언트 수: 10개 (COVID-19는 6개). 모든 클라이언트가 각 라운드에 참여한다.
* GAN 사전 훈련: CIFAR-10은 36,000회, 다른 데이터셋은 18,000회.
* 합성 샘플 수: 각 클라이언트가 PS에 4,000개의 합성 샘플을 업로드.
* 총 라운드 수: 200회.
* PS의 전역 모델 업데이트: CIFAR-10은 10회, 다른 데이터셋은 50회 반복.
* 로컬 스텝 수 $E$: 지도 학습 90, 준지도 학습 40.
* 옵티마이저: SGD, 학습률 $\alpha=0.03$.
* 배치 크기 $B$: 지도 학습 64, 준지도 학습 80 (레이블 있는 샘플 16개, 레이블 없는 샘플 64개 포함).
* 하이퍼파라미터: $\gamma=10.0$, $\lambda_2=1.0$, 의사 레이블 임계값 $\tau=0.95$.
* 모델 아키텍처:
  * MNIST, FashionMNIST 분류: 간단한 CNN (두 개의 합성곱 계층, 두 개의 완전 연결 계층).
  * CIFAR-10, SVHN, COVID-19 분류: ResNet18.
  * GAN: 생성자는 4개의 디컨볼루션(deconvolution) 계층, 판별자는 4개의 합성곱 계층과 완전 연결 계층.

### 주요 정량적 또는 정성적 결과

#### 1. 연합 지도 학습 성능

* **다양한 벤치마크 데이터셋**: Table 2에서 SDA-FL은 클라이언트당 클래스 수가 다양할 때 모든 기준선 방법보다 훨씬 뛰어난 성능을 보인다. 예를 들어, CIFAR-10에서 클라이언트당 3개 클래스인 경우, Naivemix 및 FedMix보다 최소 5.0% 이상 정확도가 높다. 이는 GAN 기반 데이터 증강이 데이터 이질성의 해로운 영향을 완화하는 데 효과적임을 입증한다.
* **COVID-19 데이터셋**: Table 3에서 SDA-FL은 COVID-19 데이터셋에서 FedDPGAN보다 1.68% 높은 정확도를 달성한다. 이는 FL 프레임워크 기반의 전역 GAN 훈련 방식보다 개별적으로 훈련된 GAN이 더 높은 품질의 합성 데이터를 생성함을 시사한다. 더욱이 SDA-FL은 non-IID 문제를 해결할 뿐만 아니라, 이상적인 IID 분포의 FedAvg(FedAvg (IID))보다도 1.14% 더 높은 정확도를 기록하여 의료 분야 적용에 있어 그 장점을 보여준다.

#### 2. 연합 준지도 학습 성능

* Figure 2는 SDA-FL이 연합 준지도 학습 설정에서 다른 알고리즘보다 더 빠른 수렴과 더 나은 성능을 달성함을 보여주며, 이는 견고성과 일반화 가능성을 나타낸다. FashionMNIST 분류 작업에서 Semi-FL보다 거의 10% 정확도 향상을 보인다. CIFAR-10 데이터셋의 경우, 기준선 방법들이 사용 가능한 전역 모델을 훈련하지 못하는(40% 미만 정확도) 반면, SDA-FL은 이 어려운 시나리오에서도 수렴하여 테스트 정확도를 크게 향상시킨다. 이는 제안된 의사 레이블링 메커니즘이 합성 및 레이블 없는 샘플에 대해 고품질 레이블을 제공하기 때문이다.

#### 3. 프라이버시 예산(Privacy Budget)과 모델 성능 간의 절충

* Figure 3는 프라이버시 예산 $\epsilon$의 영향을 분석한다. 엄격한 프라이버시 예산 ($\epsilon=5$)은 생성 샘플의 FID 점수를 증가시키고, 이는 생성 샘플 품질 저하를 의미한다. 이로 인해 MNIST에서 약 0.61%, FashionMNIST에서 약 2.59%의 정확도 하락이 발생한다. CIFAR-10도 비슷한 경향을 보인다. 그러나 SDA-FL은 이러한 엄격한 프라이버시 요구 사항 하에서도 여전히 우수한 성능을 유지한다. (MNIST, FashionMNIST, CIFAR-10의 실제 샘플 FID 점수는 각각 10.54, 23.17, 42.70으로, 합성 데이터의 FID보다 훨씬 높다.)

#### 4. 서버 업데이트 및 의사 레이블 업데이트의 효과

* **서버 업데이트**: Figure 4(a)는 서버에서 전역 모델을 합성 데이터로 업데이트하는 것이 성능 향상에 기여함을 보여준다. 서버 업데이트가 전혀 없을 경우(0 steps), FashionMNIST에서 모델 성능이 약 3% 감소한다. 하지만 PS가 전역 모델을 너무 많이 업데이트하면(무한대 steps), 합성 데이터의 과도한 개입으로 인해 성능이 저하될 수 있다(66.0% 정확도). 이는 합성 데이터와 로컬 데이터를 신중하게 활용할 필요성을 강조한다.
* **의사 레이블 업데이트**: Figure 4(b)는 SDA-FL 프레임워크에서 매 라운드마다 의사 레이블을 지속적으로 업데이트하는 것이 모델 성능을 향상시킴을 보여준다. 의사 레이블 업데이트 라운드 수가 증가할수록 정확도가 증가하며, 이는 훈련 과정 동안 의사 레이블의 신뢰도 수준이 향상됨을 의미한다.

#### 5. AC-WGAN-GP와의 성능 비교

* Table 4는 SDA-FL에서 WGAN-GP를 사용하는 경우와 레이블 정보를 GAN 훈련에 포함하는 AC-WGAN-GP를 사용하는 경우를 비교한다. 동일한 훈련 반복 횟수에서 WGAN-GP는 더 높은 FID 점수를 가지지만(즉, 생성 품질이 약간 낮을 수 있음), AC-WGAN-GP는 레이블 있는 합성 데이터를 생성함에도 불구하고 WGAN-GP가 더 나은 정확도 성능을 보인다. 이는 제안된 의사 레이블링 메커니즘이 WGAN-GP로 생성된 고품질 합성 데이터에 대해 신뢰도 높은 의사 레이블을 제공하기 때문이다.

## 🧠 Insights & Discussion

### 논문에서 뒷받침되는 강점

* **Non-IID 문제에 대한 효과적인 해결책**: SDA-FL은 각 클라이언트가 로컬로 차등 프라이버시 GAN을 훈련하여 합성 데이터를 생성하고, 이를 PS와 공유함으로써 클라이언트 간의 데이터 이질성 문제를 성공적으로 완화한다. 이는 기존의 모델 정규화나 집계 전략 변경보다 더 근본적인 데이터 레벨에서 문제를 해결하는 접근 방식이다.
* **강력한 프라이버시 보호**: 차등 프라이버시 메커니즘을 GAN 훈련 과정에 통합하여 원시 데이터 공유 없이도 합성 데이터를 안전하게 공유할 수 있다. 실험 결과, 엄격한 프라이버시 예산 하에서도 탁월한 성능을 유지한다.
* **유연성 및 일반화 가능성**: SDA-FL 프레임워크는 지도 학습과 준지도 학습 환경 모두에서 우수한 성능을 보이며, 특히 레이블이 부족한 준지도 학습 시나리오에서 그 효과가 두드러진다. 이는 실제 다양한 연합 학습 시나리오에 적용될 수 있는 잠재력을 의미한다.
* **효과적인 의사 레이블링 메커니즘**: 반복적인 의사 레이블 업데이트 메커니즘은 FL 훈련 과정에서 로컬 모델의 개선과 함께 합성 데이터의 의사 레이블 신뢰도를 지속적으로 높여 모델 성능 향상에 크게 기여한다.
* **서버의 효율적인 역할**: PS가 단순히 모델 집계만을 수행하는 것이 아니라, 공유된 합성 데이터를 활용하여 전역 모델을 추가로 훈련함으로써 전역 모델의 성능을 더욱 향상시키는 이점을 제공한다.

### 한계, 가정 또는 미해결 질문

* **GAN 구조의 한정성**: 이 연구에서는 주로 WGAN-GP 및 AC-WGAN-GP에 초점을 맞추었다. 논문은 다른 GAN 구조의 탐구가 필요하다고 언급하며, 이는 향후 연구를 위한 질문으로 남아있다. 다양한 GAN 아키텍처가 SDA-FL 프레임워크 내에서 어떻게 작동하고 성능에 어떤 영향을 미칠지는 미해결 질문이다.
* **클라이언트 측 계산 자원 요구 사항**: WGAN-GP는 클라이언트 측에서 상당한 계산 자원을 요구한다. 이는 SDA-FL의 적용 가능성을 제한할 수 있으며, 특히 컴퓨팅 자원이 제한적인 엣지 장치에서의 활용에 제약을 가한다. 계산 효율적인 GAN 기반 구조를 개발하는 것이 향후 연구의 중요한 과제로 언급된다.
* **의사 레이블링 임계값($\tau$)의 민감도**: 의사 레이블링 메커니즘은 미리 정의된 임계값 $\tau$에 의존한다. 이 임계값 설정이 모델 성능에 미치는 영향 및 최적의 $\tau$ 값을 동적으로 결정하는 방법에 대한 깊이 있는 분석은 명확히 제시되지 않았다.
* **합성 데이터의 품질 한계**: 프라이버시 예산이 엄격할수록 생성되는 합성 데이터의 품질이 저하되는 경향이 있다 (FID 점수 증가). 이는 프라이버시와 유틸리티 사이의 본질적인 절충점이며, 이 절충점을 최적화하는 추가적인 연구가 필요할 수 있다.
* **커뮤니케이션 오버헤드**: 합성 데이터를 PS로 업로드하는 초기 단계에서 상당한 커뮤니케이션 오버헤드가 발생할 수 있다. 논문은 의사 레이블을 전송하는 추가 커뮤니케이션 오버헤드는 무시할 수 있다고 언급하지만, 초기 대규모 합성 데이터 업로드에 대한 자세한 통신 비용 분석은 부족하다.

### 논문에 근거한 간략한 비판적 해석 및 논의사항

SDA-FL은 연합 학습의 고질적인 non-IID 문제를 해결하기 위한 매우 흥미롭고 효과적인 접근 방식을 제시한다. 특히, 로컬 GAN과 차등 프라이버시를 결합하여 데이터 이질성과 프라이버시 보호라는 두 가지 주요 과제를 동시에 해결하려는 시도는 매우 고무적이다. 기존 Mixup 기반 데이터 증강 방법들이 프라이버시 문제에 취약했던 점을 고려할 때, SDA-FL의 차등 프라이버시 통합은 중요한 진전이다.

하지만 클라이언트 측에서 GAN을 훈련하고 대량의 합성 데이터를 생성해야 하는 계산 및 통신 오버헤드에 대한 실질적인 분석은 더 필요해 보인다. 특히, 각 클라이언트가 4,000개의 합성 샘플을 PS에 업로드한다는 점은 초기 커뮤니케이션 비용이 상당할 수 있음을 의미한다. 미래 연구에서 경량 GAN 모델이나 효율적인 합성 데이터 전송 전략에 대한 탐색이 이루어진다면, SDA-FL의 실용적인 적용 가능성이 크게 확대될 것이다. 또한, 의사 레이블링 임계값 $\tau$의 선택이 결과에 미치는 영향에 대한 더 깊은 민감도 분석이나 동적 임계값 조절 메커니즘에 대한 연구는 프레임워크의 견고성을 더욱 높일 수 있을 것이다. 그럼에도 불구하고, SDA-FL은 데이터 증강 기반 연합 학습 연구의 중요한 방향을 제시하며, 향후 다양한 데이터 이질성 시나리오와 프라이버시 요구 사항을 가진 응용 분야에서 핵심적인 역할을 할 잠재력을 가진다.

## 📌 TL;DR

이 논문은 연합 학습(FL)에서 클라이언트 간의 non-IID(비독립 동일 분포) 데이터 문제와 프라이버시 보호 문제를 동시에 해결하기 위한 새로운 프레임워크인 SDA-FL(Synthetic Data Aided Federated Learning)을 제안한다. 핵심 아이디어는 각 클라이언트가 로컬 데이터를 사용하여 차등 프라이버시(Differentially Private)를 보장하는 GAN(Generative Adversarial Network)을 훈련하고, 이를 통해 생성된 합성 데이터를 매개변수 서버(PS)에 공유하여 전역 공유 합성 데이터셋을 구축하는 것이다. 이 데이터셋에는 PS에서 반복적인 의사 레이블(pseudo label) 부여 메커니즘을 통해 신뢰도 높은 레이블이 할당된다. 이후 클라이언트는 로컬 개인 데이터와 의사 레이블이 부여된 합성 데이터를 Mixup 기법으로 결합하여 로컬 모델을 훈련하고, PS는 집계된 로컬 모델과 함께 합성 데이터셋을 활용하여 전역 모델을 추가로 업데이트한다. 광범위한 실험 결과, SDA-FL은 다양한 벤치마크 데이터셋과 실제 COVID-19 데이터셋에서 지도 학습 및 준지도 학습 설정 모두에서 기존 기준선 방법들보다 훨씬 뛰어난 성능을 보였으며, 엄격한 차등 프라이버시 보호 하에서도 이러한 강점을 유지한다.

이 연구는 데이터 이질성 문제를 데이터 생성 및 공유 방식으로 해결하면서 강력한 프라이버시 보호를 제공함으로써, 연합 학습의 실제 적용 가능성을 크게 높일 수 있다. 특히, 준지도 학습 환경에서의 탁월한 성능은 레이블 부족 문제가 흔한 실제 시나리오에서 중요한 역할을 할 수 있음을 시사한다. 향후 연구에서는 GAN 구조의 다양화와 클라이언트 측 계산 효율성 개선을 통해 이 프레임워크의 적용 범위를 더욱 확장할 수 있을 것으로 기대된다.
