# FedCV: A Federated Learning Framework for Diverse Computer Vision Tasks

Chaoyang He, Alay Dilipbhai Shah, Zhenheng Tang, Di Fan, Adarshan Naiynar Sivashunmugam, Keerti Bhogaraju, Mita Shimpi, Li Shen, Xiaowen Chu, Mahdi Soltanolkotabi, Salman Avestimehr (2021)

## 🧩 Problem to Solve

분산 학습 패러다임인 연합 학습(Federated Learning, FL)은 엣지 기기에 분산된 데이터셋으로부터 전역 또는 개인화된 모델을 학습할 수 있어, 중앙 집중식 학습이 해결하기 어려운 개인 정보 보호 문제(예: 의료 분야), 데이터 전송 및 유지보수 비용, 독점 데이터의 민감성(예: 얼굴 인식)과 같은 다양한 문제를 해결할 잠재력을 가지고 있다. FL은 모델 정확도, 개인화, 시스템 효율성(통신 및 계산), 공격에 대한 견고성, 그리고 개인 정보 보호 등 여러 최적화 목표 사이의 절충점을 찾는 기술이다.

그러나 컴퓨터 비전(Computer Vision, CV) 도메인에서 FL의 모델 성능은 중앙 집중식 학습에 비해 크게 뒤떨어져 있다. 이는 다음과 같은 연구 격차 때문이다.

1. **제한적인 연구 범위:** FL 커뮤니티의 연구는 주로 소규모 데이터셋과 모델을 사용한 이미지 분류 분야의 분산 최적화 방법에 집중되어 있다. 반면 CV 연구는 대규모 지도/자기 지도 사전 학습과 효율적인 CNN 또는 Transformer 모델을 사용하여 ImageNet 및 객체 탐지, 이미지 분할과 같은 다운스트림 태스크의 성능을 크게 향상시키고 있다. 이러한 다양한 태스크에 대한 탐색 부족으로 FL 모델 성능이 중앙 집중식 학습에 비해 낮게 나타난다.
2. **구현의 한계:** CV 모델 학습은 일반적으로 분산 컴퓨팅 환경에서 대규모 컴퓨팅 자원을 요구하지만, 현재 FL 알고리즘은 대부분 독립형 시뮬레이션으로만 발표되어 연구 격차를 더욱 확대한다. 예를 들어, FedVision 라이브러리는 객체 탐지를 지원하지만 단일 GPU 학습만 가능하다.
3. **알고리즘 효능의 불확실성:** 제안된 FL 알고리즘이 다양한 CV 태스크에서 얼마나 효과적인지는 아직 불분명하며, 개인 정보 보호, 보안, 공정성과 같은 다중 최적화 목표와 결합될 경우 더욱 어려워진다. 현재는 소규모 데이터셋 및 모델을 이용한 이미지 분류에서만 이러한 알고리즘의 효능이 주로 평가되고 있다.

이러한 장애물로 인해 순수 알고리즘 연구와 CV 애플리케이션 중심 연구 간의 간극을 좁힐 필요가 있다. 논문의 목표는 이미지 분류, 이미지 분할, 객체 탐지와 같은 세 가지 가장 대표적인 컴퓨터 비전 태스크에서 FL을 평가하기 위한 연합 학습 라이브러리 및 벤치마킹 프레임워크인 FedCV를 제안하여 이 간극을 해소하고 CV 태스크를 위한 FL 개발을 촉진하는 것이다.

## ✨ Key Contributions

이 논문의 핵심 아이디어는 컴퓨터 비전(CV) 분야의 다양한 태스크를 위한 연합 학습(Federated Learning, FL)을 위한 통합 연구 프레임워크인 FedCV를 설계하고 구현하는 것이다. 이는 순수 FL 알고리즘 연구와 실제 CV 애플리케이션 중심 연구를 연결하려는 철학을 기반으로 한다.

주요 기여 사항은 다음과 같다.

1. **다양한 CV 태스크 지원:** FedCV는 이미지 분류, 이미지 분할, 객체 탐지와 같은 세 가지 핵심 컴퓨터 비전 태스크에 대한 FL을 지원한다. 이를 통해 FL 연구의 적용 범위를 확장하고 다양한 CV 문제에 대한 FL의 효능을 평가할 수 있도록 한다.
2. **포괄적인 벤치마크 스위트 제공:** 실제와 유사한 연합 데이터셋을 구성하기 위해 비동일 독립 분포(non-I.I.D.) 데이터셋을 위한 파티셔닝 방법을 제공한다. 또한 각 태스크에 맞는 대표적인 모델(예: EfficientNet, MobileNet, ViT, DeepLabV3+, U-Net, YOLOv5)과 다양한 FL 알고리즘(FedAvg, FedOpt, FedNova, FedProx, FedMA 등)의 표준 구현을 제공하여 공정한 비교와 추가 연구 탐색을 돕는다.
3. **사용자 친화적인 API 및 분산 컴퓨팅 지원:** FedML 라이브러리를 기반으로 하여, 새로운 FL 알고리즘을 분산 컴퓨팅 환경에서 쉽게 구현할 수 있는 유연하고 사용하기 쉬운 API를 제공한다. 이는 다중 GPU 분산 학습을 지원하여 CV 모델의 연합 학습 시간을 합리적인 수준으로 단축시킨다.
4. **도전 과제 식별 및 벤치마킹 결과 제시:** FedCV 프레임워크를 사용하여 광범위한 실험을 수행하고, 연합 학습 환경에서 딥러닝 학습 기법의 직접적인 적용 어려움, non-I.I.D. 데이터셋이 모델 정확도에 미치는 영향, 그리고 대규모 모델의 파라미터 수와 클라이언트당 메모리 비용으로 인한 시스템 효율성 개선의 어려움 등 향후 연구가 필요한 여러 도전 과제를 식별한다.

이러한 기여를 통해 FedCV는 연구자들이 컴퓨터 비전과 연합 학습의 교차점에서 모델 개선, 시스템 최적화, 연합 최적화 방법 등 다양한 연구 주제를 탐색할 수 있는 사용하기 쉬운 플랫폼 역할을 할 것으로 기대된다.

## 📎 Related Works

논문은 FedCV의 관련 연구로 연합 학습을 이미지 데이터셋에 적용한 초기 연구와 특정 컴퓨터 비전 태스크에 대한 기존 연합 학습 접근 방식을 소개한다.

1. **초기 연합 이미지 분류 연구:**
    * [33]은 연합 학습을 실제 이미지 데이터셋인 Google Landmark [78]에 처음 적용한 연구이다. Google Landmark는 현재 연합 학습 연구의 표준 이미지 데이터셋으로 자리 잡았다.

2. **의료 영상 분할에서의 연합 학습:**
    * [6, 41, 43]은 의료 영상 분할 태스크에 연합 학습을 적용하여, 데이터 프라이버시 규정으로 인해 단일 의료 기관에서 학습 데이터를 사용할 수 없는 문제를 해결하는 데 중점을 둔다.

3. **객체 탐지에서의 연합 학습:**
    * [84]는 non-I.I.D. 데이터로 인한 모델 정확도 손실을 완화하기 위해 KL 발산(KL divergence) 방법을 제안한다.

4. **FedVision과의 차별점:**
    * FedCV는 FedVision [49]과 밀접한 관련이 있지만, FedVision은 스마트 시티 시나리오에서 FastRCNN 및 YOLOv3와 같은 모델을 사용한 객체 탐지를 지원하는 연합 학습 프레임워크이다.
    * **FedVision의 한계:** FedVision은 FedAvg 알고리즘과 단일 GPU 학습만을 지원한다.
    * **FedCV의 차별점:** FedCV는 **다양한 컴퓨터 비전 태스크(이미지 분류, 이미지 분할, 객체 탐지)를 지원하고, FedAvg 외에도 여러 FL 알고리즘(FedOpt, FedNova, FedProx, FedMA, FedGKT, Decentralized FL, VFL, Split Learning, FedNAS, Turbo-Aggregate)을 제공하며, 다중 GPU 분산 학습을 지원**하여 계산 비용을 절감하고 연구의 폭을 넓힌다.

논문은 다른 애플리케이션 도메인의 연합 학습에 대해서는 포괄적인 비전 논문 [37]을 참고하도록 한다. 이러한 관련 연구들을 통해 FedCV가 기존 접근 방식들의 한계를 극복하고 컴퓨터 비전 분야의 연합 학습을 위한 보다 포괄적이고 유연한 플랫폼을 제공한다는 점을 강조한다.

## 🛠️ Methodology

### 연합 학습 (Federated Learning, FL) 개요 및 목표 함수

연합 학습은 분산 학습 패러다임으로, 분산되고 고립된 데이터셋을 활용하여 각 클라이언트(참여자)를 위한 전역 또는 개인화된 모델을 학습하는 동시에 개인 정보 보호, 규제 준수, 통신 및 저장 비용 절감을 달성한다. 모든 클라이언트가 전역 모델을 협력적으로 학습해야 한다고 가정할 때, 가장 직접적인 목적 함수는 다음과 같다.

$$
\min_{W} F(W) \stackrel{\text{def}}{=} \min_{W} \sum_{k=1}^{K} \frac{N^{(k)}}{N} \cdot f^{(k)}(W) \\
f^{(k)}(W) = \frac{1}{N^{(k)}} \sum_{i=1}^{N^{(k)}} \ell(W; X_i, y_i)
$$

여기서:

* $W$: 전역 모델의 가중치 또는 매개변수이다. 컴퓨터 비전에서는 CNN 또는 Transformer 모델(예: ViT)이 될 수 있다.
* $K$: 전체 클라이언트의 수이다.
* $N^{(k)}$: $k$번째 클라이언트의 로컬 데이터셋 $D_k$에 있는 데이터 샘플의 수이다.
* $N$: 전체 클라이언트의 모든 데이터 샘플 수의 합, 즉 $N = \sum_{k=1}^{K} N^{(k)}$ 이다.
* $f^{(k)}(W)$: $k$번째 클라이언트의 로컬 목적 함수이며, 이질적인 데이터셋 $D_k$에 대한 로컬 경험적 위험을 측정한다.
* $\ell(W; X_i, y_i)$: 전역 CNN 모델의 손실 함수이다. 이미지 분류 태스크의 경우 교차 엔트로피(cross-entropy) 손실이 사용된다.

### FedAvg 알고리즘

이러한 연합 최적화 문제를 해결하기 위해, FedAvg (Federated Averaging)는 FL 개념을 처음 제안한 연합 최적화 알고리즘이다. FedAvg의 최적화 과정은 다음과 같다.

알고리즘 1: FedAvg 알고리즘

1. **초기화 (Initialization):** 네트워크에 여러 클라이언트가 있으며, 각 클라이언트 $k$는 로컬 데이터셋 $D_k$를 가진다. 각 클라이언트의 로컬 모델은 $W_0$로 초기화된다.
2. **서버 실행 (Server_Executes):**
    * 각 라운드 $t = 0, 1, 2, \ldots$ 에 대해 다음을 반복한다.
        * $S_t \leftarrow$ (무작위로 클라이언트 집합을 샘플링한다.)
        * 병렬로 $S_t$의 각 클라이언트 $k$에 대해 다음을 수행한다.
            * $W_k^{t+1} \leftarrow \text{ClientUpdate}(k, W_t)$ (클라이언트는 서버로부터 받은 현재 전역 모델 $W_t$를 사용하여 로컬 업데이트를 수행한다.)
        * $W_{t+1} \leftarrow \sum_{k=1}^{K} \frac{n_k}{n} W_k^{t+1}$ (서버는 선택된 클라이언트로부터 업데이트된 로컬 모델 $W_k^{t+1}$를 수신하고, 이들을 데이터셋 크기 $n_k$에 비례하여 가중 평균하여 새로운 전역 모델 $W_{t+1}$를 생성한다.)
3. **클라이언트 훈련 (Client_Training$(k, W)$):** (클라이언트 $k$에서 실행)
    * $B \leftarrow$ ($D_k$를 배치(batch)로 분할한다.)
    * 각 로컬 에포크 $i$ ($i = 1, 2, \ldots$)에 대해 다음을 반복한다.
        * 각 배치 $b \in B$에 대해 다음을 수행한다.
            * $W \leftarrow W - \eta \nabla_W F(W; b)$ (로컬 데이터 $b$에 대한 경사 하강법을 사용하여 로컬 모델 $W$를 업데이트한다. 여기서 $\eta$는 학습률이다.)
        * $W$를 서버로 반환한다.

#### FL과 기존 분산 학습의 주요 차이점

* **데이터 이질성 (Non-I.I.D. Data) 및 레이블 부족:** FL에서는 엣지 기기에서 데이터가 생성되므로 비동일 독립 분포(non-I.I.D.) 특성을 가진다. 예를 들어, 스마트폰 사용자는 기기나 행동의 차이로 인해 다양한 해상도, 품질, 콘텐츠의 이미지/비디오를 생성한다. 또한, 사생활 문제로 인해 개인 데이터를 레이블링하도록 유도하는 것이 어렵다.
* **시스템 제약 및 이질성:** 엣지 기기는 메모리, 대역폭, GPU 가속기 부족 등 시스템 능력이 이질적이며 제한적이다. 이는 대규모 DNN 모델 학습에 큰 어려움을 초래한다.
* **강건성 (Robustness) 및 개인 정보 보호 (Privacy):** FL 환경은 전통적인 데이터 센터 환경보다 데이터 및 모델 오염 공격에 취약하다. 또한, 클라이언트와 서버 간에 교환되는 기울기(gradient)가 어느 정도 프라이버시 유출로 이어질 수 있다는 연구 결과가 있어, 추가적인 개인 정보 보호 기술이 필요하다.

### FedCV 시스템 아키텍처 및 기여

이러한 CV 태스크의 도전 과제를 해결하기 위해 FedCV는 유연하고 효율적인 분산 학습 프레임워크를 목표로 한다. FedCV는 FedML 연구 라이브러리 [27]를 기반으로 구축되었으며, FedML이 이미지 분류, ResNet 및 간단한 CNN 모델만 지원했던 한계를 극복한다.

**FedCV의 주요 기여 모듈:**

* **다양한 CV 태스크를 위한 벤치마크 스위트 (Benchmark Suite for Diverse CV Tasks):**
  * **지원 태스크:** 이미지 분류(Image Classification), 이미지 분할(Image Segmentation), 객체 탐지(Object Detection).
  * **데이터셋 및 데이터 로더:** 관련 데이터셋과 데이터 로더를 제공한다. 사용자는 제공된 데이터 분포를 사용하거나 하이퍼파라미터 설정을 통해 non-I.I.D. 분포를 직접 조작할 수 있다.
  * **모델:** 벤치마크 평가를 위해 선별된 모델들을 제공한다.

* **대표 FL 알고리즘을 위한 참조 구현 (Reference Implementation for Representative FL Algorithms):**
  * **표준 FL 알고리즘:** FedAvg [52], FedOpt (server Adam) [58], FedNova (client optimizer) [75], FedProx [62], FedMA [73] 등 여러 최신 FL 알고리즘의 표준 구현을 포함한다.
  * **다양한 훈련 패러다임 및 네트워크 토폴로지 알고리즘:** FedGKT (efficient edge training) [24], Decentralized FL [29], Vertical Federated Learning (VFL) [82], Split Learning [20, 72], Federated Neural Architecture Search (FedNAS) [23], Turbo-Aggregate [65] 등을 지원한다.
  * **다중 GPU 분산 학습:** 이러한 알고리즘들은 다중 GPU 분산 학습을 지원하여 합리적인 시간 내에 훈련을 완료할 수 있도록 한다.

* **알고리즘 사용자 정의를 위한 쉬운 API (Easy-to-use APIs for Algorithm Customization):**
  * FedML API 설계를 활용하여 다양한 네트워크, 워커/클라이언트 간의 유연한 정보 교환, 다양한 훈련 절차를 가능하게 한다. 새로운 FL 알고리즘을 분산 컴퓨팅 환경에서 쉽게 구현할 수 있도록 한다.

* **기타 기능 (Other Functionality):**
  * **다중 클라이언트 단일 GPU 로딩:** 연구자들이 더 적은 GPU로 더 많은 클라이언트를 확장할 수 있도록 한다.
  * **Tensor-aware RPC:** 다른 데이터 센터(예: 다른 의료 기관)에 위치한 서버 간 통신을 가능하게 한다.
  * **보안 및 개인 정보 보호 모듈:** 안전한 집계(secure aggregation)와 같은 기술을 지원하는 향상된 보안 및 개인 정보 보호 기본 모듈을 추가한다.

### FedCV 벤치마크 스위트: 데이터셋, 모델 및 알고리즘

**1. 데이터셋:**

* **이미지 분류:**
  * **Google Landmarks Dataset 23k (GLD-23K) [78]:** 스마트폰 사용자로부터 얻은 자연스러운 연합 데이터셋으로, TensorFlow Federated [19]에서 제안된 203개 클래스, 233개 클라이언트, 23080개 이미지를 포함한다.
  * **CIFAR-100 [39]:** non-I.I.D. 분할이 적용된 100개 클래스의 데이터셋이다.
* **이미지 분할:**
  * **증강된 PASCAL VOC [22]:** 20개의 전경 객체 클래스와 1개의 배경 클래스를 포함하는 11355개의 이미지로 구성된다.
* **객체 탐지:**
  * **COCO [48]:** 일상생활의 객체를 탐지하고 분할하기 위한 실제 이미지를 포함하는 데이터셋이다.

**2. 모델:**

* **이미지 분류:**
  * **EfficientNet [67], MobileNet-V3 [32]:** 경량 CNN 모델.
  * **Vision Transformer (ViT) [12]:** 어텐션 기반 Transformer 모델 (ViT-B/16).
* **이미지 분할:**
  * **DeepLabV3+ [?], U-Net [55]:** 대표적인 이미지 분할 모델. 백본으로 ResNet-101 및 MobileNet-V2를 활용한다.
* **객체 탐지:**
  * **YOLOv5 [36]:** YOLOv4 [3]의 최적화된 버전으로, 네 가지 네트워크 모델(YOLOv5s, YOLOv5m, YOLOv5l, YOLOv5x)을 사전 학습된 모델로 사용한다.

**3. Non-I.I.D. 전처리:**

* **LDA (Latent Dirichlet Allocation) [71]:** CIFAR-100 및 PASCAL VOC 데이터셋에 적용되어 합성(synthetic) 연합 데이터셋을 얻는 데 사용되는 일반적인 FL 연구 방법이다. Figure 3에 예시가 시각화되어 있다.
* **COCO 데이터셋 특수 분할:** 객체 탐지 태스크는 하나의 그림에 여러 레이블이 있는 경우가 많으므로 LDA 대신 다른 분할 방법을 사용한다.
    1. 모든 이미지에서 각 객체의 빈도를 계산한다 (하나의 객체는 한 이미지에서 한 번만 카운트).
    2. 객체 빈도를 정렬하고 가장 높은 빈도의 객체를 한 클라이언트에 할당한다.
    3. 남은 이미지를 정렬하고 모든 이미지가 클라이언트에 할당될 때까지 첫 번째 단계를 반복한다. Figure 10은 COCO 데이터셋의 non-I.I.D. 분포를 보여준다.

**4. FL 알고리즘:**

* **지원 알고리즘:** FedAvg, FedOpt 등 다양한 대표 알고리즘을 지원한다. (전체 목록은 부록에 제공됨)

**5. 훈련 설정:**

* **옵티마이저:** FedAvg의 바닐라 SGD와 달리 모멘텀 SGD(Momentum SGD)를 사용한다. 중앙 집중식 훈련의 모멘텀 SGD는 전통적인 모멘텀 SGD와 동일하지만, FedAvg에서는 클라이언트가 서버로부터 전역 모델을 받을 때 누적된 기울기가 초기화된다. 모멘텀 계수는 모든 실험에서 0.9이다.
* **학습률 스케줄러 (Learning Rate Scheduler):** 모든 태스크의 중앙 집중식 훈련에는 각 에포크마다 선형 학습률 감소(linear learning rate decay)를 사용한다. FedAvg에서는 기본적으로 학습률 스케줄러를 사용하지 않지만, 이미지 분류 태스크에서 그 효과를 평가하기 위해 시도한다. 학습률 감소는 로컬 에포크가 아닌 통신 라운드를 기반으로 한다.
* **FedAvg 훈련 설정:** 이미지 분류에서는 라운드당 10개 클라이언트, 1 로컬 에포크. 이미지 분할 및 객체 탐지에서는 라운드당 4 또는 8개 클라이언트, 1 또는 2 로컬 에포크를 사용한다. (Table 10 참조)
* **이미지 변환 (Image Transform):** 연합 학습에서는 클라이언트가 다른 클라이언트의 데이터 분포를 알 수 없다고 가정하므로, 모든 이미지의 평균 및 표준 편차 대신 평균 RGB 값과 표준 편차를 각각 [0.5, 0.5, 0.5]로 설정한다.
* **레이어별 학습 (Layer-wise Learning):** 이미지 분할 모델은 인코더-디코더 아키텍처를 가지며, 인코더 레이어에는 ImageNet-1K 사전 학습된 백본을 사용하고 디코더 레이어는 처음부터 학습한다. 이를 위해 디코더 레이어의 학습률을 인코더 레이어보다 10배 높게 설정한다.
* **데이터 증강 (Data Augmentation):** 연합 학습 환경에서는 서버나 클라이언트가 전체 데이터 분포를 알 수 없으므로 AutoAugment [9]나 GAN [18]과 같은 고급 데이터 증강 기술을 직접 사용할 수 없다. 이는 성능 저하로 이어질 수 있다.

### 전체 파이프라인 또는 시스템 구조

FedCV의 시스템 아키텍처는 Figure 2에 요약되어 있다. 이는 클라이언트 측과 서버 측으로 구성되며, 각각 데이터 로더, 모델, 알고리즘 구현, API 모듈 등을 포함한다. FedCV는 FedML-core APIs를 재사용하지만, Tensor-aware RPC 및 향상된 보안/프라이버시 기본 모듈을 추가하여 기능을 확장한다. 클라이언트들은 로컬에서 모델을 훈련하고 업데이트를 서버로 보낸 후, 서버는 이 업데이트들을 집계하여 새로운 전역 모델을 생성하고 클라이언트들에게 배포하는 FedAvg와 유사한 순환적 과정을 따른다.

### 6.1. 이미지 분류 (Image Classification)

**구현 세부 사항:**

* 클라이언트 수: 라운드당 10명.
* 하드웨어: GTX 2080Ti 컴퓨팅 클러스터, 각 클라이언트 1 GPU, 통신 대역폭 10 Gbps.
* 모델: EfficientNet [67], MobileNet V3 [32], ViT [12].
* 데이터셋: CIFAR-100 [39], GLD-23K [78] [19].
* 중앙 집중식 학습 배치 크기: 256. FedAvg 배치 크기: 32.
* 옵티마이저: 모멘텀 SGD (모멘텀 계수 0.9).
* 학습률 스케줄러: 중앙 집중식은 선형 학습률 스케줄러 (step size 0.97), FedAvg는 스케줄러 없음 (일부 실험에서 평가).

**실험 결과:**

* **GLD-23k NonIID vs. IID:** Figure 4(a)에 따르면, EfficientNet 및 MobileNet의 중앙 집중식 훈련 테스트 정확도가 FedAvg 훈련보다 10% 더 높게 나타난다. ViT의 경우 중앙 집중식 훈련 정확도는 FedAvg와 유사하다. 이는 FedAvg가 중앙 집중식 훈련에 비해 여전히 성능 격차가 있음을 보여준다.
* **Non-I.I.D. 정도의 영향:** Figure 4(b) 및 부록의 Figure 15는 non-I.I.D. 정도가 EfficientNet 및 MobileNetV3의 훈련 성능에 미치는 영향을 보여준다. $\alpha$ 값이 높을수록(non-I.I.D. 정도가 낮을수록) 테스트 정확도가 증가한다. 이는 LDA [71] 결과와 일치한다.
* **미세 조정(Fine-tuning) vs. 처음부터 훈련(Training from scratch):** Figure 4(b), 4(c), Figure 15에 따르면, 미세 조정의 성능이 처음부터 훈련하는 것보다 훨씬 효과적이다. 수렴 속도 면에서 미세 조정은 60%의 테스트 정확도를 달성하는 데 처음부터 훈련하는 것보다 약 20배 빠르며, 최종 정확도에서도 20%가량 더 우수하다.
* **모멘텀 SGD vs. SGD:** Figure 4(d) 및 부록 Figure 14(a)-(b)에 따르면, 모멘텀을 사용한 SGD가 바닐라 SGD보다 항상 더 나은 성능을 보장하지는 않는다. EfficientNet을 CIFAR-100($\alpha=0.5$)에 적용했을 때, 미세 조정 시 모멘텀 SGD와 SGD는 유사한 성능을 보이지만, 처음부터 훈련할 경우 모멘텀 SGD가 훨씬 높은 테스트 정확도를 보인다. $\alpha=0.1$에서는 모멘텀 SGD 성능이 미세 조정에 의해 크게 영향을 받지 않지만, 바닐라 SGD는 상당한 개선을 보인다. GLD-23K에서는 모멘텀 SGD가 SGD보다 성능이 나빴다. 이는 모멘텀 SGD의 누적 기울기가 로컬 클라이언트의 다양한 최적 모델 매개변수 방향으로 지나치게 이동하게 하여 모델 매개변수의 다양성을 증가시킬 수 있기 때문으로 분석된다.
* **학습률 스케줄러:** Figure 5 및 부록 Figure 14(c)-(d)는 선형 학습률 감소가 성능을 개선하지 못하거나 심지어 저하시킬 수 있음을 보여준다. 한 가지 이유는 마지막 훈련 에포크에서 너무 작은 학습률로 인해 각 클라이언트가 수렴하지 못할 수 있기 때문이다. 그러나 학습률 감소는 훈련 과정을 더 안정적으로 만드는 경향이 있다.
* **효율성 분석:** Table 3은 세 모델의 시스템 성능을 요약한다. ViT와 같은 대규모 딥러닝 모델을 연합 설정에서 훈련할 경우, 작은 모델에 비해 엄청난 통신 오버헤드가 발생함을 보여준다.

### 6.2. 이미지 분할 (Image Segmentation)

**구현 세부 사항:**

* 클라이언트 수: 라운드당 4명 또는 8명.
* 하드웨어: Quadro RTX 5000 그래픽 카드 클러스터, 각 클라이언트 1 GPU, 통신 대역폭 15.754 GB/s.
* 모델: DeepLabV3+, U-Net.
* 백본: ResNet-101, MobileNetV2.
* 백본 훈련 유형 (TT): 미세 조정 (FT), 백본 동결 (FZ), 처음부터 연합 학습 (SFL).
* 메트릭: mIoU (mean Intersection over Union).

**실험 결과:**

* **백본 훈련 vs. 미세 조정:** Figure 6(a)에 따르면, 사전 학습된 백본을 미세 조정하는 것이 처음부터 훈련하는 것보다 약간(2% 미만) 더 나은 성능을 보인다. 이는 연합 이미지 분할 정확도에 사전 학습된 백본이 도움이 되지만 필수적이지는 않음을 시사한다. 이는 사전 학습된 백본이 유용하지 않을 수 있는 의료 영상과 같은 고급 태스크에서 처음부터 끝까지 훈련하는 것이 유일한 대안임을 보여준다.
* **배치 크기 vs. 메모리 트레이드오프:** Figure 6(b) 및 Table 6은 배치 크기를 10에서 4로 줄이면 메모리가 약 절반으로 줄어들지만, 정확도는 거의 2% 손실된다는 것을 보여준다. 이는 연합 학습 설정에서 엣지 기기의 메모리 제약을 고려할 때 중요한 트레이드오프이다.
* **데이터 분포의 영향 분석:** Figure 7(a)는 $\alpha$ 값에 따른 클래스 분포를 보여주는데, 파티션 인자가 완전히 동질적인 경우에서 매우 이질적인 경우로 변하더라도 정확도는 약 2%만 감소한다. 이는 연합 분할 학습이 로컬 클라이언트에 충분한 일반화 능력을 부여하여 알 수 없는 데이터에서도 잘 수행될 수 있음을 보여주며, 중앙 집중식 또는 광범위하게 분산된 데이터의 필요성을 없앤다.
* **클라이언트 수 증가에 대한 회복력:** 클라이언트 수가 증가할수록 모델이 수렴하는 데 필요한 라운드 수가 증가한다 (Figure 7(b)). 이론적으로 각 로컬 클라이언트가 더 많은 데이터 포인트를 훈련하기 때문에 더 적은 클라이언트 수가 더 나은 성능을 보일 것으로 예상되지만, 충분한 라운드 후에는 최종 정확도에 거의 영향을 미치지 않는다.
* **시스템 성능 분석:** ResNet은 이미지 분할 태스크에서 인코더-디코더 아키텍처에 널리 사용되지만, 높은 컴퓨팅 비용을 가진다. 비용을 줄이는 두 가지 방법은 사전 학습된 백본을 동결하거나(FZ), 계산 효율적인 백본(예: MobileNetV2)을 사용하는 것이다. Figure 8은 이러한 전략을 DeepLabV3+ 및 U-Net 아키텍처에 적용했을 때의 성능 변화를 보여준다. ResNet-101 백본의 정확도가 다른 대안보다 확연히 높지만, Table 7에서 보듯이 대안들은 성능 저하를 감수하고 매우 효율적이다.

### 6.3. 객체 탐지 (Object Detection)

**구현 세부 사항:**

* 모델: 사전 학습된 YOLOv5.
* 알고리즘: FedAvg.
* 클라이언트 수: 4명 및 8명.
* 하드웨어: 각 클라이언트 1 GPU (NVIDIA V100).
* 메트릭: mAP@0.5 (mean Average Precision, IOU 임계값 0.5).

**실험 결과:**

* **학습률:** Figure 9(a)는 연합 설정에서 다양한 학습률을 평가한 결과, $lr=0.01$이 다른 학습률보다 더 나은 결과를 보인다는 것을 보여준다.
* **Non-I.I.D. 평가:** YOLOv5를 사용할 때 FedAvg가 non-I.I.D. 데이터셋에 대해 중앙 집중식 학습과 동일한 결과를 달성하기 어렵다는 것을 발견했다. * **다른 클라이언트 수에 대한 평가:** Figure 9(b)는 4개 모델에서 클라이언트 수가 다를 때의 성능을 보여준다. $C=8$일 때 $C=4$보다 성능이 낮게 나타난다.

## 🧠 Insights & Discussion

### 논문에서 뒷받침되는 강점

FedCV는 컴퓨터 비전과 연합 학습 사이의 중요한 연구 격차를 해소하려는 시도로서 여러 강점을 가진다.

* **포괄적인 벤치마크 제공:** FedCV는 이미지 분류, 분할, 객체 탐지와 같은 세 가지 주요 CV 태스크에 대한 포괄적인 벤치마크를 제공한다. 이는 FL 연구가 주로 이미지 분류에 한정되어 있던 기존의 한계를 넘어선다. 다양한 데이터셋, 모델, 그리고 FL 알고리즘의 구현은 연구자들이 광범위한 시나리오에서 FL의 성능을 체계적으로 평가하고 비교할 수 있는 기반을 마련한다.
* **실제적인 도전 과제 반영:** 논문은 non-I.I.D. 데이터 분포, 시스템 이질성, 개인 정보 보호 및 보안과 같은 연합 학습의 핵심 도전 과제들을 명확히 인식하고 이를 벤치마크 설계에 반영한다. 특히 non-I.I.D. 데이터셋을 생성하는 방법론과 그 영향에 대한 실험은 실제 FL 환경의 복잡성을 이해하는 데 기여한다.
* **사용자 친화성과 확장성:** FedML을 기반으로 구축된 FedCV는 사용하기 쉬운 API와 다중 GPU 분산 학습 지원을 통해 연구자들이 새로운 FL 알고리즘을 개발하고 대규모 CV 모델을 효율적으로 훈련할 수 있도록 돕는다. 이는 FL 연구의 진입 장벽을 낮추고, 알고리즘 구현에서 발생하는 엔지니어링 부담을 줄여준다.
* **실험적 통찰력 제시:** 광범위한 벤치마킹 실험을 통해, 미세 조정의 중요성, 모멘텀 SGD 및 학습률 스케줄러의 복잡한 영향, 그리고 대규모 모델의 통신 오버헤드와 같은 중요한 실용적 통찰력을 제공한다. 이러한 결과는 향후 FL 알고리즘 및 시스템 설계에 대한 구체적인 방향을 제시한다.

### 한계, 가정 또는 미해결 질문

논문은 FedCV를 통해 여러 도전 과제를 식별하며, 이는 FedCV 자체의 한계 또는 향후 연구 방향을 시사한다.

* **중앙 집중식 학습 기법의 직접 적용 한계:** 논문의 실험 결과(특히 이미지 분류 및 객체 탐지)는 중앙 집중식 훈련에서 효과적인 딥러닝 훈련 기법(예: 특정 옵티마이저, 학습률 스케줄러)이 FL 환경에서 직접적으로 적용될 수 없거나 예상치 못한 성능 저하를 초래할 수 있음을 명확히 보여준다. 이는 FL을 위한 맞춤형 최적화 기법 개발이 시급함을 의미한다.
* **Non-I.I.D. 데이터의 성능 저하:** non-I.I.D. 데이터셋은 모델 정확도를 저하시키는 경향이 있음을 확인했으며, 특히 객체 탐지 태스크에서 중앙 집중식 학습과의 큰 성능 격차를 보였다. 이는 데이터 이질성 문제에 대한 더 강력한 해결책이 필요함을 강조한다.
* **시스템 효율성 문제:** 대규모 모델(예: ViT, YOLOv5l/x)의 경우, 방대한 매개변수 수와 클라이언트당 메모리 비용으로 인해 연합 훈련의 시스템 효율성을 개선하는 것이 매우 어렵다. 이는 통신 오버헤드를 줄이고 엣지 기기의 리소스 제약을 완화하는 효율적인 FL 시스템 설계의 필요성을 제기한다.
* **옵티마이저 및 학습률 스케줄러의 최적화:** 모멘텀 SGD가 SGD보다 항상 우수하지 않으며, 선형 학습률 감소가 오히려 성능을 저하시킬 수 있다는 발견은 FL 환경에 최적화된 옵티마이저 및 학습률 스케줄러 설계가 복잡한 문제임을 나타낸다. 특히 로컬 그라디언트 축적의 이질성 문제와 모델 발산 가능성은 미해결 질문으로 남는다.
* **이미지 변환 및 데이터 증강의 어려움:** FL 설정에서는 클라이언트가 전체 데이터 분포를 알 수 없으므로, 적합한 이미지 변환 및 AutoAugment, GAN과 같은 고급 데이터 증강 기술을 직접 적용하기 어렵다. 이는 FL 환경에서 데이터 전처리 및 증강 전략에 대한 새로운 접근 방식이 필요함을 시사한다.
* **제한적인 알고리즘 평가 범위:** FedCV는 다양한 FL 알고리즘을 구현했지만, 보고서의 실험 섹션에서는 주로 FedAvg와 관련된 결과만을 상세히 다룬다. 다른 알고리즘들이 제시된 도전 과제에 대해 어떤 성능을 보이는지, 그리고 FedAvg 대비 어떤 장단점을 가지는지에 대한 심층적인 분석은 부족하다.

### 논문에 근거한 간략한 비판적 해석 및 논의사항

FedCV는 컴퓨터 비전 분야의 연합 학습 연구를 위한 필수적인 인프라를 구축했다는 점에서 매우 중요한 기여를 한다. 이 벤치마크는 기존 FL 연구의 주된 한계였던 이미지 분류 중심의 소규모 데이터셋 문제를 해결하고, 실제 애플리케이션에 가까운 객체 탐지 및 이미지 분할 태스크로 연구의 지평을 넓혔다. 특히 non-I.I.D. 데이터 환경에서 미세 조정의 효과를 강조하고, 중앙 집중식 학습 기법이 FL에서 항상 잘 작동하지 않음을 실증적으로 보여준 것은 향후 연구 방향을 설정하는 데 큰 통찰력을 제공한다.

그러나 논문에서 제시된 결과들은 FL이 여전히 중앙 집중식 학습에 비해 성능 격차를 보이며, 특히 non-I.I.D. 데이터와 대규모 모델 환경에서 여러 가지 해결해야 할 과제를 안고 있음을 명확히 보여준다. 옵티마이저와 학습률 스케줄러에 대한 흥미로운 발견은 기존 딥러닝의 "모범 사례"가 FL에서는 재고되어야 함을 시사한다. 이는 FL이 단순히 분산된 환경에서 모델을 학습하는 것을 넘어, 데이터 이질성, 시스템 제약, 그리고 개인 정보 보호라는 고유한 특성을 고려한 새로운 이론적, 알고리즘적 기초가 필요하다는 점을 강조한다. FedCV는 이러한 새로운 연구를 위한 발판을 마련했지만, 궁극적으로는 이 프레임워크를 활용하여 위에서 언급된 미해결 질문들을 탐구하고 실질적인 해결책을 제시하는 것이 다음 단계의 과제가 될 것이다.

## 📌 TL;DR

FedCV는 컴퓨터 비전(CV) 분야의 연합 학습(FL) 연구를 촉진하기 위해 제안된 포괄적인 라이브러리 및 벤치마크 프레임워크이다. 이 연구는 FL이 중앙 집중식 학습에 비해 CV 태스크, 특히 객체 탐지 및 이미지 분할에서 뒤처져 있다는 문제의식에서 출발한다. FedCV는 이미지 분류, 이미지 분할, 객체 탐지 등 세 가지 대표적인 CV 태스크를 지원하며, non-I.I.D. 데이터셋, EfficientNet, ViT, DeepLabV3+, YOLOv5와 같은 최첨단 모델들, 그리고 FedAvg, FedOpt 등 다양한 FL 알고리즘의 참조 구현을 제공한다. 또한, 유연한 API와 다중 GPU 분산 학습 지원을 통해 연구자들이 새로운 FL 알고리즘을 쉽게 개발하고 대규모 CV 모델을 효율적으로 훈련할 수 있도록 한다.

벤치마크 연구 결과는 다음과 같은 주요 도전 과제를 식별한다: (1) 중앙 집중식 훈련 기법이 FL에 직접 적용되기 어려움, (2) non-I.I.D. 데이터셋이 모델 정확도를 저하시킴, (3) 대규모 모델의 매개변수 수와 클라이언트당 메모리 비용으로 인해 연합 훈련의 시스템 효율성 개선이 어려움. 특히, 미세 조정(fine-tuning)이 처음부터 훈련하는 것보다 훨씬 효과적이며, 모멘텀 SGD나 선형 학습률 스케줄러가 FL 환경에서 항상 성능 개선을 보장하지 않는다는 실증적 통찰력을 제공한다.

이 연구는 CV 태스크를 위한 FL의 격차를 해소하고, 모델, 시스템, 연합 최적화 방법 개선과 같은 미래 연구 방향에 대한 중요한 기반을 제공하여 실제 애플리케이션 및 향후 연구에 핵심적인 역할을 할 가능성이 높다.
