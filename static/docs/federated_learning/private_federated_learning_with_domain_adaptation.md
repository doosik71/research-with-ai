# Private Federated Learning with Domain Adaptation

Daniel W. Peterson, Pallika Kanani, Virendra J. Marathe (2019)

## 🧩 Problem to Solve

이 논문은 분산 머신러닝 패러다임인 Federated Learning (FL)이 가진 두 가지 주요 문제점을 해결하고자 한다. 첫째, FL은 여러 참여자가 데이터를 공유하지 않고 공유 모델을 재훈련할 수 있어 확장성과 개인 정보 보호에 이점을 제공하지만, 개별 사용자의 데이터가 다양하고 이질적인 도메인 분포를 가질 때 모델의 정확도 저하가 발생할 수 있다. 이는 각 사용자가 고유한 데이터 생성 과정을 가질 수 있기 때문에 흔히 발생하는 문제이다. 둘째, 데이터 개인 정보 보호를 강화하기 위해 FL에 차분 프라이버시(Differential Privacy, DP)를 적용할 경우, 훈련 과정에 의도적인 노이즈가 주입되어 모델 예측 정확도가 저해될 수 있다. 특히, 이러한 노이즈는 때로는 비협력적(non-collaborative)인 단독 학습 모델보다 FL 모델의 성능이 떨어지게 만들 수 있다.

따라서 이 논문의 목표는 FL 환경에서 개인 정보 보호를 유지하면서도, 개별 사용자 도메인의 다양성으로 인해 발생하는 정확도 저하를 완화하고, 특히 차분 프라이버시 노이즈가 모델 성능에 미치는 부정적인 영향을 최소화하여 모든 사용자에게 향상된 모델 정확도를 제공하는 프레임워크를 개발하는 것이다.

## ✨ Key Contributions

이 논문의 핵심 아이디어는 개인 정보 보호 FL을 통해 학습된 공용(generalist) 모델과 각 사용자의 개별 도메인에 특화된 사설(private) 모델을 결합하는 것이다. 이러한 아이디어를 구현하기 위해 Mixture of Experts (MoE) 아키텍처를 사용하여 두 모델의 출력을 가중 평균 방식으로 결합하며, 각 사용자는 자신만의 가중치(gating function)를 학습한다.

이 접근 방식의 중심적인 직관은 다음과 같다.

1. **도메인 적응(Domain Adaptation)**: 각 사용자가 자신의 고유한 데이터 도메인에 특화된 사설 모델을 가짐으로써, 이질적인 데이터 분포로 인한 성능 저하를 줄이고 개별 사용자의 정확도를 향상시킨다.
2. **노이즈 없는 업데이트**: 사설 모델은 다른 파티와 공유되지 않고 오직 해당 사용자의 로컬 데이터로만 학습되므로, 차분 프라이버시와 관련된 노이즈 없이 업데이트될 수 있다. 이는 차분 프라이버시가 적용된 공용 모델의 성능 저하를 상쇄하는 중요한 역할을 한다.
3. **유연한 모델 결합**: MoE 기반의 가중치 함수(gating function)를 통해 공용 모델과 사설 모델 중 어떤 모델의 예측을 특정 입력에 대해 더 신뢰할지 동적으로 학습함으로써, 각 모델의 강점을 최대한 활용하고 부드러운 예측을 가능하게 한다.

이 시스템을 'federated learning with domain experts (FL+DE)'라고 명명하며, 차분 프라이버시가 적용된 FL 모델의 정확도를 크게 능가함을 보인다.

## 📎 Related Works

이 논문은 기존 연구들이 다루었던 Federated Learning, 차분 프라이버시, 그리고 도메인 적응 분야의 교차점에 위치한다.

* **Federated Learning (FL)**: 데이터를 공유하지 않고 여러 파티가 공유 모델을 공동으로 훈련하는 분산 머신러닝 패러다임이다. 초기에는 수백만 개의 모바일 장치를 대상으로 했으나, 기업 환경에서도 개인 정보 보호 문제로 인해 유용하게 활용될 수 있다.
* **Differential Privacy (DP) in FL**: FL만으로는 데이터 개인 정보 보호를 완전히 보장하기 어렵다는 인식하에, 많은 연구에서 차분 프라이버시를 FL에 도입할 것을 제안했다. 차분 프라이버시는 단일 데이터 포인트의 포함 또는 배제에 따른 모델 출력의 변동에 $\epsilon$이라는 상한을 제공한다. 이를 위해 훈련 과정에 노이즈를 주입하지만, 이는 모델 예측의 정확도를 저하시킬 수 있는 한계가 있다.
* **Domain Adaptation (DA)**: 비-FL 시스템에서 광범위하게 연구된 분야로, 소스 도메인의 데이터셋으로 훈련된 모델을 다른 타겟 도메인의 데이터셋에 적응시켜 성능을 향상시키는 기술이다.
* **기존 접근 방식과의 차별점**:
  * **Daumé와 Marcu의 작업 (7)**: 이들은 세 가지 모델(소스 도메인, 타겟 도메인, 일반 모델)을 훈련하는 것을 제안했지만, 훈련 및 추론 과정이 복잡하고 두 개의 도메인을 대상으로 하는 반면, 이 논문의 접근 방식은 단순한 가중 평균 알고리즘을 사용하며 FL 환경에 더 적합하여 각 참여 파티에 대한 다수의 도메인에 적용 가능하다.
  * **프라이버시-보존 도메인 적응 (12)**: 기존 연구는 고전적인 도메인 적응 설정에서 프라이버시를 보호하는 데 초점을 맞추었으나, FL과 같은 협력적 일반 모델 학습과 결합하지는 않았다.
  * **보안 FL (18)**: 덧셈 동형 암호화(additively homomorphic encryption)를 사용하여 두 파티 FL 환경에서 프라이버시를 보장하지만, $\epsilon$-차분 프라이버시와는 다르며 협력적 일반 모델을 유지하지 않는다.
  * **Attention 기반 FL (14)**: 다른 사용자로부터의 업데이트에 가중치를 부여하여, 극히 이례적인 경사 업데이트를 가진 사용자가 일반 FL 모델에 미치는 방해 효과를 줄이는 데 사용되었다. 이는 일반 모델의 일반화 능력을 향상시키지만, 이례적인 데이터셋을 가진 사용자가 자신의 도메인에서 예측 품질을 향상시키는 데 직접적으로 기여하지는 않는다.
  * **PATE 아키텍처 (23)**: 분산 ML 시스템의 또 다른 클래스로, 참여 파티의 사설 훈련 모델을 사용하여 새로운 사용자를 위한 데이터의 합의 기반 레이블링 소스로 활용한다. 이 접근 방식은 일반 모델을 구축하는 데 사용될 수 있지만, 이 논문에서는 도메인 적응 기술과 결합된 효과는 아직 테스트하지 않았다.

이 논문은 협력적 학습과 사설 도메인 적응 노력을 결합한 선행 연구가 없다는 점을 강조하며, 이러한 조합을 통해 FL의 장점을 극대화하면서도 이질적인 도메인과 강력한 프라이버시 제약 조건 하에서 발생하는 한계를 극복하고자 한다.

## 🛠️ Methodology

FL+DE (Federated Learning with Domain Experts) 시스템은 협력적으로 학습된 공용 모델과 각 사용자의 도메인에 특화된 사설 전문가 모델의 출력을 혼합하는 것을 핵심으로 한다.

### 전체 파이프라인 및 시스템 구조

시스템은 각 참여 파티(사용자)가 독립적인 레이블링된 훈련 예제를 가지고 있으며, 이를 비공개로 유지하기를 원한다는 가정에서 시작한다. 이 사용자들은 협력하여 태스크를 위한 공용 모델을 구축하는 동시에, 각 사용자별로 사설의 도메인 적응 전문가 모델을 유지한다. 최종 예측기는 공용 모델과 사설 모델의 출력에 대한 가중 평균이며, 이 가중치는 Mixture of Experts (MoE) 아키텍처를 사용하여 학습된다.

### 각 주요 구성 요소 및 역할

1. **공용 모델 ($M_G$)**:
    * **역할**: 모든 파티가 공유하며, 모든 데이터에 대해 FL을 사용하여 훈련되는 일반화된 모델이다.
    * **훈련**: 차분 프라이버시가 적용된 SGD (Differentially Private SGD, DP-SGD)를 사용하여 각 파티가 일반 모델 훈련에 기여한다. 이를 통해 모델의 파라미터 $\Theta_G$가 업데이트된다.
    * **특징**: 노이즈가 포함된 업데이트로 인해 개인 정보 보호가 보장되지만, 정확도 손실이 발생할 수 있다.

2. **사설 모델 ($M_{P_i}$)**:
    * **역할**: 각 파티 $i$를 위한 개인 모델로, 해당 파티의 특정 도메인 데이터에 적응한다.
    * **훈련**: 다른 파티와 공유되지 않으며, 해당 파티의 로컬 데이터로만 훈련된다. 따라서 차분 프라이버시 관련 노이즈 없이 일반 SGD를 사용하여 파라미터 $\Theta_{P_i}$가 업데이트된다.
    * **특징**: 공용 모델 $M_G$와 동일한 아키텍처로 초기화되지만, 그라디언트 정보는 다른 파티와 공유되지 않아 노이즈 없는 정확한 업데이트가 가능하다.

3. **가중치 메커니즘 (Gating Function, $\alpha_i(x)$)**:
    * **역할**: MoE 아키텍처의 핵심 요소로, 특정 입력 $x$에 대해 공용 모델 $M_G$와 사설 모델 $M_{P_i}$ 중 어느 모델의 예측을 더 신뢰할 것인지 결정하는 가중치 함수이다.
    * **수식**:
        $$\hat{y}_i = \alpha_i(x) M_G(x, \Theta_G) + (1 - \alpha_i(x)) M_{P_i}(x, \Theta_{P_i})$$
        여기서 $\alpha_i(x)$는 일반적으로 시그모이드 함수 $\sigma(\cdot)$를 사용하여 다음과 같이 정의된다.
        $$\alpha_i(x) = \sigma(w_i^T \cdot x + b_i)$$
        $w_i$와 $b_i$는 학습 가능한 가중치와 편향이다. 이 함수는 사설 모델이 일반 모델보다 선호되는 영역을 학습하며, 경계에서 부드러운 혼합을 가능하게 한다.
    * **훈련**: 공용 모델 및 사설 모델과 함께 SGD를 통해 $w_i$ 및 $b_i$ 파라미터가 업데이트된다.

### 훈련 목표, 손실 함수, 추론 절차 또는 알고리즘 흐름

* **훈련 목표**: 각 사용자 $i$는 자신의 레이블된 데이터 $(x, y)$에 대해 최종 예측 $\hat{y}_i$와 실제 레이블 $y$ 간의 손실을 최소화하는 것을 목표로 한다. 이 손실은 회귀 문제의 경우 RMSE, 분류 문제의 경우 교차 엔트로피와 같은 표준 손실 함수가 될 수 있다.
* **학습 절차**:
    1. **공용 모델 훈련**: 모든 참여 파티가 차분 프라이버시가 적용된 SGD를 사용하여 공용 모델 $M_G$의 파라미터 $\Theta_G$를 업데이트한다. 이는 FL의 표준 방식으로, 각 라운드에서 클라이언트들이 로컬 그라디언트를 계산하고 노이즈를 추가한 후 중앙 서버로 전송하여 모델을 업데이트한다.
    2. **사설 모델 및 Gating Function 훈련**: 각 파티 $i$는 자신의 로컬 데이터만을 사용하여 사설 모델 $M_{P_i}$의 파라미터 $\Theta_{P_i}$와 gating function $\alpha_i(x)$의 파라미터 $w_i, b_i$를 노이즈 없이 일반 SGD를 통해 업데이트한다.
    3. **모델 초기화**: 편의상 사설 모델 $M_{P_i}$는 공용 모델 $M_G$의 정확한 복사본으로 초기화된다.
    4. **반복**: 이 과정은 모델이 수렴할 때까지 반복된다.

* **추론 절차**: 새로운 입력 $x$가 주어졌을 때, 각 파티 $i$는 공용 모델 $M_G(x, \Theta_G)$와 자신의 사설 모델 $M_{P_i}(x, \Theta_{P_i})$의 예측을 계산한다. 그런 다음 학습된 gating function $\alpha_i(x)$를 사용하여 두 예측을 가중 평균하여 최종 예측 $\hat{y}_i$를 도출한다.

이 방법론의 주요 이점은 사설 모델과 가중치 메커니즘이 개별 도메인 적응을 통해 정확도를 향상시키고, 노이즈 없는 업데이트를 허용하여 차분 프라이버시 제약으로 인한 정확도 손실을 방지한다는 점이다. 또한, 각 사용자의 gating function이 일반 모델이 실패하는 "사설" 데이터 포인트에 대해 사설 모델을 더 신뢰하도록 학습함으로써, 이례적인 도메인을 가진 사용자가 일반 모델에 미치는 영향을 줄여 일반 모델의 일반화 능력을 향상시키고 사용자 데이터의 프라이버시를 증가시킬 수 있다.

## 📊 Results

논문은 두 가지 데이터셋(합성 회귀 문제 및 실제 스팸 분류 문제)을 통해 FL+DE (Federated Learning with Domain Experts)의 효과를 입증한다. 주요 가설은 도메인 적응 기술이 FL 환경에서 정확도를 향상시키며, 이는 그라디언트 업데이트에 프라이버시 보호를 위한 노이즈가 추가될 때도 유지된다는 것이다.

### 데이터셋, 작업, 기준선, 지표

* **합성 회귀 문제 (Synthetic Regression Problem)**:
  * **데이터셋**: 두 사용자가 함수 $f(x_1, x_2) = 5x_1 - 2x_2 + 0.5x_2^2$에 대한 선형 모델을 맞추려 한다. 각 사용자의 입력 데이터는 고유한 2D 가우시안 분포에서 샘플링되어 $x_2^2$ 항에 대한 노출도가 다르다. 각 사용자에게 2500개의 훈련 예제, 500개의 검증 예제, 500개의 테스트 예제가 제공된다.
  * **작업**: 테스트 세트에서 Root Mean Squared Error (RMSE)를 최소화한다.
  * **기준선**: 각 사용자가 자신의 훈련 데이터로 단일 선형 모델을 훈련하는 '도메인 전용(domain-only)' 시스템.
  * **지표**: RMSE (낮을수록 좋다).
  * **차분 프라이버시 설정**: 낮은 노이즈 ($\sigma=2$) 및 높은 노이즈 ($\sigma=4$) 조건에서 테스트한다.

* **스팸 분류 문제 (Real-world Spam Classification Problem)**:
  * **데이터셋**: ECML PKDD 2006 Discovery Challenge의 일부로 공개된 실제 스팸 감지 데이터셋. 각 사용자에게는 제한된 양의 데이터가 있으며 (50개의 레이블된 예제, 350개의 테스트 예제), 15명의 사용자가 협력하여 스팸 분류기를 구축하는 시나리오를 시뮬레이션한다.
  * **작업**: 이메일이 스팸인지 여부를 분류한다.
  * **기준선**: 각 사용자가 자신의 인-도메인 데이터만 사용하여 분류기를 훈련하는 '도메인 전용' 시스템.
  * **지표**: 분류 정확도 (높을수록 좋다).
  * **차분 프라이버시 설정**: 낮은 노이즈 ($\sigma=0.5$) 및 높은 노이즈 ($\sigma=1.0$) 조건에서 테스트한다.

### 주요 정량적 또는 정성적 결과

#### 합성 데이터셋 결과 (표 1)

| 시스템            | User 1 RMSE | User 2 RMSE |
|:------------------|:------------|:------------|
| Baseline          | 15.32       | 10.95       |
| FL, $\sigma=0$    | 12.75       | 9.67        |
| FL, $\sigma=2$    | 13.79       | 12.68       |
| FL, $\sigma=4$    | 12.59       | 19.49       |
| FL+DE, $\sigma=0$ | 12.12       | 9.41        |
| FL+DE, $\sigma=2$ | 12.05       | 9.73        |
| FL+DE, $\sigma=4$ | 13.78       | 10.95       |

* **$\sigma=0$ (노이즈 없음)**: FL+DE가 FL 단독 모델보다 더 낮은 RMSE를 달성한다 (User 1: 12.12 vs 12.75, User 2: 9.41 vs 9.67). 이는 FL+DE가 더 표현력이 풍부한 모델(공유 선형 모델 위에 각 사용자를 위한 선형 모델과 gating function 학습)이기 때문에 예상된 결과이다.
* **$\sigma=2$ (낮은 노이즈)**: FL은 RMSE가 증가하여 13.79 (User 1) 및 12.68 (User 2)로 성능이 저하된다. 반면 FL+DE는 12.05 (User 1) 및 9.73 (User 2)로, 노이즈가 없는 FL보다도 좋은 성능을 유지하며 훨씬 더 적은 성능 저하를 보인다.
* **$\sigma=4$ (높은 노이즈)**: FL은 User 2에서 19.49로 RMSE가 급격히 증가하며, Baseline(10.95)보다 훨씬 나쁜 성능을 보인다. FL+DE는 User 1에서 13.78로 Baseline보다 약간 나빠지지만, User 2에서는 10.95로 Baseline 수준을 유지하며 FL 단독 모델보다 훨씬 안정적인 성능을 보여준다.

**정성적 결과 (그림 1)**: 합성 데이터셋에 대해 학습된 gating function $\alpha_i(x)$를 시각화한 결과, 각 사용자가 자신의 도메인에 따라 공용 모델과 사설 전문가 모델을 선호하는 영역이 다르게 학습됨을 보여준다. 어두운 영역에서는 사설 전문가 모델이 선호되고, 밝은 영역에서는 일반 모델이 선호된다.

#### 스팸 데이터셋 결과 (그림 2)

* **평균 기준선 정확도**: 81.2% (±10.5%).
* **노이즈 없음 ($\sigma=0$)**: FL은 90.8%의 평균 정확도를 달성하고, FL+DE는 92.1%로 FL보다 더 높은 정확도를 보인다.
* **낮은 노이즈 ($\sigma=0.5$)**: FL의 분류 정확도는 79.3%로 크게 하락하며, 이는 기준선보다 낮은 수준이다. 그러나 FL+DE는 92.2%의 평균 정확도를 달성하여 노이즈가 없는 경우보다도 약간 개선된 성능을 보이며 강력한 안정성을 입증한다.
* **높은 노이즈 ($\sigma=1.0$)**: FL 시스템의 정확도는 76.9%로 더욱 하락한다. FL+DE의 정확도는 91.3%로 약간 하락하지만, 여전히 매우 높은 수준을 유지하며 기준선보다 훨씬 우수하다.

### 실험의 실제 결과

실험 결과는 다음 두 가지 주요 요인으로 요약된다.

1. **도메인 적응 효과**: 사설 모델이 각 도메인에서 정확도를 증가시킨다. 실제 분류 태스크에서 도메인 적응만으로 1.3%의 절대 정확도 향상이 관찰되었다.
2. **노이즈 없는 업데이트**: 사설 모델은 사설 데이터로부터 숨길 필요가 없기 때문에 노이즈 없는 업데이트를 허용한다. 차분 프라이버시 FL 시스템의 정확도가 낮은 노이즈 설정에서 11.5%, 높은 노이즈 설정에서 13.9% 감소하는 반면, FL+DE는 낮은 노이즈 설정에서 전혀 성능 저하가 없었고, 높은 노이즈 설정에서도 0.8%만 감소하여 노이즈에 대한 뛰어난 내성을 보여준다.

전반적으로 FL+DE는 차분 프라이버시로 인한 노이즈 수준이 증가함에 따라 FL 단독 모델에 비해 성능 저하가 훨씬 적거나 심지어 향상된 결과를 보여주며, 모든 시나리오에서 기준선을 능가하는 가장 좋은 성능을 제공한다.

## 🧠 Insights & Discussion

이 논문은 Federated Learning (FL) 환경에서 개인 정보 보호와 데이터 이질성이라는 두 가지 중요한 과제를 동시에 해결하기 위한 효과적인 접근 방식을 제시한다.

### 논문에서 뒷받침되는 강점

* **향상된 정확도**: FL+DE는 차분 프라이버시가 적용되지 않은 FL 설정에서도 높은 정확도를 달성하며, 특히 차분 프라이버시 노이즈가 추가될 때 기존 FL 모델의 심각한 성능 저하를 방지하고 오히려 향상된 성능을 보여준다. 이는 사설 도메인 적응 모델과 MoE 아키텍처의 결합이 각 사용자의 고유한 데이터 분포를 효과적으로 처리하고, 노이즈 없는 로컬 업데이트의 이점을 극대화하기 때문이다.
* **차분 프라이버시 노이즈에 대한 견고성**: 사설 모델의 노이즈 없는 업데이트는 차분 프라이버시 제약으로 인해 공용 모델에 주입되는 노이즈의 부정적인 영향을 효과적으로 상쇄한다. 이로 인해 높은 노이즈 설정에서도 FL+DE의 성능 저하가 미미하며, 경우에 따라서는 노이즈 없는 FL보다도 우수한 결과를 보인다. 이는 FL 시스템이 실제 개인 정보 보호 요구사항을 충족하면서도 유틸리티를 유지할 수 있음을 시사한다.
* **유연한 도메인 적응**: Mixture of Experts (MoE) 기반의 gating function은 각 사용자가 공용 모델과 사설 모델 중 어떤 모델을 특정 입력에 대해 더 신뢰할지 동적으로 학습하게 한다. 이는 각 모델의 강점을 활용하고 이질적인 사용자 도메인에 대한 적응성을 높인다.
* **일반화 및 프라이버시 향상 가능성**: 이례적인 도메인을 가진 사용자들이 일반 모델에 미치는 영향이 줄어들 수 있어 일반 모델의 일반화 능력을 향상시키고, 사용자 데이터의 추가적인 프라이버시를 제공할 가능성이 있다.

### 한계, 가정 또는 미해결 질문

* **모델 아키텍처의 복잡성**: 각 사용자마다 공용 모델 외에 사설 모델과 gating function을 유지해야 하므로, FL 단독 모델에 비해 컴퓨팅 자원 및 저장 공간 요구사항이 증가할 수 있다. 논문에서는 편의상 공용 모델과 사설 모델의 아키텍처를 동일하게 유지했지만, 아키텍처가 다른 경우의 성능 및 구현 복잡성에 대한 논의는 부족하다.
* **Gating Function의 학습 효율성**: gating function $\alpha_i(x)$가 입력 $x$에 의존하여 학습되는데, 이 학습 과정의 효율성이나 수렴 특성에 대한 상세한 분석은 제시되지 않았다. 특히, 각 사용자의 데이터가 매우 제한적인 경우 복잡한 gating function 학습이 충분히 이루어질 수 있을지에 대한 의문이 있을 수 있다.
* **확장성**: 사용자 수가 매우 많아지는 시나리오(예: 수백만 개의 모바일 장치)에서 이 시스템의 확장성에 대한 추가적인 논의가 필요할 수 있다. 각 사용자별 사설 모델을 유지하는 것이 대규모 환경에서 합리적인지에 대한 고려가 필요하다.
* **차분 프라이버시 보장 범위**: 사설 모델은 노이즈 없이 업데이트되므로, 사설 모델 자체에 대한 차분 프라이버시 보장은 없다. 프라이버시 보장은 공용 모델에만 적용되는 것으로 보이며, 이로 인해 전체 시스템의 프라이버시 수준에 대한 심층적인 분석이 필요할 수 있다.

### 논문에 근거한 간략한 비판적 해석 및 논의사항

FL+DE는 FL 환경에서 프라이버시와 유틸리티 간의 트레이드오프 문제를 해결하는 데 중요한 진전을 보여주었다. 특히 차분 프라이버시가 적용될 때 FL 모델의 성능이 크게 저하되는 문제를 효과적으로 완화한다는 점은 이 연구의 가장 강력한 기여이다. 그러나 각 사용자에게 사설 모델과 gating function을 할당하는 방식은 중앙 서버가 아닌 클라이언트 측에서 더 많은 연산 및 저장 책임을 요구하게 된다. 이는 리소스 제약이 있는 모바일 기기 환경에서는 한계가 될 수 있으나, 기업 환경과 같이 어느 정도의 로컬 리소스가 보장되는 시나리오에서는 충분히 실용적일 수 있다.

향후 연구에서는 PATE와 같은 다른 협력 모델 구축 메커니즘이나 가설 전이 학습(hypothesis transfer learning)과 같은 대안적인 도메인 적응 기술과의 결합 가능성을 탐색할 수 있다고 언급한다. 이는 FL+DE 프레임워크가 다양한 시나리오와 모델 유형에 적용될 수 있는 일반성을 가지고 있음을 시사한다. 이 논문은 FL의 실제 적용에서 도메인 이질성과 프라이버시 보호를 동시에 고려하는 데 중요한 방향을 제시하며, 특히 차분 프라이버시의 실용성을 높이는 데 기여한다.

## 📌 TL;DR

이 논문은 차분 프라이버시(Differential Privacy, DP)가 적용된 Federated Learning (FL)에서 데이터 이질성과 노이즈로 인한 정확도 저하 문제를 해결하기 위해 'Federated Learning with Domain Experts (FL+DE)'라는 새로운 프레임워크를 제안한다. FL+DE는 모든 사용자로부터 협력적으로 학습된 공용 모델($M_G$)과 각 사용자의 특정 도메인에 특화된 사설 모델($M_{P_i}$)을 Mixture of Experts (MoE) 아키텍처를 통해 결합한다. 공용 모델은 DP-SGD를 통해 개인 정보 보호를 보장하며 학습되고, 사설 모델은 노이즈 없이 로컬 데이터로만 학습되어 도메인 적응 효과를 제공한다. 각 사용자는 gating function을 학습하여 두 모델의 예측을 동적으로 가중 평균한다.

실험 결과, FL+DE는 합성 회귀 문제와 실제 스팸 분류 문제 모두에서 FL 단독 모델 및 기준선보다 높은 정확도를 달성한다. 특히, DP 노이즈 수준이 증가할수록 FL 단독 모델의 성능은 크게 저하되지만, FL+DE는 노이즈에 대한 뛰어난 내성을 보여주며 안정적이거나 오히려 향상된 성능을 유지한다. 이는 사설 모델의 노이즈 없는 업데이트와 MoE 기반 도메인 적응 덕분이다.

이 연구는 DP가 적용된 FL 시스템의 유용성을 크게 향상시키며, 개인 정보 보호를 유지하면서 이질적인 데이터 도메인을 가진 사용자들에게 향상된 모델 정확도를 제공하는 중요한 방법을 제시한다. 이는 향후 FL이 다양한 실제 시나리오에서 광범위하게 적용될 가능성을 높인다.
