# Progressive Ensemble Networks for Zero-Shot Recognition

Meng Ye, Yuhong Guo (2019)

## 🧩 Problem to Solve

이 논문은 지도(supervised) 이미지 인식 알고리즘의 한계, 즉 레이블링된 데이터 의존성 및 이미지 카테고리의 급격한 증가로 인해 발생하는 Zero-Shot Learning (ZSL)의 도전 과제를 해결하고자 한다. ZSL은 레이블링된 클래스(seen classes)의 지식을 레이블이 없는 클래스(unseen classes)로 전이하여 사람의 레이블링 노력을 줄이는 것을 목표로 한다.

기존 ZSL 방식은 시각적 특징 공간에서 의미론적 레이블 임베딩 공간으로의 매핑 함수를 학습하는 데 중점을 두었다. 그러나 이러한 방법들은 레이블링된 seen 클래스 데이터에서만 시각-의미 매핑을 식별하므로, seen 클래스와 unseen 클래스 간의 시각적 속성 외형 차이로 인해 발생하는 근본적인 '도메인 시프트(domain shift)' 문제에 직면한다. 이는 교차 클래스 일반화(cross-class generalization) 성능에 부정적인 영향을 미친다. 또한, 기존 ZSL 연구는 지식 전이를 위해 단일 레이블 임베딩 세트에 의존하는 경향이 있는데, 이는 모든 unseen 클래스에 적합하기 어렵다.

이 논문의 목표는 도메인 시프트 문제를 극복하고 ZSL의 일반화 능력을 향상시키기 위한 새로운 ZSL 프레임워크를 제안하는 것이다.

## ✨ Key Contributions

이 논문의 핵심 아이디어는 도메인 시프트 문제를 해결하고 Zero-Shot Learning (ZSL)의 일반화 능력을 향상시키기 위해 다음 두 가지 주요 접근 방식을 결합한 '점진적 앙상블 네트워크(Progressive Ensemble Networks, PrEN)'를 제안하는 것이다.

1. **다중 투영 레이블 임베딩(Multiple Projected Label Embeddings)을 통한 앙상블 네트워크 구성**:
    * 기존 연구가 단일 레이블 임베딩에 의존하여 모든 unseen 클래스에 대한 지식 전이를 수행하는 한계를 극복한다.
    * 공유 시각 특징 추출 네트워크(shared feature extraction network)는 유지하되, 여러 개의 다른 레이블 임베딩 표현을 사용하는 다중 이미지 분류 함수로 구성된 딥 앙상블 네트워크를 구축한다.
    * 각 레이블 임베딩 표현은 seen 클래스와 특정 unseen 클래스 부분 집합 간의 레이블 연관성을 최대화하도록 적응적으로 학습되는 투영(projection)을 통해 생성된다. 이는 각 분류기가 특정 unseen 클래스 그룹에 대한 정보 전이를 촉진하고, 동시에 여러 분류기의 다양성을 강화한다.
    * 앙상블 방식으로 여러 분류기의 예측을 통합함으로써, 원본 레이블 임베딩의 예측 노이즈와 클래스 편향을 극복하고 더 견고한 zero-shot 예측을 얻고자 한다.

2. **점진적 학습 프레임워크(Progressive Training Framework)를 통한 도메인 시프트 완화**:
    * unseen 클래스의 레이블 없는 데이터를 활용하여 도메인 시프트 문제를 해결한다.
    * 반복적인 점진적 학습 절차를 통해, 현재 앙상블 네트워크가 가장 높은 확신도를 가진 unseen 클래스 이미지들에 가짜 레이블(pseudo-labels)을 할당하고, 이 가짜 레이블링된 인스턴스들을 기존 학습 데이터에 추가하여 앙상블 네트워크의 파라미터를 점진적으로 개선한다. 특히 특징 추출 컴포넌트를 정제한다.
    * 이 과정에서 새로운 인스턴스가 선택되고 이전의 잘못된 예측이 수정될 수 있는 동적인 인스턴스 선택 절차를 사용한다. 이는 seen 클래스에 대한 과적합(overfitting)을 방지하고 unseen 클래스에 대한 앙상블 네트워크의 일반화 능력을 향상시킨다.

종합적으로, PrEN은 여러 개의 특화된 지식 전이 경로를 학습하는 앙상블 접근 방식과 unlabeled unseen 클래스 데이터를 활용하여 모델을 점진적으로 개선하는 전략을 결합하여, ZSL의 주요 난제인 도메인 시프트와 일반화 문제를 효과적으로 해결하고자 한다.

## 📎 Related Works

논문은 Zero-Shot Learning (ZSL) 연구를 크게 세 가지 범주로 분류하고, 제안하는 PrEN 모델이 기존 연구와 어떻게 차별화되는지 설명한다.

### 2.1. Zero-Shot Learning (표준 ZSL)

기존 ZSL 방법론은 대부분 '레이블 임베딩(label embeddings)'을 공통 의미 공간에 배치하여 seen 클래스와 unseen 클래스 간의 간극을 메우는 데 중점을 둔다. 이는 입력 시각 특징 공간과 의미론적 레이블 공간 사이에 전이 가능한 매핑 함수를 학습하는 방식으로 이루어진다.

* **선형/비선형 투영 및 호환 함수**: ALE [1], DeViSE [10]는 시각적 특징을 의미 공간으로 매핑하는 선형 투영을 사용한다. LatEm [33]은 비선형 호환 함수를, 다른 연구들 [2, 24]은 이선형(bilinear) 호환 함수를 학습한다.
* **신경망 기반 임베딩**: [35, 3]에서는 의미 정보를 임베딩하기 위해 신경망을 사용하며, Semantic Auto-Encoders (SAE) [29, 19]는 재구성 손실(reconstruction loss)을 사용하여 더 나은 의미 공간 투영을 학습한다.
* **합성 기반 임베딩**: SynC [8]와 ConSE [22]는 unseen 인스턴스를 seen 클래스 임베딩의 선형 조합으로 임베딩한다.

**한계점**: 이러한 방법들은 주로 seen 클래스에서만 학습되므로 unseen 클래스의 시각적 외형 변화에 대한 정보를 얻지 못하여 '도메인 시프트' 문제에 취약하다.

* **생성형 ZSL (Generative ZSL)**: 최근 연구들은 생성형 방식으로 ZSL을 해결하려 시도한다. [9]는 unseen 의미 속성 벡터를 시각 특징 공간으로 매핑하는 선형 투영을 사용하고, [7]은 생성형 모멘트 매칭 네트워크(generative moment matching network)를 통해 unseen 클래스 인스턴스를 생성한다. [37]은 GAN을 사용하여 노이즈가 있는 텍스트에서 시각적 특징을 합성한다.
**한계점**: 이러한 생성된 특징들이 실제 unseen 시각 특징과 잘 정렬된다는 보장이 없으며, 여전히 도메인 시프트 문제에 직면할 수 있다.

### 2.2. Transductive Zero-Shot Learning

Transductive ZSL은 표준 ZSL과 달리 학습 단계에서 unseen 클래스의 unlabeled 인스턴스에 접근할 수 있는 설정이다. 이는 레이블이 없으므로 'zero-shot' 원칙을 위반하지 않으며, 도메인 시프트 문제를 극복하기 위해 unseen 클래스 인스턴스를 활용한다.

* **다단계 절차**: [12]는 CCA를 사용하여 시각 특징과 클래스 프로토타입을 다중 뷰 임베딩 공간에 투영하고, 테스트 인스턴스를 사용하여 하이퍼그래프를 구축하여 레이블 전파를 수행한다.
* **도메인 적응**: [18]은 비지도 도메인 적응(unsupervised domain adaptation) 관점에서 ZSL을 희소 코딩(sparse coding)을 통해 해결한다.
* **공유 모델 공간 학습**: [14]는 seen 및 unseen 데이터에 대한 공유 모델 공간을 학습하여 클래스 간 지식 전이를 촉진한다.
* **오토인코더 기반 임베딩**: [29]는 오토인코더를 사용하여 시각 및 의미 벡터의 공동 임베딩을 학습하며, 예측 손실을 최소화하기 위해 unseen 클래스 인스턴스를 활용한다.
* **가짜 레이블 할당**: [13]은 테스트 인스턴스에 가짜 레이블을 할당하고 seen 및 unseen 클래스 데이터 모두에 대해 임베딩 행렬을 학습하지만, 단일 투영 행렬을 사용한다.
* **생성 모델 정제**: [30]은 속성 벡터에서 seen 및 unseen 클래스의 데이터 분포를 예측하는 생성 모델을 학습하고, unlabeled 테스트 데이터를 사용하여 타겟 클래스의 분포 파라미터를 정제한다.
* **종단 간(End-to-End) 학습**: [28]은 seen 클래스 데이터와 unseen 테스트 데이터 모두에 대한 손실을 최적화하는 종단 간 네트워크를 훈련하여, Quasi-Fully Supervised Learning 손실을 최소화하여 학습 중 seen/unseen 편향을 줄인다.

**PrEN의 차별점**:
제안된 PrEN은 Transductive ZSL에 속하지만, 다음과 같은 주요 측면에서 기존 Transductive ZSL 연구와 다르다.

1. **다중 레이블 임베딩**: 기존 연구가 단일 레이블 임베딩 세트를 사용하는 것과 달리, PrEN은 다중 레이블 임베딩 세트를 사용한다. 각 세트는 seen 클래스와 unseen 클래스의 부분 집합 간의 레이블 관계를 강화하여 생성된다. 이는 모든 unseen 클래스로의 견고한 지식 전이를 촉진한다.
2. **점진적 학습 프레임워크**: PrEN은 unseen 클래스에서 선택된 unlabeled 인스턴스 부분 집합과 그들의 예측된 가짜 레이블을 동적으로 통합하여 앙상블 네트워크를 점진적으로 개선하고 도메인 시프트를 방지하는 프레임워크를 사용한다. 각 반복에서 동적인 인스턴스 선택 절차를 통해 새로운 인스턴스가 선택되고 이전의 잘못된 예측이 수정될 수 있는 유연성을 제공한다.

### 2.3. Progressive Training with Pseudo-Labels

정적(static) 점진적 학습 절차에서 예측된 가짜 레이블을 할당하여 unlabeled 데이터를 활용하는 것은 표준 분류 설정에서 이미 사용되어 왔다.

* **Co-training**: [5]는 두 개의 다른 분류기를 사용하여 unlabeled 데이터에 가짜 레이블을 생성한다.
* **Tri-training**: [36]은 Co-training과 유사하게 세 개의 다른 분류기의 출력을 활용한다. [26]은 비지도 도메인 적응 문제 해결에 Tri-training을 적용했다.
* **커리큘럼 학습**: [4]는 "쉬운 샘플에서 어려운 샘플로"의 사전 정의된 방식으로 모델을 훈련하기 위해 점진적 커리큘럼 학습(progressive curriculum learning)을 사용한다.
* **Self-paced Co-training**: [21]은 점진적인 "쉬운-어려운" 전략과 데이터의 두 가지 뷰를 사용하여 훈련한다.
* **점진적 샘플링**: [32]는 비디오 검색 작업을 위한 점진적 샘플링 방식을 제안한다.

**PrEN의 차별점**:
제안된 PrEN은 이러한 기존 연구와 달리, 더 도전적인 zero-shot 학습 문제에 대해 다른 레이블 임베딩을 가진 다중 분류 함수를 포함하는 새로운 앙상블 네트워크를 점진적 절차와 결합하여 사용한다.

## 🛠️ Methodology

이 논문은 레이블링된 seen 클래스 $D_s = \{(x_i, y_i)\}_{i=1}^{N_s}$와 레이블이 없는 unseen 클래스 $D_u = \{(x_j, y_j)\}_{j=1}^{N_u}$를 사용하는 Zero-Shot Learning (ZSL) 설정을 고려한다. 모든 클래스 $L=L_s+L_u$에 대한 의미론적 레이블 표현 행렬 $M \in \mathbb{R}^{m \times L}$ (예: 속성 벡터)을 가정한다. 제안하는 프레임워크는 다중 이미지 분류 함수를 포함하는 '점진적 앙상블 네트워크(Progressive Ensemble Network)'이며, 이는 공유 특징 추출 네트워크와 서로 다른 레이블 임베딩 표현을 사용한다. 점진적 학습 프레임워크는 unlabeled 인스턴스와 그들의 예측된 가짜 레이블(pseudo-labels)을 동적이고 앙상블 방식으로 통합하여 전체 앙상블 네트워크를 반복적으로 정제한다.

### 3.1. Ensemble Networks

표준 ZSL 방식에 따라, 이미지 $x$로부터 시각 특징 $f_v(x)$를 추출하는 CNN $f_v$와, 이 시각 특징을 의미 공간 $R^m$ (예: 속성 공간)으로 임베딩하는 네트워크 $f$를 사용한다. 이는 $S \cup U$의 모든 클래스에 대한 이미지 분류 함수 $f \circ f_v(x)$를 형성한다. 그러나 단일 레이블 임베딩 행렬 $M$로는 seen 클래스와 모든 unseen 클래스 간의 최적의 의미 연관성을 동시에 달성하기 어렵다.

따라서 논문은 레이블 임베딩 $M$을 $K$개의 다른 임베딩 공간으로 투영하여 $\{P_k : R^m \to R^h | k=1, \dots, K\}$ $K$개의 다른 레이블 임베딩 세트 $\{P_k(M)\}$를 생성할 것을 제안한다. 각 투영된 레이블 임베딩 행렬 $P_k(M)$에 대해, 시각 특징 $f_v(x)$를 해당 레이블 임베딩 공간으로 매핑하는 임베딩 네트워크 $f_k$를 생성하여 zero-shot 분류 함수 $f_k \circ f_v(x)$를 만든다. 이 $K$개의 분류 함수를 앙상블 방식으로 활용하여 원본 레이블 임베딩 $M$의 노이즈와 클래스 편향 영향을 줄여 견고한 zero-shot 이미지 인식을 목표로 한다.

#### 3.1.1 Label Embedding Projection

각 $k$-번째 레이블 임베딩 투영 $P_k$는, 투영된 레이블 임베딩 공간에서 seen 클래스 $S$와 무작위로 선택된 unseen 클래스의 부분 집합 $Z_k \subset U$ 간의 가중 유사도 점수를 최대화하는 방식으로 적응적으로 수행된다. 선형 투영 함수 $P_k(M) = P^{(k)}M$을 가정하며, 투영 행렬 $P^{(k)} \in R^{h \times m}$는 직교 행(orthogonal rows)을 갖는다 ($P^{(k)}P^{(k)T} = I$).

레이블 임베딩 투영은 다음 최대화 문제로 공식화된다:
$$
\max_{P^{(k)}} \sum_{i \in S, j \in Z_k} \text{tr}(P^{(k)} M_{:i} A_{ij} M_{:j}^T P^{(k)T}) \quad \text{subject to } P^{(k)}P^{(k)T} = I
$$
여기서 $M_{:i}$는 행렬 $M$의 $i$-번째 열을 나타내고, $\text{tr}(\cdot)$은 트레이스 함수를 나타낸다. 연관성 가중치(association weight) $A_{ij}$는 원본 레이블 표현 공간에서 해당 $i$-번째 및 $j$-번째 클래스 간의 코사인 유사도로 정의된다.

이 최대화 문제는 다음과 같은 닫힌 형식 해(closed-form solution)를 갖는다:
$$
P^{(k)} = [u_1, u_2, \dots, u_h]^T
$$
여기서 $\{u_i\}_{i=1}^h$는 행렬 $\sum_{i \in S, j \in Z_k} \frac{1}{2}(M_{:i} A_{ij} M_{:j}^T + M_{:j} A_{ij} M_{:i}^T)$의 상위 $h$개 고유 벡터(eigenvectors)이다.

$K$개의 서로 다른 unseen 클래스 부분 집합 $\{Z_k\}_{k=1}^K$를 무작위로 선택함으로써 $K$개의 서로 다른 레이블 임베딩 투영 행렬 $\{P^{(k)}\}_{k=1}^K$을 생성할 수 있다. 각 결과 레이블 임베딩 행렬 $P^{(k)}M$은 seen 클래스와 unseen 클래스 간의 서로 다른 지식 전이 구조를 인코딩한다.

#### 3.1.2 Loss Function of the Ensemble Network

레이블링된 훈련 인스턴스 $D_{train} = \{(x_i, y_i)\}_{i=1}^N$가 주어졌을 때, $K$개의 분류 함수 $\{f_k \circ f_v(x)\}_{k=1}^K$를 갖는 딥 앙상블 신경망은 다음 음의 로그-우도 손실 함수(negative log-likelihood loss function)를 최소화함으로써 훈련된다:
$$
L(\omega_v, \omega_1, \dots, \omega_K) = \frac{1}{N} \sum_{i=1}^N \sum_{k=1}^K \ell_k(v_i^k, y_i)
$$
여기서 $(\omega_v, \omega_1, \dots, \omega_K)$는 모델 파라미터를 나타내고, $v_i^k = f_k \circ f_v(x_i)$는 $k$-번째 분류기가 인스턴스 $x_i$에 대해 해당 레이블 임베딩 공간에서 예측한 벡터이다. $\ell_k(\cdot, \cdot)$는 $k$-번째 분류기의 소프트맥스(softmax) 예측 점수에 대해 계산된 음의 로그-우도 손실 함수이다:
$$
\ell_k(v_i^k, c) = -\log p_k(c|v_i^k) = -\log \frac{\exp(v_i^{k T} P^{(k)} M_{:c})}{\sum_{c' \in S \cup U} \exp(v_i^{k T} P^{(k)} M_{:c'})}
$$
위 소프트맥스 함수는 seen 클래스와 unseen 클래스 *모두*에 대해 정의된다는 점에 유의해야 한다. 이는 seen 클래스뿐만 아니라 unseen 클래스의 훈련 인스턴스(가짜 레이블링된 인스턴스)를 포함하도록 설계되었다. 초기에는 레이블링된 훈련 데이터 $D_{train}$가 seen 클래스 $D_s$만 포함하지만, 점진적 훈련을 통해 unseen 클래스에서 온 가짜 레이블링된 데이터 세트를 포함하도록 확장된다.

#### 3.1.3 Ensemble Zero-Shot Prediction

앙상블 네트워크에서 학습된 다중 분류 함수를 사용하여, 각 unlabeled 인스턴스 $x_i$에 대해 zero-shot 예측을 수행한다.

1. **개별 분류기 예측**: 먼저, $K$개 분류기 각각을 사용하여 유사도 점수 기반으로 예측을 수행한다:
    $$
    \hat{y}_i^{(k)} = \arg \max_{c \in Z_k} \langle f_k \circ f_v(x_i), P^{(k)}M_{:c} \rangle
    $$
    여기서 $\langle \cdot, \cdot \rangle$는 두 벡터의 내적을 나타낸다. $k$-번째 레이블 임베딩 세트는 seen 클래스와 unseen 클래스 부분 집합 $Z_k$의 레이블 연관성을 최대화하여 생성되었으므로, $k$-번째 분류기는 $Z_k$에 대한 zero-shot 예측에만 사용된다.

2. **앙상블 예측**: 다음으로, 정규화된 다수결 투표(normalized majority voting) 전략을 사용하여 $K$개의 모든 예측을 통합하여 최종 예측 클래스를 결정한다:
    $$
    \hat{y}_i = \arg \max_c \varphi(x_i, c)
    $$
    여기서 투표 점수 $\varphi(x_i, c)$는 다음과 같이 정의된다:
    $$
    \varphi(x_i, c) = \frac{\sum_{k=1}^K I[c = \hat{y}_i^{(k)}]}{\sum_{k=1}^K I[c \in Z_k]}
    $$
    $I[\cdot]$은 주어진 조건이 참일 때 1을 반환하는 지표 함수(indicator function)이다.

**Generalized ZSL (GZSL)**: 테스트 인스턴스 $x_i$가 seen 또는 unseen 클래스 중 하나에서 올 수 있는 GZSL의 경우, unseen 클래스 $c$에 대한 투표 점수는 Eq.(7)의 정규화된 투표 점수를 그대로 사용한다. 그러나 seen 클래스 $c$에 대한 투표 점수는 모든 $K$개 분류기가 해당 클래스에 대해 예측한 평균 점수로 계산한다:
$$
\varphi(x_i, c) = \frac{1}{K} \sum_{k=1}^K \langle f_k \circ f_v(x_i), P^{(k)}M_{:c} \rangle
$$

### 3.2. Progressive Ensemble Networks

seen 클래스에서 레이블링된 인스턴스만으로 훈련하면 도메인 시프트 문제에 취약하다. 제안된 앙상블 네트워크는 unseen 클래스 인스턴스에 대한 투표 기반 예측을 수행하고, pseudo-labeled 인스턴스를 훈련 과정에 통합하기 위한 자연스러운 기반을 제공한다. 이에 따라, seen 클래스 $D_s$의 레이블링된 데이터로 초기 훈련된 앙상블 네트워크를 반복적이고 동적으로 개선하기 위한 '점진적 학습 절차(progressive training procedure)'를 제안한다.

#### Algorithm 1: Progressive Training of Ensemble Networks

**입력**:

* seen 클래스의 레이블링된 데이터 $D_s$
* unseen 클래스의 unlabeled 데이터 $D_u$
* 레이블 임베딩 행렬 $M$

**초기화**:

1. $D_{train} \leftarrow D_s$
2. $D_{pseudo} \leftarrow \emptyset$
3. $\{P^{(k)}\}_{k=1}^K$에 대한 레이블 임베딩 투영을 수행한다.
4. $D_{train}$에 대해 종단 간(end-to-end) 딥 앙상블 네트워크를 훈련한다.

**반복 (MaxIter까지)**:

1. Eq.(6) 및 Eq.(7)을 사용하여 $D_u$의 가짜 레이블을 예측한다.
2. 각 unseen 클래스 $c \in U$에서 가장 큰 예측 점수 $\varphi(x_i, c)$를 가진 상위 $N_{pseudo}$개 인스턴스를 선택하여 가짜 레이블링된 세트 $D_{pseudo}$를 생성한다.
3. 훈련 세트를 업데이트한다: $D_{train} \leftarrow D_s \cup D_{pseudo}$
4. $D_{train}$에 대해 앙상블 네트워크 파라미터를 정제(refine)한다.

이 점진적 학습 절차는 각 반복에서 현재 앙상블 네트워크를 사용하여 unseen 클래스의 각 unlabeled 인스턴스 $x_i$에 대해 Eq.(6)을 통해 가짜 레이블 $\hat{y}_i$를 예측한다. 그런 다음, 각 unseen 클래스 $c \in U$에 대해 예측 점수 $\varphi(x_i, c)$가 가장 높은 상위 $N_{pseudo}$개 인스턴스를 선택한다. 모든 unseen 클래스에서 선택된 이 인스턴스들과 그들의 예측된 레이블은 가짜 레이블 세트 $D_{pseudo} = \{(x_i, \hat{y}_i)\}_{i=1}^{N_p}$를 형성한다.

앙상블 네트워크 파라미터는 확장된 훈련 세트 $D_{train} = D_s \cup D_{pseudo}$에 대해 Eq.(3)의 손실 함수를 최소화하여 정제된다. 확장된 훈련 세트가 seen 클래스와 unseen 클래스 모두의 데이터를 포함하므로, 정제된 앙상블 네트워크는 의미 특징의 시각적 외형 측면에서 도메인 시프트 문제를 극복하고 zero-shot 예측 성능을 향상시킬 것으로 기대된다.

더욱이, 가짜 레이블 세트를 점진적으로 증가시키는 대신, 이 세트를 매 반복마다 점진적으로 개선된 앙상블 네트워크로 동적으로 업데이트하여 이전 가짜 레이블 세트의 잠재적 레이블 오류를 수정한다.

**구현 세부 사항**:

* **특징 추출**: ResNet-34 [15]의 마지막 평균 풀링 레이어에서 추출된 512차원 벡터를 시각 특징으로 사용한다. ResNet은 ImageNet으로 사전 학습된 모델로 초기화된다.
* **임베딩 함수**: 두 개의 은닉 레이어(각 512 유닛)와 ReLU 활성화 함수를 가진 다층 퍼셉트론(MLP)을 각 임베딩 함수로 사용한다.
* **옵티마이저**: Adam [17]을 사용하며, 학습률 $\eta=0.001$이다.
* **반복 횟수**: MaxIter=20으로 설정하며, 각 반복에서 배치 크기 64로 100개 배치를 사용하여 모델을 훈련한다.
* **가짜 인스턴스 수**: $N_{pseudo} = \min(\rho N_{avg}, N_{max})$로 설정하며, $N_{avg}$는 각 훈련 클래스의 평균 이미지 수이고, $\rho=0.25$, $N_{max}=20$이다.
* **분류기 수**: $K=50$개의 다른 레이블 임베딩(즉, 분류기 수)을 사용한다.
* **unseen 클래스 부분 집합 크기**: $\{Z_k\}_{k=1}^K$는 무작위로 선택되며, 각 $Z_k$의 크기는 unseen 클래스 수의 절반으로 설정한다.
* **투영 차원**: 원본 레이블 임베딩을 더 낮은 차원 $h$로 투영하며, 실험에서는 $h=70$을 사용한다.

### 4.1. Experiment Settings

#### 4.1.1 Datasets

세 가지 널리 사용되는 ZSL 데이터셋과 레이블 속성 벡터를 사용하여 실험을 수행했다.

* **Caltech-UCSD-Birds 200-2011 (CUB) [31]**: 조류 종에 대한 세밀한(fine-grained) 데이터셋으로, 200종의 조류 이미지 11,788개와 312개의 속성 주석을 포함한다.
* **SUN [23]**: 717가지 장면에서 14,340개의 이미지를 포함하며, 각 이미지에 102개의 속성이 주석으로 달려 있다.
* **Animal with Attributes 2 (AWA2) [34]**: AWA [20]의 업데이트 버전으로, 50개 동물 클래스에서 37,322개의 이미지를 포함하며 각 클래스에 대해 85개의 수치 속성 값을 제공한다. AWA는 원본 이미지 데이터가 공개되지 않아 AWA2를 사용한다.

레이블 임베딩 행렬 $M$은 속성 벡터에서 추출되었다.

#### 4.1.2 Seen/Unseen Splits

ZSL 연구의 과학적 타당성을 위해, ImageNet [25]으로 사전 학습된 CNN 모델을 사용할 때 ZSL 테스트 클래스가 ImageNet 1K 클래스와 겹치지 않도록 해야 한다. 이러한 겹침은 'zero-shot' 규칙을 위반하고 성능에 불공정한 이점을 줄 수 있다. 따라서 [34]에서 제안한 ZSL 분할(PS, Proposed Splits)을 사용하였다. PS 분할은 표준 분할(SS, Standard Splits)과 동일한 수의 테스트 클래스를 갖지만, ImageNet 클래스와의 겹침이 없음을 보장한다. SUN 데이터셋의 경우 [16]에서 제안된 10개 테스트 클래스 분할(SUN10)과 72개 테스트 클래스 분할(SUN72)을 모두 사용한다.

#### 4.1.3 Evaluation Metric

ZSL 예측 성능을 평가하기 위해 널리 사용되는 Top-1 정확도(accuracy)를 채택했다. Top-1 정확도는 각 테스트 클래스에서 올바르게 레이블링된 인스턴스의 비율을 계산한 후, 이 클래스들 전체에 대해 평균을 낸다. 일부 기존 연구와의 비교를 위해 필요에 따라 다중 클래스 분류 정확도(multi-class classification accuracy, MACC)도 보고한다.

* **PS vs SS 분할**: 'PS' 분할에서의 결과는 대부분 'SS' 분할보다 나빴다. 특히 AWA2 데이터셋에서 이러한 경향이 두드러지는데, 이는 ImageNet 1K 클래스와 테스트 클래스의 겹침이 성능에 추가적인 이점을 제공했음을 시사한다. transductive ZSL 방법들은 주로 'SS' 설정에서 평가되었으며, 'PS' 설정에서의 결과는 더 나쁠 것으로 예상된다.
* **PrEN vs Inductive Methods**: Top-1 정확도 측면에서 PrEN은 모든 데이터셋에서 5가지 inductive 방법 모두를 능가한다.
* **PrEN vs Transductive Methods**:
  * CUB, SUN10 및 AWA2의 'PS' 분할에서 PrEN은 QFSL [28] 다음으로 두 번째로 좋은 결과를 보였다.
  * 하지만 AWA2의 'SS' 분할, SUN72의 'SS' 및 'PS' 분할을 포함한 모든 다른 경우에서 PrEN은 가장 좋은 결과를 달성했다.
  * 특히, 더 어려운 장면 분류 데이터셋인 SUN72(unseen 클래스가 더 많고 각 클래스당 훈련 데이터가 적음)에서 PrEN은 'SS' 및 'PS' 분할에서 각각 63.3%, 62.9%를 달성하여 다른 모든 방법들을 상당한 성능 향상으로 능가했다.
  * 다중 클래스 정확도(MACC) 측면에서 PrEN은 UDA 및 DCL과 같은 transductive ZSL 방법들을 크게 능가했다 (예: CUB에서 UDA의 40.6% 대비 PrEN 66.6%, SUN10에서 DCL의 84.4% 대비 PrEN 86.4%).
* **PrEN vs Baseline Variants**: 제안된 PrEN과 두 베이스라인 변형(Progressive Training baseline, PrEN$_{w/oProj}$) 사이에 상당한 성능 차이가 있었다.
  * 앙상블 아키텍처와 다양한 레이블 임베딩이 없는 'Progressive Training baseline'은 효과적인 모델을 생성하지 못했다.
  * 레이블 임베딩 투영을 제외하고 앙상블 아키텍처를 유지한 'PrEN$_{w/oProj}$'도 PrEN보다 훨씬 낮은 성능을 보였다.
    이러한 결과는 제안된 앙상블 네트워크 아키텍처와 필수적인 레이블 임베딩 투영이 점진적 학습을 통한 가짜 레이블 통합의 견고하고 중요한 기반을 형성한다는 것을 시사한다.

**Empirical Computational Complexity**:
PrEN은 Progressive Training 변형 모델에 비해 괄목할 만한 성능 향상을 달성했지만, 계산 비용은 상대적으로 미미하다.

* 두 모델 모두 동일한 ResNet34 백본 구조(3.6 billion FLOPs)를 공유한다.
* PrEN은 Progressive baseline보다 49개의 MLP 컴포넌트(각 0.8 million FLOPs)를 더 가지며, 이는 약 0.04 billion FLOPs의 추가 비용에 불과하다.
* 이 추가 비용은 백본 ResNet의 FLOPs에 비해 상대적으로 무시할 만하며, Top-1 정확도에서 평균 16.9%, 다중 클래스 정확도에서 17.9%의 성능 향상에 기여했다. 이는 제안된 앙상블 아키텍처와 적응적 레이블 임베딩 투영의 효율성과 적합성을 다시 한번 입증한다.

### 4.3. Parameter Sensitivity Analysis

모델의 두 가지 하이퍼파라미터 $K$ (투영된 레이블 임베딩 및 분류기 수)와 $h$ (투영된 레이블 임베딩의 차원)에 대한 민감도 분석을 수행했다.

* **$h$에 대한 민감도**: CUB 데이터셋의 'PS' 분할에서 $K=50$으로 고정하고 $h \in \{10, 20, \dots, 70\}$에 대해 실험했다.
  * 테스트 ZSL 정확도는 $h=10$에서 $h=30$으로 증가함에 따라 빠르게 증가했으며, 이후에는 증가폭이 매우 작아졌다. 최적 성능은 $h=50$에서 달성되었다.
  * 이는 더 큰 차원이 투영된 레이블 임베딩에서 유용한 정보를 보존하는 데 도움이 되며, 원본 차원의 10% 정도의 작은 차원에서도 매우 좋은 성능을 얻을 수 있음을 시사한다. CUB에서는 $h \in (30, 70]$ 범위가 안전한 선택이 될 수 있다.
* **$K$에 대한 민감도**: CUB 데이터셋에서 $h=70$으로 고정하고 $K \in \{10, 20, \dots, 60\}$에 대해 실험했다.
  * $K=10$일 때 ZSL 정확도는 매우 낮았다.
  * $K=10$에서 $K=30$으로 증가함에 따라 ZSL 성능이 급격히 증가했으며, $K$가 60으로 더 증가해도 변화는 매우 작았다.
  * 이 결과는 제안된 모델이 $K > 30$과 같은 합리적인 범위 내에서 설정될 경우 하이퍼파라미터 $K$에 크게 민감하지 않음을 시사한다.

U: unseen classes Top-1 accuracy, S: seen classes Top-1 accuracy, H: Harmonic mean

문헌의 대부분 ZSL 연구는 테스트 클래스가 unseen 클래스로만 구성된다는 Conventional ZSL 설정에 초점을 맞추었으나, 이는 지나치게 엄격할 수 있다. 이에 논문은 테스트 인스턴스가 seen 및 unseen 클래스 모두에서 올 수 있는 Generalized ZSL (GZSL) 설정에서 PrEN의 성능을 비교했다.

* **PrEN의 GZSL 확장**: PrEN 모델 내의 분류기들은 모든 클래스에 대한 다중 클래스 분류를 수행하므로 GZSL로 편리하게 확장될 수 있다.
* **훈련 전략**: GZSL의 주요 문제점은 많은 unseen 클래스 인스턴스가 잘못 분류되어 seen 클래스로 분류될 수 있다는 것이다. 이를 해결하기 위해 점진적 학습 과정의 처음 몇 반복에서는 unseen 클래스에 대해서만 가짜 인스턴스를 선택하고, 이후 반복에서는 균형 잡힌 성능을 위해 seen 클래스와 unseen 클래스 모두에 대해 가짜 인스턴스를 선택한다.
* **평가 설정**: [34]의 종합 연구를 따라 'PS' 분할을 사용했다. 각 seen 클래스에 대해 무작위로 20%의 인스턴스를 분리하여 테스트 세트에 추가했다. unseen 및 seen 클래스에 대한 Top-1 테스트 정확도를 각각 평가하고, 이들의 조화 평균(harmonic mean)을 GZSL 정확도 결과로 사용했다.
* **QFSL과의 비교 제외**: [28]의 QFSL도 GZSL 결과를 보고했지만, 그들은 unlabeled 인스턴스가 seen 클래스에 속하는지 unseen 클래스에 속하는지 여부를 아는 추가 지식을 가정한 비표준적이고 제한적인 설정에서 GZSL을 수행했으므로 공정성을 위해 비교에서 제외했다.

**결과 분석**:

* 일부 비교 방법은 seen 클래스에서 상당히 좋은 성능을 달성했지만, unseen 클래스에서의 zero-shot 정확도는 매우 낮았다 (예: SynC는 AWA2에서 unseen 11.5%, seen 70.9%). 모든 클래스에 대한 전체 성능(컬럼 'H')은 여전히 좋지 않았다.
* seen 클래스와 unseen 클래스 간의 성능에는 일반적으로 트레이드오프(trade-off)가 존재하며, 조화 평균은 전체 성능을 측정한다.
* 제안된 PrEN은 seen 클래스에서 최고의 성능을 달성하지는 못했지만, **unseen 클래스에서의 zero-shot 예측 성능이 다른 비교 방법보다 훨씬 우수했다.**
* 전반적인 GZSL 성능(조화 평균) 측면에서, PrEN은 모든 비교 방법들을 큰 차이로 능가했다. 이는 GZSL 설정에서도 제안된 모델의 효과를 입증한다.

### 🧠 Insights & Discussion

이 논문에서 제안하는 PrEN (Progressive Ensemble Networks) 모델은 Zero-Shot Learning (ZSL)의 주요 도전 과제인 도메인 시프트(domain shift)와 unseen 클래스에 대한 일반화 능력 부족 문제를 효과적으로 해결하기 위한 강력한 접근 방식을 제시한다.

**논문에서 뒷받침되는 강점**:

1. **다중 레이블 임베딩을 통한 유연한 지식 전이**: PrEN은 단일 고정 레이블 임베딩에 의존하는 대신, seen 클래스와 unseen 클래스의 특정 부분 집합 간의 레이블 연관성을 최대화하도록 적응적으로 투영된 $K$개의 다른 레이블 임베딩을 사용한다. 이 접근 방식은 다양한 unseen 클래스에 대한 지식 전이 구조의 다양성을 확보하여, 원본 레이블 임베딩의 예측 노이즈와 클래스 편향을 효과적으로 줄이고 견고한 zero-shot 예측을 가능하게 한다. 실험 결과에서 PrEN$_{w/oProj}$와의 비교를 통해 레이블 임베딩 투영의 중요성이 명확히 드러났다.
2. **점진적 학습을 통한 도메인 시프트 완화 및 일반화 강화**: unlabeled unseen 클래스 데이터에 가짜 레이블(pseudo-labels)을 할당하고 이를 훈련 데이터에 동적으로 통합하여 앙상블 네트워크를 점진적으로 정제하는 과정은 seen 클래스에 대한 과적합을 방지하고 unseen 클래스에 대한 일반화 능력을 크게 향상시킨다. 이는 점진적 훈련 절차를 통해서 unseen 클래스 인스턴스를 모델 훈련에 통합하여 시각적 외형의 도메인 시프트 문제를 해결하는 핵심적인 부분이다. 또한, 매 반복마다 가짜 레이블 세트를 동적으로 업데이트함으로써 이전 반복의 잠재적인 잘못된 예측을 '수정'할 수 있는 능력을 제공한다.
3. **뛰어난 성능과 효율성**: Conventional ZSL 및 Generalized ZSL (GZSL) 모두에서 PrEN은 최신 방법들을 능가하는 우수한 성능을 보여주었다. 특히 어려운 SUN72 데이터셋에서 그 효과가 두드러진다. 또한, $K$개의 분류기를 사용함에도 불구하고 추가적인 계산 복잡성(FLOPs)이 ResNet 백본에 비해 상대적으로 미미하다는 점은 이 방법의 효율성을 강조한다. 이는 실용적인 적용 가능성을 높인다.
4. **GZSL로의 자연스러운 확장성**: 제안된 모델이 모든 클래스에 대한 다중 클래스 분류를 수행할 수 있도록 설계되었기 때문에, GZSL 설정으로의 확장이 용이하다. unseen 클래스에 대한 가짜 레이블 선택과 seen/unseen 클래스에 대한 가짜 레이블 선택을 균형 있게 조절하는 전략을 통해 GZSL에서 seen/unseen 클래스 성능 간의 트레이드오프를 효과적으로 관리하며 전체 성능(조화 평균)을 크게 향상시켰다.

**한계, 가정 또는 미해결 질문**:

1. **가짜 레이블의 신뢰성**: 점진적 학습 과정에서 가짜 레이블의 품질은 모델 성능에 매우 중요하다. "가장 높은 확신도를 가진" 인스턴스를 선택하는 전략을 사용하지만, 초기 단계 또는 특히 어려운 클래스에서 잘못된 가짜 레이블이 할당될 위험은 여전히 존재할 수 있다. 논문은 '수정' 능력을 언급하지만, 잘못된 예측이 누적되는 것을 완전히 방지하는지에 대한 심층적인 분석은 제한적이다.
2. **하이퍼파라미터 민감도**: $K$와 $h$에 대한 민감도 분석을 수행했으나, $N_{pseudo}$ (각 클래스에서 선택되는 가짜 인스턴스 수)와 같은 다른 중요한 하이퍼파라미터에 대한 분석은 명시적으로 제시되지 않았다. 이러한 값들의 선택이 최종 성능에 미치는 영향에 대한 이해가 더 필요할 수 있다.
3. **$Z_k$의 무작위 선택**: unseen 클래스의 부분 집합 $Z_k$를 무작위로 선택하는 방식은 다양한 레이블 임베딩을 유도하는 데 효과적이지만, 최적의 $Z_k$ 선택 전략에 대한 추가 연구가 있을 수 있다. 어떤 $Z_k$가 더 효과적인 지식 전이 구조를 생성하는지에 대한 분석은 현재로서는 명시적이지 않다.
4. **ResNet-34 사전 학습 모델의 영향**: 모델이 ImageNet으로 사전 학습된 ResNet-34를 특징 추출기로 사용한다. 이는 ZSL의 'zero-shot' 원칙을 위반하지 않기 위해 'PS' 분할을 사용하는 등 노력을 기울였지만, 사전 학습된 모델 자체의 바이어스(bias)가 unseen 클래스 인식에 미치는 영향에 대한 더 깊은 논의는 제한적이다.

**논문에 근거한 간략한 비판적 해석 및 논의사항**:

PrEN은 ZSL 분야에서 도메인 시프트 문제를 다루는 데 있어 실질적인 진전을 이루었다. 다중 투영 임베딩을 통한 앙상블 학습과 동적 가짜 레이블링 기반의 점진적 훈련 프레임워크는 기존 방법들의 한계를 효과적으로 극복하는 혁신적인 조합이다. 특히, 제한된 추가 계산 비용으로 상당한 성능 향상을 이룬 점은 이 방법의 강점이다.

다만, 가짜 레이블의 품질 관리와 관련된 더 정교한 메커니즘이나 동적인 하이퍼파라미터 조정 전략에 대한 탐구는 향후 연구 방향이 될 수 있다. 그럼에도 불구하고, PrEN은 unlabeled 데이터와 앙상블 학습을 효과적으로 결합하여 ZSL의 일반화 및 도메인 적응 문제를 해결하는 데 있어 중요한 이정표를 제시한 것으로 평가할 수 있다.

## 📌 TL;DR

이 논문은 Zero-Shot Learning (ZSL)에서 seen 클래스와 unseen 클래스 간의 **도메인 시프트 문제**를 해결하고 **일반화 성능**을 향상시키기 위해 **Progressive Ensemble Networks (PrEN)**를 제안한다. 핵심 아이디어는 두 가지이다:

1. **다중 투영 레이블 임베딩을 가진 앙상블 네트워크**: 단일 레이블 임베딩 대신,seen 클래스와 무작위로 선택된 unseen 클래스 부분 집합 간의 레이블 관계를 최대화하도록 학습된 $K$개의 서로 다른 레이블 임베딩 투영을 사용한다. 이를 통해 $K$개의 분류 함수를 구축하고 앙상블 방식으로 예측하여 노이즈와 클래스 편향을 줄인다.
2. **점진적 학습 프레임워크**: unlabeled unseen 클래스 데이터에 현재 앙상블 네트워크가 예측한 가짜 레이블을 할당하고, 이 가짜 레이블링된 인스턴스들을 기존 훈련 데이터에 동적으로 추가하여 앙상블 네트워크 파라미터를 점진적으로 정제한다. 이는 seen 클래스에 대한 과적합을 방지하고 도메인 시프트를 완화한다.

실험 결과는 PrEN이 Conventional ZSL과 Generalized ZSL (GZSL) 모두에서 최신 방법들을 능가하는 우수한 성능을 보여주며, 특히 복잡한 데이터셋에서 효과적임을 입증했다. 추가 계산 비용은 미미하여 실용적인 활용 가능성이 높다. 이 연구는 레이블이 부족한 새로운 카테고리에 대한 이미지 인식 능력 향상에 중요한 기여를 하며, 실제 환경에서의 광범위한 적용 가능성을 시사한다.
