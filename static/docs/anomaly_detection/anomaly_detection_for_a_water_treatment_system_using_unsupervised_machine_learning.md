# Anomaly Detection for a Water Treatment System Using Unsupervised Machine Learning

- **저자**: Jun Inoue, Yoriyuki Yamagata, Yuqi Chen, Christopher M. Poskitt, Jun Sun
- **발표연도**: 2017
- **arXiv**: https://arxiv.org/abs/1709.05342

## 1. 논문 개요

이 논문은 Cyber-Physical System(CPS)에서 발생하는 이상(anomaly)을, 시스템의 물리 법칙이나 제어 로직을 사람이 직접 모델링하지 않고도 탐지할 수 있는지 살펴본 연구다. 저자들은 **비지도 학습(unsupervised machine learning)** 기반 이상 탐지를 물 처리 시스템인 SWaT(Secure Water Treatment) 테스트베드에 적용하고, 두 가지 방법인 **DNN 기반 확률적 이상 탐지기**와 **one-class SVM**을 비교한다.

연구 문제는 분명하다. CPS는 센서, 액추에이터, 네트워크, PLC, 물리 프로세스가 강하게 얽혀 있어 전통적인 규칙 기반 또는 모델 기반 방법으로 정확한 이상 탐지 모델을 만들기 어렵다. 특히 실제 산업 설비에서는 정상 동작 로그는 많이 확보할 수 있어도, 다양한 실제 공격이나 고장 데이터를 충분히 수집하기는 어렵다. 따라서 정상 로그만으로 정상 행동의 분포를 학습하고, 여기에 어긋나는 패턴을 이상으로 판단하는 비지도 접근이 매우 중요하다.

이 문제가 중요한 이유는 단순히 보안 공격 탐지에만 있지 않다. 논문에서도 밝히듯, 이상은 공격뿐 아니라 고장, 운용자 실수, 소프트웨어 버그, 설정 오류에서도 발생할 수 있다. 즉, 이상 탐지는 **방어 메커니즘**이면서 동시에 **운영 유지보수 도구**이기도 하다. 특히 물 처리 플랜트처럼 공공 인프라에 해당하는 CPS에서는 이상 탐지의 성능이 안전성과 직결된다.

이 논문은 SWaT의 7일 정상 로그로 탐지기를 학습하고, 4일간 36개 공격 시나리오가 포함된 로그에서 성능을 평가한다. 전체 결론은 다음과 같다. **DNN은 false positive를 더 적게 내서 precision이 높고, one-class SVM은 약간 더 많은 이상을 잡아 recall이 조금 높다. 전체적으로는 DNN의 F measure가 소폭 더 좋다.**

## 2. 핵심 아이디어

이 논문의 핵심 아이디어는 시간에 따라 변하는 CPS 로그를 정상 패턴의 확률 구조로 학습한 뒤, **현재 관측이 정상 분포에서 얼마나 드문지**를 점수화하여 이상을 탐지하는 것이다. 저자들은 이 아이디어를 서로 다른 관점의 두 모델로 구현했다.

첫 번째는 DNN 기반 방법이다. 이 방법은 시계열 로그의 과거 이력을 바탕으로 현재 시점의 액추에이터 상태와 센서 값을 생성할 확률을 모델링한다. 직관적으로 말하면, “이전까지의 로그가 이랬다면 지금 이런 값이 나올 가능성이 얼마나 높은가?”를 계산한다. 가능성이 매우 낮으면 outlier factor가 커지고, 이를 이상으로 본다. 이 접근은 CPS를 명시적으로 수학 모델로 기술하지 않아도, 데이터만으로 비선형 동작과 시간 의존성을 학습할 수 있다는 장점이 있다.

두 번째는 one-class SVM이다. 이 방법은 확률 자체를 모델링하기보다, 슬라이딩 윈도우로 잘라낸 시계열 조각이 **정상 데이터의 경계 안에 있는지 밖에 있는지**를 분류한다. 즉, 정상 시계열의 모양(shape)을 고차원 공간에서 둘러싸는 경계를 학습하고, 그 밖에 있는 패턴을 이상으로 본다.

기존 접근과의 차별점은 두 가지다. 첫째, 저자들은 CPS의 복잡한 물리 모델이나 제어 규칙을 수작업으로 설계하지 않는다. 둘째, DNN 쪽에서는 단순한 prediction error 기반이 아니라, **다음 상태의 확률분포를 직접 모델링**한다. 논문은 이 점을 분명히 강조한다. 특히 센서와 액추에이터가 혼합된 로그에서, 단순 오차 기반 방식은 threshold 설정이 어렵고 이산값/연속값 혼합 처리도 까다로운데, 확률 밀도 추정 방식은 이를 더 자연스럽게 다룰 수 있다는 주장이다.

또 하나의 중요한 메시지는, 이상 탐지 성능이 단순히 “더 많이 잡느냐”만으로 결정되지 않는다는 점이다. SVM은 더 민감해서 recall이 조금 높지만, 정상적인 급격한 변화까지 이상으로 보는 경향이 있어서 false positive가 늘어난다. 반대로 DNN은 더 보수적이어서 precision이 높다. 즉, 이 논문은 단일 최고 성능 모델을 주장하기보다, 두 방법의 **탐지 성향의 차이**를 실험적으로 보여주는 데도 의미가 있다.

## 3. 상세 방법 설명

### 3.1 전체 문제 설정

SWaT 로그는 시간 순서대로 기록된 센서와 액추에이터의 상태열이다. 저자들은 한 시점의 로그 엔트리를 액추에이터 벡터와 센서 벡터의 결합으로 본다. 액추에이터는 밸브 열림/닫힘, 펌프 on/off처럼 이산적인 경우가 많고, 센서는 수위, 유량, 압력처럼 연속값이다. 따라서 모델은 **이산값과 연속값이 섞인 고차원 시계열**을 다뤄야 한다.

### 3.2 DNN 기반 이상 탐지

#### 3.2.1 기본 아이디어

DNN 방법은 현재 로그 엔트리 $l_i$가 직전까지의 이력 $\mathbf{l}_{i-1}$ 아래에서 얼마나 가능성 높은지 계산한다. 이상 점수는 다음과 같은 음의 로그 확률 형태다.

$$
-\log q(l_i \mid \mathbf{l}_{i-1})
$$

이 값이 클수록 현재 관측은 정상 데이터 분포에서 드문 사건이므로 이상일 가능성이 높다.

하지만 실제로는 한 시점의 전체 로그 엔트리에 대해 직접 확률을 쓰기 어렵다. 이유는 액추에이터 조합 수가 많고, 센서 값은 연속 변수라 경우의 수가 무한대이기 때문이다. 그래서 논문은 이를 조건부 확률의 연쇄 형태로 분해한다.

$$
-\sum_{j=1}^{n}\log q_j!\left(a_j^{(i)} \mid \mathbf{l}_{i-1}, a_1^{(i)}, \dots, a_{j-1}^{(i)}\right) - \sum_{k=1}^{m}\log r_k!\left(s_k^{(i)} \mid \mathbf{l}_{i-1}, \bar{a}^{(i)}, s_1^{(i)}, \dots, s_{k-1}^{(i)}\right)
$$

여기서 첫 번째 합은 액추에이터들에 대한 조건부 확률이고, 두 번째 합은 센서들에 대한 조건부 확률이다. 논문의 설명에 따르면 $q_j$는 **discrete distribution**, $r_k$는 **Gaussian distribution**으로 근사된다. 즉, 액추에이터는 범주형 출력으로, 센서는 평균과 분산을 예측하는 가우시안 출력으로 다룬다.

이 분해는 상당히 중요하다. 왜냐하면 이 방식 덕분에 하나의 복합적인 CPS 상태를 “순차적으로 생성되는 변수들”처럼 표현할 수 있고, 혼합형 데이터도 단일 네트워크 안에서 다룰 수 있기 때문이다.

#### 3.2.2 네트워크 구조

논문에 따르면 DNN은 먼저 과거 로그들을 **LSTM**에 입력하여 시계열 문맥을 요약한다. LSTM의 출력은 현재 시점의 액추에이터 상태 예측에 사용된다. 액추에이터 위치는 softmax를 통해 각 상태의 확률로 예측된다.

그다음, 현재 시점의 **실제 액추에이터 값**을 hidden representation과 결합하여 첫 번째 센서의 분포를 예측한다. 이 결합에 논문은 **bilinear function**을 사용한다고 설명한다. 다시 말해, 예측은 단순히 “과거 이력”에만 의존하지 않고, 현재 시점에서 이미 관측된 변수들에도 순차적으로 조건부 의존을 둔다. 이후 첫 번째 센서의 실제 값까지 반영하여 다음 센서 분포를 예측하고, 이런 과정을 센서 끝까지 반복한다.

정리하면 DNN의 흐름은 다음과 같다.

과거 로그 $\rightarrow$ LSTM 문맥 벡터 생성
문맥 벡터 $\rightarrow$ 현재 액추에이터 분포 예측
실제 액추에이터 값 + hidden state $\rightarrow$ 첫 센서의 평균/분산 예측
실제 첫 센서 값까지 반영 $\rightarrow$ 다음 센서 예측
이 과정을 모든 센서까지 반복
각 확률의 로그를 더해 outlier factor 계산

즉, 이 모델은 “현재 시점의 전체 상태”를 한 번에 뱉는 것이 아니라, **조건부 생성 모델**처럼 한 변수씩 순차적으로 모델링한다.

#### 3.2.3 학습 목표와 손실 함수

논문은 cross entropy를 출발점으로 설명한다. 실제 분포 $p$와 모델 분포 $q$ 사이의 cross entropy는 다음과 같이 정의된다.

$$
C(p, q)=\int -p(z)\log q(z), d\mu
$$

하지만 실제 데이터 생성 분포 $p$는 알 수 없기 때문에, 저자들은 관측값 $x$ 하나를 중심으로 한 Dirac delta 함수로 근사한다. 그러면 최종적으로 최소화해야 하는 값은 관측값의 negative log-likelihood가 된다.

$$
C(p, q) = -\log q(x)
$$

로그 전체에 대해 합치면

$$
\sum_{i=1}^{N} -\log q(l_i \mid \mathbf{l}_{i-1})
$$

이 되며, 결국 **훈련 데이터 전체의 outlier factor 합**이 cost function이 된다. 쉽게 말해, 정상 로그가 모델 아래에서 높은 확률을 갖도록 훈련하는 것이다.

이 설정은 이상 탐지 관점에서 매우 자연스럽다. 정상 데이터로 학습된 모델은 정상 패턴에 높은 확률을 주고, 비정상 패턴에는 낮은 확률을 주게 된다. 따라서 테스트 시에는 outlier factor가 큰 시점을 이상으로 판단한다.

#### 3.2.4 임계값(threshold)

실제로 이상/정상을 나누기 위해서는 outlier factor threshold가 필요하다. 논문은 이 값을 이론적으로 정하기 어렵다고 명시한다. 그래서 threshold는 실험적으로 조정한다. 이 점은 실제 적용에서 중요한 제약이다. 즉, DNN이 anomaly score를 잘 계산하더라도, 어느 선부터 경보를 울릴지는 별도 tuning이 필요하다.

### 3.3 One-class SVM 기반 이상 탐지

#### 3.3.1 시계열을 윈도우 벡터로 변환

one-class SVM은 시계열을 직접 입력으로 받지 않기 때문에, 논문은 **sliding window**를 사용한다. 로그 엔트리를 $l_i$라 하고 window size를 $w$라 하면, 하나의 윈도우는

$$
W_i = \langle l_i, l_{i+1}, \dots, l_{i+w-1}\rangle
$$

로 정의된다. 전체 길이가 $k$인 로그에서 총 $k-w+1$개의 윈도우를 추출할 수 있다.

즉, one-class SVM은 “한 시점”이 아니라 “짧은 구간의 시계열 패턴”을 입력으로 본다. 각 윈도우를 고차원 feature vector로 간주하고, 정상 윈도우들이 존재하는 영역을 학습한다.

#### 3.3.2 학습과 테스트

훈련 단계에서는 정상 로그에서 생성한 모든 윈도우를 사용해 one-class SVM을 학습한다. 테스트 단계에서는 각 윈도우에 대해 label을 만든다. 윈도우 안에 포함된 로그 엔트리 중 하나라도 이상이면 그 윈도우 전체를 abnormal로 표기한다. 이후 분류기가 내놓은 정상/이상 판정을 window label과 비교한다.

이 방식의 의미는 명확하다. 이 모델은 이상이 윈도우 어느 위치에서 일어났는지는 크게 중요하게 보지 않고, **그 구간 전체가 정상 패턴에서 벗어났는지**를 판단한다. 따라서 정확한 시점 localization 능력은 떨어질 수 있다. 논문도 window 시작 시점에 판정을 정렬해서 표시하기 때문에, 실제 이상 발생 시점과 최대 $w-1$ 엔트리 정도 차이가 날 수 있다고 설명한다.

#### 3.3.3 커널과 하이퍼파라미터

논문은 비선형 결정 경계를 위해 **RBF kernel**을 사용한다. 주요 하이퍼파라미터는 세 개다.

- $w$: 슬라이딩 윈도우 크기
- $\nu$: 정상 데이터를 이상으로 잘못 분류하는 정도와 모델 복잡도 사이의 trade-off를 조절하는 값
- $\gamma$: RBF kernel의 계수

feature vector의 차원은 한 로그 엔트리에 52개 변수(25 sensors + 26 actuators라고 본문에 있으나 합은 51이어서, 여기서는 논문이 하이퍼파라미터 부분에서 사용한 $52w$ 표기를 그대로 따른다)가 들어간다고 두고, 윈도우 전체 차원은 $52w$로 설정되어 있다. 이 부분은 본문 내 숫자 설명에 약간의 불일치가 보이지만, 논문은 SVM 설정에서 분명히 $n=52w$라고 적고 있다. 따라서 보고서는 그 표기를 그대로 따른다.

#### 3.3.4 정규화의 영향

이 논문에서 SVM 설명 중 특히 중요한 부분은 **정규화 방식이 성능에 매우 큰 영향을 준다**는 관찰이다. 테스트 데이터를 훈련 데이터의 평균과 분산으로 정규화하면 F measure가 20–30% 수준인데, 테스트 데이터를 자기 자신의 평균과 분산으로 정규화하면 거의 80%까지 오른다. 저자들은 이를 두고, SVM은 절대값 자체보다 **장기 추세 대비 상대적 변화**에 더 민감한 것일 수 있다고 해석한다.

반대로 DNN은 훈련 데이터의 평균/분산을 테스트 데이터에도 적용하는 방식이 더 좋다고 보고한다. 즉, 두 방법은 같은 데이터라도 정상화 기준에 따라 전혀 다른 성향을 보인다.

### 3.4 구현과 계산 비용

DNN은 Chainer로 구현되었고, SVM은 scikit-learn과 libsvm 기반이다. 계산 비용 차이는 매우 크다. 100차원 hidden layer DNN은 58 epochs 학습에 약 2주가 걸리고, 평가에도 약 8시간이 걸린다. 반면 최고 성능 SVM은 약 30분 학습, 10분 평가로 끝난다. 따라서 성능 차이가 미세하다면, 실무에서는 계산 비용까지 함께 고려해야 한다.

### 3.5 하이퍼파라미터 튜닝

DNN은 intermediate layer dimension과 threshold만 주로 조정했다. 학습은 batch size 10, 100 steps 단위 truncated backpropagation, activation function은 sigmoid를 사용한다. ReLU도 시도했지만 training error가 빠르게 NaN이 되었다고 보고한다. 또한 bilinear 상단의 intermediate layer를 제거한 단순 구조도 실험했지만 training error 안정성이 떨어졌다고 한다.

흥미로운 점은 DNN에서 training epoch가 많다고 F measure가 단조롭게 좋아지지 않는다는 것이다. training error는 줄어드는데, F measure는 첫 epoch부터 이미 거의 최고 수준에 도달하고, 10–30 epoch 구간은 오히려 성능이 나빠진다. held-out normal day 기준으로는 약 13 epochs에서 early stopping이 시사되지만, 공격 데이터에서의 실제 anomaly detection 성능은 더 오래 학습한 모델이 낫다고 한다. 즉, **normal reconstruction/prediction loss와 실제 anomaly detection 성능의 관계가 단순하지 않다**는 뜻이다.

SVM은 먼저 로그 스케일 grid search를 수행했고, 이후 $w=4$로 고정한 뒤 random search를 통해 더 좋은 조합을 찾았다. 최종 best instance는 다음과 같다.

- $w = 4$
- $\gamma = 0.0008181483058667633$
- $\nu = 0.004584962079820046$

## 4. 실험 및 결과

### 4.1 데이터셋과 평가 환경

실험 대상은 SWaT testbed이다. SWaT는 6단계 raw water purification plant를 축소 구현한 실제 동작 시스템으로, 분당 5갤런의 음용수를 생산할 수 있다. 물리 프로세스와 디지털 제어가 결합된 전형적인 CPS이며, 센서, 액추에이터, PLC, SCADA, historian을 포함한다.

데이터셋은 총 11일 로그로 구성된다. 이 중 7일은 정상 운영 데이터이고, 나머지 4일은 36개의 공격 시나리오가 포함된 데이터다. 공격은 네트워크 통신 링크를 통해 센서 데이터와 액추에이터 신호를 조작하는 방식으로 수행되었다. 예를 들어 밸브가 열리지 않게 하거나, 물탱크 수위를 실제보다 낮게 보고하게 만드는 식이다.

논문은 이 공격들이 단일 지점 공격뿐 아니라 multi-point, multi-stage 공격도 포함한다고 설명한다. 따라서 평가가 단순한 toy setting은 아니다. 다만 여전히 단일 테스트베드에서의 실험이며, 실제 현장 고장 데이터가 아니라 인위적으로 주입된 공격이라는 한계는 있다.

### 4.2 전체 정량 결과

논문의 핵심 비교 결과는 다음과 같다.

- DNN: Precision 0.98295, Recall 0.67847, F measure 0.80281
- One-class SVM: Precision 0.92500, Recall 0.69901, F measure 0.79628

이 결과의 해석은 단순하다. DNN은 경보를 덜 울리지만 더 정확하고, SVM은 조금 더 많은 이상을 잡지만 false alarm도 더 많다. F measure는 DNN이 근소하게 높다.

다만 논문은 직접적인 수치 비교 시 주의점을 분명히 말한다. DNN의 true positive/false positive는 **log entry 단위**로, SVM은 **window 단위**로 집계된다. 즉, 두 방법의 점수가 완전히 동일한 단위 위에서 계산된 것은 아니다. 저자들은 이후의 정성적 분석이 이런 인상을 뒷받침한다고 설명한다.

### 4.3 시계열 수준에서의 관찰

논문 설명에 따르면 DNN의 outlier factor는 공격 구간에서 크게 상승하는 경향이 있다. 하지만 모든 공격에서 그렇지는 않다. 즉, 일부 이상은 outlier factor를 크게 올리지 못한다. 반면 SVM은 간헐적인 false positive를 낸다. 논문은 비공격 구간에서 SVM false alarm이 약 0.8%의 non-attack windows에 대해 발생한다고 적고 있다.

저자들이 제시한 해석은 설득력이 있다. SVM은 고정 길이 moving window 안의 값들만 보고 장기 추세를 직접적으로 보지 않으므로, 어떤 센서 값의 **급격한 정상 변화**도 이상처럼 보일 수 있다. 실제 false positive 사례 그림에서도 abrupt change가 원인일 가능성을 지적한다.

### 4.4 공격별 recall 분석

공격별 recall 표는 이 논문의 중요한 기여 중 하나다. 전체 평균 성능만 보면 두 방법이 비슷해 보이지만, 어떤 유형의 공격을 잘 잡는지 살펴보면 차이가 드러난다.

먼저 DNN은 **센서 값이 일정한 비정상 상수값으로 고정되는 공격**을 대체로 잘 잡는다. 예를 들어 attack #10에서 FIT-401 값을 0.7보다 작게 설정하는 공격은 DNN이 recall 1.0, SVM은 0.433이다. attack #39에서도 DNN은 0.940인데 SVM은 0이다. 이는 DNN의 확률 모델이 정상 문맥 아래에서 “이런 값 조합이 나올 가능성이 매우 낮다”는 점을 잘 포착했음을 시사한다.

반면 DNN은 **점진적 변화(gradual change)**나 **이상한 actuator movement**를 잘 못 잡는다. attack #3의 “LIT-101을 매초 1mm씩 증가”는 DNN과 SVM 모두 0이고, attack #16의 점진적 감소도 모두 0이다. 이는 두 방법 모두 느린 drift에는 취약하다는 저자들의 결론과 일치한다.

SVM은 일부 **out-of-range 값**에는 더 강한 모습을 보인다. 예를 들어 attack #7 “LIT-301 increased above HH”는 DNN 0, SVM 0.888이고, attack #32 “LIT-301 to above HH”는 DNN 0, SVM 0.905다. 하지만 항상 그런 것은 아니다. attack #31, #33에서는 out-of-range 계열로 보이는 공격도 SVM이 잡지 못한다. 따라서 SVM의 장점을 단순히 “범위 초과 탐지에 강하다”로 일반화하기는 어렵고, 논문도 그렇게 조심스럽게 서술한다.

흥미로운 사례는 두 모델 모두 아주 잘 잡는 공격들이다. 예를 들어 attack #22는 DNN 0.998, SVM 1.0이고, attack #37은 둘 다 1.0이다. 즉, 시스템 상태를 충분히 크게 뒤흔드는 복합 공격은 양쪽 모두에게 명백한 비정상 패턴으로 보였다고 해석할 수 있다.

반대로 둘 다 거의 못 잡는 공격도 많다. attack #1, #2, #4, #13, #14, #24, #29, #34, #35 등은 recall이 거의 0이다. 이는 두 가지 가능성을 시사한다. 하나는 공격이 실제 물리 상태 변화로 충분히 전파되지 않았을 수 있다는 점이고, 다른 하나는 로그 기반 비지도 탐지 자체가 제어 로직 수준의 이상을 충분히 반영하지 못할 수 있다는 점이다. 논문도 actuator behavior를 더 잘 포착하려면 controller logic을 고려할 필요가 있다고 말한다.

### 4.5 계산 효율과 실용성

정확도 차이는 매우 근소하지만, 계산 비용 차이는 압도적이다. DNN은 2주 학습과 8시간 평가가 필요했고, SVM은 최고 설정에서도 30분 학습과 10분 평가면 충분했다. 따라서 실제 시스템 도입 시에는 “정확도가 약간 더 높다”는 사실만으로 DNN을 선택하기 어렵다. 반대로 false positive를 줄이는 것이 더 중요한 운영 환경이라면 DNN의 높은 precision이 큰 장점이 될 수 있다.

즉, 이 논문의 실험 결과는 단순한 우열보다, **precision 중심 운영이면 DNN, 계산 효율과 약간 높은 sensitivity가 중요하면 SVM**이라는 실무적 판단 근거를 제공한다.

## 5. 강점, 한계

이 논문의 가장 큰 강점은 CPS 이상 탐지라는 어려운 문제를, 이론 설명에 그치지 않고 **실제 산업형 테스트베드 SWaT의 장기간 로그와 다수의 공격 시나리오** 위에서 비교 평가했다는 점이다. 특히 SWaT 전체 시스템을 대상으로 하고, 36개의 공격 시나리오에 대해 분석했다는 점은 당시 관련 연구 대비 상당히 의미 있다. 또한 단순 평균 점수뿐 아니라 공격별 recall을 제시하여 모델의 탐지 성향 차이를 구체적으로 드러낸 것도 장점이다.

또 다른 강점은 DNN 설계 자체에 있다. 이 모델은 시계열 문맥을 다루기 위해 LSTM을 사용하고, 액추에이터의 이산 분포와 센서의 연속 분포를 하나의 네트워크 안에서 함께 다룬다. 이는 혼합형 CPS 로그를 다루는 데 적절한 설계다. 단순 prediction error thresholding보다 확률 밀도 추정에 기반한다는 설명도 논리적으로 일관된다.

반면 한계도 분명하다. 첫째, 실험은 SWaT 하나에만 국한되어 있다. 논문도 직접 밝히듯, 다른 CPS에 일반화될지는 알 수 없다. 물 처리 시스템의 동역학과 제어 로직, 공격 특성은 다른 산업 설비와 다를 수 있다.

둘째, 이상 데이터가 실제 현장의 자연 발생 고장이나 공격이 아니라 **의도적으로 주입된 공격 시나리오**다. 따라서 실제 이상이 더 미묘하거나 더 복합적일 때 성능이 유지될지는 미지수다.

셋째, DNN과 SVM 모두 **점진적 변화**와 **비정상 actuator 동작**에 약하다. 이는 로그 기반 비지도 학습이 관측값의 통계 패턴은 잡아도, 제어 목적과 제약 조건 같은 semantics를 충분히 반영하지 못하기 때문일 수 있다. 논문도 controller logic을 고려해야 할 가능성을 언급한다.

넷째, DNN의 threshold 선택은 실험적으로 해야 하며, epoch 수와 실제 anomaly detection 성능의 관계도 직관적이지 않다. normal held-out error로는 좋은 stopping point가 잡히지 않았고, 실제 공격 탐지 성능과 어긋났다. 즉, 실무에서 재현성과 튜닝 난이도가 낮다고 보기는 어렵다.

다섯째, SVM과 DNN의 precision/recall 집계 단위가 다르다는 점은 비교 해석을 다소 복잡하게 만든다. DNN은 log entry 기준, SVM은 window 기준이므로, 표의 수치를 완전히 동등한 척도로 받아들이면 안 된다. 논문이 이를 명시한 점은 정직하지만, 동시에 비교 실험 설계의 제약을 보여준다.

비판적으로 보면, 논문은 DNN의 확률 모델링이 왜 특정 공격 유형에 강하고 어떤 유형에 약한지에 대한 더 깊은 분석까지는 제공하지 않는다. 예를 들어 gradual drift가 왜 잘 안 잡히는지, actuator anomaly를 강화하려면 어떤 표현이 필요한지에 대한 구조적 해석은 제한적이다. 하지만 이 점은 후속 연구 과제로 남겼다고 보는 것이 공정하다.

## 6. 결론

이 논문은 CPS 이상 탐지 문제에 대해, **정상 로그만으로 학습하는 비지도 학습 기반 탐지**가 실제로 의미 있는 성능을 낼 수 있음을 보여준다. 저자들은 SWaT 물 처리 테스트베드에서 DNN 기반 확률적 이상 탐지기와 one-class SVM을 비교했고, DNN이 더 높은 precision과 약간 더 높은 F measure를, SVM이 약간 더 높은 recall을 보인다는 점을 확인했다.

논문의 실질적 기여는 세 가지로 요약할 수 있다. 첫째, CPS 시계열 로그에 대해 LSTM 기반 확률 모델을 설계하고, 이를 outlier factor 계산으로 연결했다. 둘째, SWaT 전체 시스템과 36개 공격 시나리오에 대한 비교 실험을 수행했다. 셋째, 두 방법이 어떤 유형의 이상에 강하고 약한지 실험적으로 드러냈다.

실제 적용 측면에서 이 연구는 “정상 로그만 충분하면, 복잡한 시스템 모델을 수작업으로 만들지 않고도 이상 탐지가 가능하다”는 가능성을 보여준다. 특히 공정 산업, 스마트 인프라, ICS/SCADA 보안 같은 분야에서 출발점이 될 수 있다. 다만 계산 비용, threshold tuning, gradual anomaly 탐지 한계, actuator behavior 포착 한계가 남아 있으므로, 실서비스에 바로 투입할 수준의 완성형 방법이라기보다 **유망한 기반 연구**로 보는 것이 적절하다.

향후 연구 방향도 논문 안에서 비교적 설득력 있게 제시된다. 장기 추세를 더 잘 반영하는 feature 설계, 센서/액추에이터별 outlier factor 분해, 시뮬레이터를 활용한 더 폭넓은 공격 실험, 다른 neural/statistical/specification mining 방법과의 비교가 필요하다. 따라서 이 논문은 “비지도 CPS 이상 탐지의 초기 실증 연구”로서 가치가 크며, 이후 산업 시계열 anomaly detection 연구에 충분히 연결될 수 있는 작업이다.
