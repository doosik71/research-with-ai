# Explainable AI

## 서론

### 1. 연구 배경

이 보고서는 **Explainable Artificial Intelligence (XAI)** 분야의 연구 동향을 종합적으로 분석한다. 연구 범위는 XAI 의 개념적 정립, 모델 투명성 분류, 사후 설명 기법, 사회적 투명성 프레임워크 등 이론적 기초부터 실증적 비교 분석, 도메인별 요구사항 도출 및 적용 방안까지 포함한다. 분석 대상은 XAI 분류 체계 및 개념 정립, 모델 특정적 기법 비교, 사용자 중심 요구 탐색, 플랫폼 통합 등 다양한 관점에서 XAI 방법을 체계화한 12 편의 논문이다.

### 2. 문제의식 및 분석 필요성

XAI 는 AI 시스템의 투명성, 책임성, 신뢰성 확보를 목표로 하되, 방법론별로 서로 다른 문제 접근과 적용 맥락을 가진다. 단일 XAI 방법론의 보편적 우월성은 존재하지 않으며, 데이터셋/도메인/사용자 특성에 따라 효과성이 크게 달라진다. 이에 따라 본 연구는 **이론적 프레임워크와 실증적 검증, 도메인 적용**이라는 세 가지 차원에서 XAI 연구 지형을 종합 분석하고, 방법론별 특성과 트레이드오프를 규명하는 것을 목적으로 한다.

### 3. 보고서의 분석 관점

본 보고서는 다음 세 가지 관점에서 문헌을 정리한다. 첫째, **연구체계 분류**로 XAI 연구의 이론적, 실증적, 도메인별 연구 유형을 대분류화한다. 둘째,**방법론 분석**으로 분석·분류, 실증·비교, 설계·탐색, 규제·플랫폼이라는 4 가지 방법론 계열의 차이점과 선택 가이드를 제시한다. 셋째,**실험결과 분석**으로 데이터셋 의존성, 도메인 특수성, 평가 지표 불일치 등 맥락 의존성을 규명한다.

### 4. 보고서 구성

본 보고서는 다음 세 장으로 구성된다. **1 장 연구체계 분류**에서는 XAI 연구가 이론적 프레임워크 수립, 실증적 비교 분석, 도메인별 요구사항 도출이라는 세 가지 차원에서 어떻게 전개되는지 정리한다.**2 장 방법론 분석**에서는 4 가지 방법론 계열의 공통 문제 설정, 구조적 패턴, 학습 방식, 선택 기준을 비교 분석한다.**3 장 실험결과 분석**에서는 주요 실험의 평가 구조, 성능 패턴, 일반화 한계 등을 종합적으로 검토한다.

이 요청을 이해했습니다. 제공된 12 편의 논문 요약문을 바탕으로 "1 장. 연구체계 분류" 마크다운 문서를 작성하겠습니다.

먼저 논문들을 분석하여 분류 기준을 설정하고, 이를 바탕으로 체계적인 분류를 구성하겠습니다.

## 2장. 방법론 분석

### 1. 공통 문제 설정 및 접근 구조

#### 1.1 연구 문제의 공통성

제공된 12 편의 논문은 모두 **AI 시스템의 설명 가능성 (Explainability) 확보**라는 공통 문제를 다룬다. 그러나 각 논문은 설명 가능성 문제를 서로 다른**입력-처리-출력 구조**로 접근한다.

```text
┌────────────────────────────────────────────────────────────────┐
│                   공통 문제: 설명 가능성 확보                  │
├────────────────────────────────────────────────────────────────┤
│  입력                    처리                   출력           │
│ ──────────               ──────────             ──────────     │
│ • AI 모델/시스템         • 분석/평가/적용       • 설명 생성    │
│ • 도메인 데이터/맥락     • 분류/비교/추상화                    │
│ • 사용자 요구사항        • 방법론 도출                         │
│ • 법적/윤리적 규제       • 프레임워크 제시                     │
│ • 지식 그래프/트리플                                           │
└────────────────────────────────────────────────────────────────┘
```

| **실증 중심**   | 모델/기술 적용, 성능 비교, 시각화              | Knowledge Graph Embeddings (2020), Comparative Study (2025) |

### 2. 방법론 계열 분류

제공된 논문들은 **4 가지 주요 방법론 계열**로 분류된다.

#### (1) 분석·분류 계열

문헌 검토, 분류 체계 구축, 개념 정의를 중심으로 설명 가능성 방법론을 체계화하는 접근법.

| 방법론 계열         | 논문명                           | 핵심 특징                                                                                                                  |
| ------------------- | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **분류 체계 구축**  | Explainable AI the Latest (2025) | 범위 기반 (전역/지역) + 순서 기반 (모델링 전/중/후) 이원 분류, 메타 추론, 보상 투영等新 패러다임 제시                      |
| **책임 프레임워크** | Explainable AI (2019)            | 청중 중심 정의, 모델 투명성 3 수준 (시뮬레이션/분해/알고리즘적), ML/딥러닝별 사후 설명 기법 분류, 책임 있는 AI 3 요소 통합 |
| **문헌 종합**       | Drug Discovery (2022)            | 선행 연구 문헌 분석, ChatGPT 초안 생성 후 인간 편집 메타 연구 병행, 데이터/윤리/기술적 장애물 체계적 검토                  |

#### (2) 실증·비교 계열

구체적인 모델이나 기법에 XAI 방법을 적용하여 성능, 효과, 한계를 실증적으로 분석하는 접근법.

| 방법론 계열                | 논문명                            | 핵심 특징                                                                                            |
| -------------------------- | --------------------------------- | ---------------------------------------------------------------------------------------------------- |
| **모델 특정적 기법**       | Knowledge Graph Embeddings (2020) | KG 임베딩과 논리적 추론 결합, 마진 손실/부정적 샘플링 최적화, 텍스트/이미지/논리 외부 정보 통합      |
| **모델 불가지론적 기법**   | Comparative Study (2025)          | ResNet50 이미지 분류에 LIME/SHAP(불가지론적) vs Grad-CAM/Guided Backprop(모델 특정적) 적용 비교 분석 |
| **시각화 기법**            | Comparative Study (2025)          | 특징 가중치 계산, 활성화 맵 합성, 지역적 선형 근사, Shapley 값, 기울기 가중치 등 설명 생성           |
| **기존 vs 신규 방법 비교** | Explainable AI the Latest (2025)  | 지상 수준 데이터 모델 한계 극복, 보상 중심 설명 가능성 + 메타 추론 결합, 다양한 도메인 일반화 검증   |

#### (3) 설계·탐색 계열

| **시나리오 기반 설계** | Generative AI Code (2022)       | 3 가지 코드 생성 시나리오 (번역/자동완성/자연어-코드), 9 개 워크숍×43 명, 디자인 프로브 (4 가지 XAI 기능 유형) 활용, 11 가지 요구사항 범주 도출 |

### 6. 종합 정리

제공된 12 편의 논문은 **분석·분류**,**실증·비교**,**설계·탐색**,**규제·플랫폼**이라는 4 가지 주요 방법론 계열로 구성되어 있다. 분석 계열은 분류 체계와 책임 프레임워크 구축에 중점을 두고, 실증 계열은 모델별 기법 적용과 성능 비교를 수행한다. 설계 계열은 사용자 요구와 디자인 권장사항 도출을 탐구하며, 규제 계열은 법률 준수와 플랫폼 통합을 목표로 한다.

방법론들은 **도메인 전이 필요**,**구체적 성능 측정**,**사용자 요구 파악**,**규제 준수**,**성능 최적화**,**통합 솔루션**이라는 6 가지 선택 기준에 따라 선택 가능하다. 초기에는 개념 정의와 분류 체계가 주를 이루었으나, 점차 도메인별 적용과 사용자 요구 파악이 강조되었고, 최근에는 방법론 비교와 플랫폼 통합이 새로운 패러다임으로 부상했다.

### 3. 성능 패턴 및 경향 분석

#### 공통적으로 나타나는 성능 개선 패턴

**KGE 모델 성능 패턴 (2020)**

- ComplEx 모델은 링크 예측 작업에서 다른 모델 대비 안정적인 성능을 유지
- DistMult 모델의 경우 하이퍼파라미터 튜닝 정도에 따라 성능 변동이 큼
- 데이터셋 필터링 설정 (테스트 트리플 외 참인 트리플 제거) 이 성능 평가에 중요한 영향 요인

**ST 정보의 영향 (2021)**

- 사회적 투명성 (ST) 정보 노출 시 모든 참가자 (26 명 중 26 명) 에게 제안 가격 감소 효과
- 기술적 투명성만 있는 경우 29 명 모두 불충분하다고 판단
- ST 가 기술적/의사결정/조직적 3 수준 맥락 가시화에 효과적

**XAI 방법론의 방법적 특화 (2025)**

- SHAP: 전역적 특징 기여도 제공에 적합 (일관성 우수)
- Grad-CAM: 공간적 활성화 강조에 효과적
- Guided Backpropagation: 가장자리 감지 능력에서 우수
- 단일 방법의 보편적 우월성 부재 확인

#### 특정 조건에서만 성능이 향상되는 경우

**데이터셋 의존성 (2020)**

- FB15k/WN18 데이터셋이 역관계 편향으로 성능 과대평가될 수 있음
- 개선 데이터셋 (FB15k-237, WN18RR) 사용 시 성능 재평가 필요
- 모델별 성능 차이가 데이터셋 선택에 크게 의존

**참가자 배경 의존성 (2022)**

- 조류 식별 앱 사용자의 AI 배경 지식 (낮음/중간/높음) 과 관심도 차이가 평가 결과에 영향
- 일부 Prototype 은 모호하거나 clattered하다고 평가 (P₁, P₁₃, P₁₈)

**이미지 유형 의존성 (2025)**

- 개 이미지: 얼굴/눈 집중
- 야생 동물 이미지: 배경 편향
- 새 이미지: 머리와 부리 집중
- 무당벌레 이미지: 색상/패턴/가장자리 강조

#### 논문 간 상충되는 결과

**기술적 vs 사회적 설명 가능성 (2021 vs 2022)**

- Expanding Explainability (2021): ST 기능이 기술적 투명성 대비 제안 가격 감소와 자신감 향상 효과
- Help Me Help the AI (2022): Prototype 기반 설명 선호도 높으나, Concept 기반 설명은 수치 정보 과부하 지적
- 단일 XAI 기법으로는 다양한 사용 요구 충족 불가 확인

**모델 불가지론적 vs 모델 특정적 방법 (2025)**

- 모델 불가지론적 방법 (LIME, SHAP): 전역적 설명에 강점이나 계산 비용 높음
- 모델 특정적 방법 (Grad-CAM, Guided Backprop): 지역적 설명에 강점이나 아키텍처 제약 존재
- 각 방법론이 상충하는 강점/약점 보유

### 7. 종합 정리

실험 결과는 XAI 방법론의 효과성이 데이터셋, 도메인, 사용자 특성 등 맥락에 크게 의존함을 보여준다. 지식 그래프 임베딩에서는 ComplEx 가 다양한 데이터셋에서 안정적인 성능을 보이지만, 데이터셋 편향 (FB15k/WN18 의 역관계 편향) 이 성능 평가를 왜곡할 수 있음이 확인되었다. 사회적 투명성 (ST) 기능은 기술적 투명성 대비 모든 참가자 에게 제안 가격 감소와 자신감 향상 효과를 제공했으나, 도메인 전이 가능성은 불확실하다. XAI 기능 유형별 (Heatmap, Prototype, Concept, Example) 선호도는 사용자 배경과 사용 목적에 따라 다르게 나타나며, 단일 방법론의 보편적 우월성은 존재하지 않음을 시사한다. 모델 특정적 방법 (Grad-CAM, Guided Backprop) 은 공간적 활성화와 가장자리 감지에서 우수한 반면, 모델 불가지론적 방법 (SHAP) 은 전역적 설명에 강점을 보이며, 하이브리드 접근이 포괄적 해석에 효과적이다. 모든 실험에서 XAI 결과를 인간 전문가 검증과 인간-AI 협업이 필수적임이 강조되었으며, 이는 AI 예측 도구와 생성 콘텐츠의 신뢰성 부족 (ChatGPT 참고 문헌 정확도 6%, KGE 모델의 재현성 문제) 을 반영한다. 실험 설계는 각 분야별 데이터셋과 평가 지표의 불일치, 사용자 규모 제한, 도메인 특정성 등의 한계를 가지나, 이러한 맥락 의존성이 XAI 효과의 일반화 가능성을 제한하는 핵심 요인이 된다.
