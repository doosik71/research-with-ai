# Chain-of-Thought

## 서론

### 1. 연구 배경

Chain-of-Thought(연역 사고) 분야는 거대언어모델(LLM) 의 추론 능력을 향상시키기 위한 연구로 크게 (1) 프롬프팅 전략 설계, (2) 디코딩 및 집계 전략, (3) 신경-기호적 접근으로 구분된다. 프롬프팅 전략 개선은 CoT 의 기본 예시 설계에서 문제 분해를 통한 복잡성 관리까지 다양한 프롬프트 구조를 탐구하며, 디코딩 전략은 단일 최적 경로 대신 다중 경로 샘플링과 집계를 통해 견고성 확보에 집중한다. 신경-기호적 접근은 LLM 의 생성 능력과 외부 계산 시스템(Python) 의 정확성을 결합하여 계산 오류 문제를 해결한다.

### 2. 문제의식 및 분석 필요성

거대언어모델의 추론 능력 향상, 복잡한 자연어 추론 문제 해결, 계산/논리적 정확성 확보를 위해 체계적인 분석이 필요하다. Direct CoT 계열의 중간 추론 단계 추가, Sampling & Aggregation 계열의 Decoding 다양화와 다수결, Problem Decomposition 계열의 문제 분해 및 순차적 해결, Hybrid/System 계열의 NL→코드 변환과 외부 인터프리터 실행 등 다양한 접근법이 존재한다. 이러한 다양한 방법론들을 비교·분석함으로써 각 기법의 장단점과 적합한 적용 상황을 명확히 이해할 수 있다.

### 3. 보고서의 분석 관점

본 보고서는 연구체계 분류, 방법론 분석, 실험결과 분석이라는 세 가지 축으로 문헌을 정리한다. 연구체계 분류는 문제 정의, 접근 관점, 분류 우선순위, 계층적 구조 원칙에 따라 CoT 연구들을 체계적으로 분류한다. 방법론 분석은 공통 문제 설정, 방법론 계열 분류, 핵심 설계 패턴 분석을 통해 각 기법의 구조적 차이를 비교한다. 실험결과 분석은 다양한 데이터셋에서의 성능과 검증 패턴을 종합적으로 평가한다.

### 4. 보고서 구성

**1장 (연구체계 분류)**에서는 프롬프팅 전략 개선, 디코딩 및 집계 전략, 신경-기호적 접근이라는 세 가지 주요 범주로 CoT 연구들을 체계적으로 분류하고 각 범주의 대표적 논문들과 분류 근거를 제시한다.

**2 장 (방법론 분석)**에서는 공통 문제 설정을 바탕으로 각 방법론 계열의 핵심 특징을 비교하며, 핵심 설계 패턴(Chain-of-Thought 구조, Sequential Processing, Neuro-Symbolic Integration 등) 을 분석한다. 각 접근법의 문제 접근 방식, 적용 대상, 복잡도, 확장성과 트레이드오프를 종합적으로 검토한다.

**3 장 (실험결과 분석)**에서는 산술·상식·상징적 추론 등 다양한 벤치마크 데이터셋에서의 실험 결과를 정렬하고, 성능 패턴 및 경향을 분석한다. 각 방법론의 성능 향상 조건, 일반화 한계, 평가 지표의 제한점을 종합적으로 평가하며, 결과 해석의 경향과 실제 관찰 결과 간의 차이를 고찰한다.

## 1장. 연구체계 분류

### 1. 연구 분류 체계 수립 기준

본 연구 분류 체계는 다음 원칙을 바탕으로 수립되었다.

**분류 원칙:**

1. **문제 정의 관점**: 각 연구가 해결하려는 핵심 문제의 본질 (LLM 추론 능력 한계, CoT 의 일반화 문제, 계산 오류, 디코딩 한계 등) 을 기준으로 분류
2. **접근 관점**: 프롬프팅 전략, 디코딩 방법론, 시스템 구성 관점 등 기술적 접근의 차이를 반영
3. **분류 우선순위**: 하나의 논문에 대한 대표적 접근을 선택하며, 다중 범주 가능 시 가장 특징적인 접근만 배치
4. **계층적 구조**: 대분류는 주요 문제 유형을, 하위 범주는 구체적 접근 방식이나 방법론적 관점을 반영

### 2. 연구 분류 체계

#### 2.1 프롬프팅 전략 개선

프롬프팅 예시의 구조와 내용을 체계화하여 LLM 의 추론 능력을 끌어내는 기법들을 포함한다.

| 분류                                         | 논문명                                                                            | 분류 근거                                                                                             |
| -------------------------------------------- | --------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------- |
| 프롬프팅 전략 개선 > CoT 프롬프팅 설계       | Chain-of-Thought Prompting Elicits Reasoning in Large Language Models (2022)      | 프롬프트 예시에 <입력, 사고의 사슬, 출력>을 포함하여 중간 추론 단계를 명시적으로 설계한 프롬프팅 기법 |
| 프롬프팅 전략 개선 > 문제 분해 기반 프롬프팅 | Least-to-Most Prompting Enables Complex Reasoning in Large Language Models (2022) | 복잡한 문제를 하위 문제로 분해하고 이전 해답을 활용하는 프롬프팅 전략을 설계                          |
| 프롬프팅 전략 개선 > 프로그램 생성 프롬프팅  | PAL: Program-aided Language Models (2022)                                         | NL/PL 번갈기 프롬프팅과 Python 코드 생성을 유도하는 프롬프트 구조 설계                                |

#### 2.2 디코딩 및 집계 전략

LLM 의 생성 과정을 다양화하거나 결과를 집계하여 추론 견고성을 높이는 기법들을 포함한다.

| 분류                                             | 논문명                                                                         | 분류 근거                                                                                   |
| ------------------------------------------------ | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------- |
| 디코딩 및 집계 전략 > 샘플링 기반 추론 경로 수집 | Self-Consistency Improves Chain of Thought Reasoning in Language Models (2022) | 탐욕스러운 디코딩 대신 온도/Top-k/Nucleus 샘플링으로 다양한 추론 경로 샘플링 후 다수결 집계 |

#### 2.3 신경-기호적 접근

LLM 의 자연어处理能力와 기호적 계산을 결합하는 하이브리드 방식을 포함한다.

| 분류                                  | 논문명                                    | 분류 근거                                                                                  |
| ------------------------------------- | ----------------------------------------- | ------------------------------------------------------------------------------------------ |
| 신경-기호적 접근 > 프로그램-기반 추론 | PAL: Program-aided Language Models (2022) | LLM 이 생성한 코드를 결정론적 Python 인터프리터에 위임하는 신경-기호적 하이브리드 아키텍처 |

### 3. 종합 정리

Chain-of-Thought 분야에서 연구된 접근법은 크게 (1) 프롬프팅 전략 설계, (2) 디코딩 및 집계 전략, (3) 신경-기호적 접근으로 구분된다. 프롬프팅 전략 개선은 CoT 의 기본 예시 설계에서 문제 분해를 통한 복잡성 관리까지 다양한 프롬프트 구조를 탐구하며, 디코딩 전략은 단일 최적 경로 대신 다중 경로 샘플링과 집계를 통해 견고성 확보에 집중한다. 신경-기호적 접근은 LLM 의 생성 능력과 외부 계산 시스템 (Python) 의 정확성을 결합하여 계산 오류 문제를 해결한다. 이러한 분류 체계는 CoT 연구의 기술적 다양성을 체계적으로 보여주는 동시에, LLM 추론 문제 해결을 위한 다층적 접근법의 발전 양상을 잘 반영한다.

## 2 장. 방법론 분석

### 1. 공통 문제 설정 및 접근 구조

| 구분            | 내용                                                                                       |
| --------------- | ------------------------------------------------------------------------------------------ |
| **공통 문제**   | 거대언어모델(LLM) 의 추론 능력 향상, 복잡한 자연어 추론 문제 해결, 계산/논리적 정확성 확보 |
| **입력**        | 자연어 질문, 프롬프트, CoT 예시, 분해 프롬프트                                             |
| **출력**        | 자연어 형태의 최종 답변, Python 정답, 다수결 집계 결과                                     |
| **일반적 구조** | Few-shot 프롬프팅 기반 → 처리 단계 추가/변형 → 최종 답변 산출                              |
| **모델 의존성** | 약 100 억 파라미터 이상에서 Emergent Ability 발현 필요                                     |

### 2. 방법론 계열 분류

| 방법론 계열                     | 논문명                                                                            | 핵심 특징                                                  |
| ------------------------------- | --------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| **Direct CoT 계열**             | Chain-of-Thought Prompting Elicits Reasoning in Large Language Models (2022)      | 중간 추론 단계 추가, Greedy Decoding, 단일 경로 처리       |
| **Sampling & Aggregation 계열** | Self-Consistency Improves Chain of Thought Reasoning in Language Models (2022)    | Decoding 샘플링, 다수결 투표, 여러 경로 집계               |
| **Problem Decomposition 계열**  | Least-to-Most Prompting Enables Complex Reasoning in Large Language Models (2022) | 문제 분해, 순차적 해결, 이전 결과 재사용                   |
| **Hybrid/System 계열**          | PAL: Program-aided Language Models (2022)                                         | NL→코드 변환, 외부 인터프리터 실행, 신경 - 기호 하이브리드 |

### 3. 핵심 설계 패턴 분석

| 패턴명                           | 정의                                                            | 해당 계열/논문                     |
| -------------------------------- | --------------------------------------------------------------- | ---------------------------------- |
| **Few-shot In-context Learning** | 추가 학습 없이 프롬프트 예시만으로 모델 파라미터 변경 없이 동작 | 모든 논문                          |
| **Chain-of-Thought 구조**        | 입력 - 추론 단계 - 출력의 3 요소 포함                           | Direct CoT, Sampling & Aggregation |
| **Sequential Processing**        | 하위 문제 또는 추론 단계를 순차적으로 처리                      | Least-to-Most, Self-Consistency    |
| **Neuro-Symbolic Integration**   | 신경망 모델 + 결정론적 외부 인터프리터 결합                     | PAL                                |
| **Greedy vs Sampling Decoding**  | 탐욕적 디코딩 vs 확률적 샘플링 비교                             | Direct CoT vs Self-Consistency     |

### 4. 방법론 비교 분석

| 비교 차원          | Direct CoT                 | Sampling & Aggregation   | Problem Decomposition   | Hybrid/System    |
| ------------------ | -------------------------- | ------------------------ | ----------------------- | ---------------- |
| **문제 접근 방식** | 추가 추론 단계만 추가      | Decoding 다양화 + 집계   | 문제 분해               | NL→코드→실행     |
| **구조/모델 차이** | 단일 경로, Greedy Decoding | 여러 경로 샘플링, 다수결 | 분해→순차적 처리        | LLM + 인터프리터 |
| **적용 대상**      | 일반 추론, 산술, 상식      | 복잡한 추론 경로         | 이지 - 투 - 하드 일반화 | 계산 정밀성 요구 |
| **복잡도**         | 낮음                       | 중간 (샘플링 매개변수)   | 중간 (분해 프롬프트)    | 중간 (코드 생성) |
| **확장성**         | 제한적                     | 샘플링 횟수에 비례       | 분해 난이도 의존        | 도메인 코드 제한 |

**트레이드오프:**

- **정확성 vs 효율성**: Sampling & Aggregation 은 정확도 향상되지만 계산 비용 증가
- **단일 경로 vs 다양성**: Direct CoT 은 빠른 반면, Sampling 은 다양한 관점 확보
- **프롬프트 기반 vs 시스템**: Hybrid 은 정확하지만 도메인 코드 제한적

## 3 장. 실험결과 분석

### 1. 평가 구조 및 공통 실험 설정

네 편 논문을 통해 공통적으로 관찰되는 평가 구조를 정리한다.

**주요 데이터셋 유형**

| 유형        | 포함 데이터셋                                                                                             |
| ----------- | --------------------------------------------------------------------------------------------------------- |
| 산술 추론   | GSM8K, SVAMP, ASDiv, AQuA, MAWPS, MultiArith                                                              |
| 상식 추론   | CSQA, StrategyQA, CommonsenseQA, ARC-challenge, DateUnderstanding, SportsUnderstanding, SayCan            |
| 상징적 추론 | Last Letter Concatenation, Coin Flip, SCAN, DROP, Colored Objects, Penguins, Object Counting, Repeat Copy |

**평가 환경**

- 대부분 퓨샷 (few-shot) 또는 퓨샷 프롬프팅 환경 사용
- 대부분 실험 환경 (실제 모델 추론, 파인튜닝 없이 in-context learning)
- 일부 시뮬레이션 요소 (Python 인터프리터 실행: PAL)

**비교 방식**

| 비교 대상                              | 사용 논문               |
| -------------------------------------- | ----------------------- |
| 표준 프롬프팅 vs CoT 프롬프팅          | CoT (2022), PAL (2022)  |
| CoT vs 프롬프트 샘플링/앙상블          | Self-Consistency (2022) |
| CoT vs 문제 분해 접근                  | Least-to-Most (2022)    |
| CoT/퓨샷 프롬프팅 vs 프로그램-에이디드 | PAL (2022)              |

**평가 지표**

- 정확도 (accuracy): 모든 논문 공통
- 성능 향상 점수 (percentage points): Self-Consistency, PAL
- 계산기 보정 성능: PAL, CoT
- OOD 일반화 능력: CoT

### 3. 성능 패턴 및 경향 분석

#### 공통적으로 나타나는 성능 개선 패턴

**CoT 프롬프팅 효과:**

- ~100 억 파라미터 이상에서만 emergent ability 관찰
- 작은 모델에서는 오히려 성능 저하
- CoT 는 약 8 개 수동 예시로 구현 가능
- 산술·상식·상징적 추론 모두에서 긍정적 효과

**Self-Consistency 효과:**

- 모든 모델 규모 (UL2-20B ~ PaLM-540B) 에서 일관된 성능 향상
- 샘플링 경로 수 40 개까지 정확도持续提升
- 작은 모델에서도 적용 가능

**Least-to-Most 효과:**

- 문제 분해 구조가 명확한 작업 (Last-letter-concatenation, SCAN) 에서 특히 우수
- 동일 도메인 내 일반화 어려움
- 분해 프롬프트는 도메인 간 이동 시 성능 저하

**PAL 효과:**

- 계산이 필요한 작업 (GSM8K, GSM-HARD, 상징적 알고리즘 작업) 에서 특히 우수
- CoT 대비 큰 숫자 문제에서 견고성 입증 (GSM-HARD: 65.6→20.1% 급락 vs 61.2% 유지)
- 다수결 ensemble 적용 시 추가적 성능 향상 (GSM8K: 80.4%)

#### 특정 조건에서만 성능이 향상되는 경우

| 방법론           | 조건                   | 성능 향상               |
| ---------------- | ---------------------- | ----------------------- |
| CoT              | ~100 억 파라미터 이상  | emergent ability 관찰   |
| Self-Consistency | 추론 경로 5~10 개 이상 | 대부분 성능 향상        |
| Least-to-Most    | 동일한 도메인 내 문제  | 분해 어려움 시 한계     |
| PAL              | 산술/코드 생성 필요    | 인터프리터 필요 시 제한 |

#### 데이터셋/환경에 따른 성능 차이

**산술 추론:**

- PAL 이 가장 견고 (GSM-HARD 포함)
- Self-Consistency 일관적 향상
- Least-to-Most는 CoT 대비 큰 개선 (39.07% → 45.23% 이상, 5 단계 이상)

**상식 추론:**

- CoT 가 SOTA 달성
- Self-Consistency 추가적 개선

**상징적 추론:**

- 작업 유형별 최적 방법론 상이 (Last-letter: Least-to-Most, 알고리즘: PAL)

### 6. 결과 해석의 경향

#### 저자들의 공통 해석 경향

**CoT 해석:**

- 자연어 중간 단계가 핵심
- OOD 시퀀스 일반화 및 길이 일반화 가능
- 표준 프롬프팅 대비 극적인 성능 스케일링 증가

**Self-Consistency 해석:**

- 다양한 사고방식이 동일한 정답으로 귀결되는 직관 활용
- 단일 언어 모델에서 작동하는 자기 앙상블 유사 동작
- 추가 훈련 없이 즉시 적용 가능

**Least-to-Most 해석:**

- recursive 접근 방식이 핵심 성능 향상 요인
- 교육 심리학에서 영감받은 이지-투-하드 일반화 달성
- 교육 심리학 이론과 연결

**PAL 해석:**

- 신경-기호 시너지가 견고한 추론 성능 제공
- 코드 LLM 이지만 자연어 LLM 에서도 일반화 가능
- LLM 의 문제 분해 능력과 기호 인터프리터 계산 능력 결합

#### 실제 관찰 결과 vs 저자 해석

| 관찰 결과                         | 저자 해석                                 |
| --------------------------------- | ----------------------------------------- |
| CoT 는 작은 모델에서 성능 저하    | emergent ability 는 ~100 억 파라미터 이상 |
| Self-Consistency 일관적 성능 향상 | 다양한 사고방식이 동일한 정답으로 귀결    |
| Least-to-Most 도메인 제한성       | 분해 프롬프트 도메인 간 일반화 어려움     |
| PAL 의 변수명 의존성              | 의미 있는 변수명 중요                     |

### 7. 종합 정리

네 편 논문에서 공통적으로 나타나는 핵심 발견은 **LLM 의 추론 능력 향상 방법이 작업 유형과 모델 규모에 따라 상이한 최적 방법론을 가진다**는 점이다. 산술 및 계산이 필요한 작업에서는 PAL 이 CoT 기반 모델보다 15%p 이상 우수하며, 큰 숫자 문제에서도 견고성을 입증한다. Self-Consistency 는 모든 모델 규모에서 일관된 성능 향상 (+1.7~+17.9 pp) 을 보이며, 특히 계산 비용은 증가하더라도 5~10 개의 추론 경로 샘플링 만으로도 대부분 성능 향상을 획득할 수 있다. Least-to-Most 는 문제 분해 구조가 명확한 상징적 추론 작업 (Last-letter-concatenation, SCAN) 에서 CoT 대비 2~8 배 더 우수한 성능을 보이나, 도메인 간 일반화에는 제한이 있다. CoT 는 ~100 억 파라미터 규모의 모델에서만 emergent 추론 능력을 활성화하며, 작은 모델에서는 오히려 성능이 저하된다. 이러한 결과는**추론 문제의 특성 (계산 필요 여부, 분해 가능성) 과 모델 규모에 따라 최적의 프롬프팅 전략이 달라진다는 점**을 시사한다.
