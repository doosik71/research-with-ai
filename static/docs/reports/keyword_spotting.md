# Keyword Spotting

## 서론

### 1. 연구 배경

키워드 스포팅 (KWS) 분야는 모바일, 엣지, MCU 등 제한적 자원 환경에서 음성 인식 정확도와 연산 효율성을 동시에 확보하는 기술적 과제를 중심으로 발전해 왔다. 본 보고서에서는 2017 년부터 2025 년까지의 KWS 연구들을 체계적으로 분류·분석하여, 온디바이스 및 경량 모델 아키텍처, 전이학습 및 소수샘플 학습, 손실함수 및 학습 전략 최적화, 교차모달 및 텍스트 기반 접근법, 모델 효율화 및 압축, 데이터셋 및 벤치마크, 시스템 통합 및 적용 등 7 가지 주요 범주로 연구 지형을 구조화했다.

### 2. 문제의식 및 분석 필요성

기존 KWS 연구들은 각자의 접근법과 평가 기준을 따르지만, 모바일/엣지 환경 배포 시 나타나는 자원 제약 해결, Few-shot/Zero-shot 일반화 전략, 실시간 처리 요구사항 등 핵심 문제들이 서로 상충되는 목표를 가진다. 또한, 정확도 대비 효율성 트레이드오프, 데이터 효율성 문제, 노이즈 강건성 확보 등의 기술적 난제들이 명확하게 정리되어 있지 않아, 연구자들이 최적의 아키텍처를 선택할 때 기준이 부족하게 된다. 이러한 문제의식을 해결하기 위해 본 보고서는 다양한 방법론 계열과 실험 결과를 비교 분석함으로써, 특정 시나리오에 적합한 접근법을 체계적으로 제시하고자 한다.

### 3. 보고서의 분석 관점

본 보고서는 연구체계 분류, 방법론 분석, 실험결과 분석이라는 세 가지 관점을 바탕으로 문헌을 정리한다. 연구체계 분류에서는 (1) 적용 대상 하드웨어/환경, (2) 주요 학습 방법론, (3) 핵심 시스템 아키텍처 세 가지 기준으로 각 논문의 가장 대표적인 범주 하나만 선택하여 명확한 분류 체계를 구성한다. 방법론 분석에서는 전trained 임베딩, 효율적 컨볼루션, 스트리밍 처리, Few-shot/Zero-shot 학습, 오픈보캐블러리/교차모달, 논리/이산 모델, NAS, 비지도 학습 등 8 가지 방법론 계열의 공통 특징과 차이점을 비교하며, 특징 추출 패턴, 음향 모델 구조, 손실함수, 학습 전략 등을 분석한다. 실험결과 분석에서는 정확도, 효율성, 일반화 성능 등의 주요 결과를 정렬하고, ablation study, 민감도 분석, 조건 변화 실험 등의 추가 검증 패턴을 종합한다.

### 4. 보고서 구성

본 보고서는 총 3 장으로 구성된다. 1 장에서는 연구체계 분류를 통해 KWS 연구들을 7 가지 주요 범주로 정돈하고, 각 논문의 분류 근거와 대표적 접근법을 체계적으로 제시한다. 2 장에서는 방법론 분석을 수행하여, 다양한 방법론 계열의 공통 특징, 설계 패턴, 구조적 차이, 적용 환경에 따른 적합성 등을 비교 분석하며, 모델 진화 흐름과 핵심 통찰을 도출한다. 3 장에서는 실험결과 분석을 통해 주요 모델의 정확도와 효율성 데이터를 정렬하고, 성능 패턴, 조건별 성능 차이, 평가 지표 한계 등 결과를 종합하며, 각 방법론의 강점과 한계를 명확히 제시한다.

## 1장. 연구체계 분류

### 1. 연구 분류 체계 수립 기준

- **분류 원칙**: 각 논문은 (1) 적용 대상 하드웨어/환경, (2) 주요 학습 방법론, (3) 핵심 시스템 아키텍처 세 가지 관점에서 종합적으로 검토됨
- **범주 설계**: 엣지/온디바이스 효율성, 전이/소수샘플 학습, 아키텍처 설계, 손실함수 최적화, 교차모달, 모델 효율화 등 핵심 기술 주제를 대분류로 구성하고, 구체적인 접근법을 하위 범주로 세분화
- **배치 기준**: 각 논문의 가장 대표적인 1 개 범주만 선택하여 중복 없이 명확한 분류 체계 제시

### 3. 종합 정리

본 분석은 온디바이스 및 경량 모델 아키텍처, 전이 학습 및 소수샘플 학습, 손실 함수 및 학습 전략 최적화, 교차 모달 및 텍스트 기반 접근법, 모델 효율화 및 압축, 데이터셋 및 벤치마크, 시스템 통합 및 적용 등 7 개 주요 범주로 2017 년부터 2025 년까지의 키워드 스포팅 연구 지형을 구조화하였다. 온디바이스 환경의 자원 제약 해결을 위한 CNN/RNN/트랜스포머 아키텍처 설계, 전이 학습과 소수샘플 학습을 통한 일반화 전략 개선, 효율적인 손실 함수와 데이터 증강 기법의 통합, 교차 모달 접근을 통한 오픈 보캐블러리 지원, 그리고 모델 양자화와 지식 증류 등을 통한 경량화 기술들이 상호 보완적으로 발전해 왔음을 확인할 수 있다. 또한 모바일/엣지 환경에서의 실시간 처리, 에너지 효율성, 잡음 강건성 요구사항에 맞춰 시스템 통합 및 최적화가 지속적으로 강조되고 있으며, 최근에는 SNN 기반 신경모방, 어두운 지식 기반 증분 학습, 다국어 전이 학습 등 새로운 기술적 돌파구가 등장하고 있다.

## 2장. 방법론 분석

### 1. 공통 문제 설정 및 접근 구조

키워드 스포팅 (KWS) 관련 논문들은 공통적으로 다음과 같은 문제를 다룹니다:

| 구성 요소     | 일반적 처리 방식                                                    |
| ------------- | ------------------------------------------------------------------- |
| **입력**      | 원시 오디오 → 특징 추출 (MFCC, Mel 스펙트로그램, FBank 등)          |
| **출력**      | 키워드 존재 확률, 클래스 확률 분포, 탐지 점수, 제로샷 탐지          |
| **핵심 목표** | 제한된 자원 환경 (모바일, 엣지, MCU) 에서 정확도와 효율성 균형 달성 |

**일반적 파이프라인:**

```text
오디오 신호 → 특징 추출 → 음향 모델 → 사후 처리 → 탐지 결정
```

### 5. 방법론 흐름 및 진화

#### 5.1. 초기 접근 (2015-2017)

| 시기          | 주요 특징                     | 대표 논문                                     |
| ------------- | ----------------------------- | --------------------------------------------- |
| **2015-2016** | RNN/CTC 기반, HMM 대체        | Online KWS with CRNN (2015), KWS & VAD (2016) |
| **2017**      | CNN 도입, ResNet, 효율성 추구 | Honk, Deep Residual Learning, ConvMixer       |
|               |                               | Cascade Architecture, Streaming S2S           |

**진화 흐름:**

```text
HMM → CTC-RNN → CNN (2D) → 효율적 CNN (1D/DS-Conv) → ResNet → 트랜스포머
```

#### 5.2. 발전된 구조 (2018-2021)

| 시기          | 주요 특징                               | 대표 논문                             |
| ------------- | --------------------------------------- | ------------------------------------- |
| **2018-2019** | 전trained 임베딩, 데이터 증강, 스트리밍 | SpecAugment, Few-Shot, Streaming KWS  |
| **2020**      | 효율적 컨볼루션, Few-shot, NAS          | MatchboxNet, Depthwise SE-ResNet, NAS |
| **2021**      | 오픈보캐블러리, 어텐션, 대조 학습       | QbyE, Open-vocab AdaIN, Contrastive   |

**진화 흐름:**

```text
효율성 극대화 → Few-shot/Zero-shot → 오픈보캐블러리 → 교차 모달
```

#### 5.3. 최근 경향 (2022-2025)

| 시기          | 주요 특징                                | 대표 논문                                 |
| ------------- | ---------------------------------------- | ----------------------------------------- |
| **2022-2023** | 다중 조건 학습, 커리큘럼, 오픈보캐블러리 | ConvMixer, Learning Audio-Text Agreement  |
| **2024**      | 대조 학습, 어두운 지식, SNN              | Contrastive Augmentation, DE-KWS, SNN-KWS |
| **2025**      | BNN 지속 학습, 종합 검토                 | BNN-CL, Comprehensive Review              |

**진화 흐름:**

```text
에너지 효율성 극대화 → 지속 학습 → 대조/비지도 학습 → SNN/논리 기반
```

### 6. 종합 정리

키워드 스포팅 방법론은 **다양한 자원 제약 환경** (모바일, 엣지, MCU) 과 **데이터 효율성** (Few-shot, Zero-shot, 오픈보캐블러리) 요구에 따라 체계적으로 진화했습니다.

**방법론 지형의 주요 축:**

1. **효율성 축:** 파라미터 수, 연산량, 에너지 소비 최적화
2. **일반화 축:** Few-shot, Zero-shot, 다국어, 오픈보캐블러리
3. **처리 축:** 스트리밍, 실시간, 지연 시간 제어
4. **학습 축:** 지도, 비지도, 자기지도, 지속 학습

**방법론 분류:**

- **전trained 임베딩:** 대규모 코퍼스에서 전이 학습
- **효율적 컨볼루션:** DS-Conv, TC-ResNet, Multi-scale
- **스트리밍:** RNN-T, 어텐션, SVDF
- **Few-shot:** Prototypical, Triplet, GE2E
- **오픈보캐블러리:** Audio-Text 매칭, AdaIN
- **논리/이산:** Tsetlin Machine, SNN, BNN
- **NAS:** DARTS, 인코더-디코더 최적화
- **비지도:** 대조 학습, 음성 재구성

**핵심 통찰:** KWS 방법론은 단일 최적 구조가 아니라, **자원 제약**, **데이터 양**, **일반화 요구**, **처리 요구**에 따라 적합한 접근법을 선택하는 것이 핵심입니다. 초기에는 CNN 기반 효율화 → Few-shot/Zero-shot 적응 → 교차 모달/오픈보캐블러리 → 에너지 효율 (논리/SNN) 으로 진화하는 흐름이 관찰됩니다.

### 3. 성능 패턴 및 경향 분석

#### 3.1 공통적으로 나타나는 성능 개선 패턴

**1) 시간 컨볼루션 (Temporal Convolution) 이 효율성을 제공한다**

- TC-ResNet8 이 ResNet15 대비 **385 배 빠른 속도** (1.1ms vs 419ms)
- TC-ResNet14-1.5: **96.6%** 정확도 (기존 2D CNN 대비 연산량 획기적 감소)
- 1D 컨볼루션이 MFCC 특징 추출에서 2D 컨볼루션 대비 더 효과적

**2) 사전 훈련 임베딩은 모든 데이터 제한 시나리오에서 우위**

- 합성 음성 3,220 개만으로 **92.6%** 정확도 (실제 데이터 4,000 개 이상 모델 대비 동등)
- 다국어 임베딩 고정 후 5-샷 미세 조정이 **87.4%** 스트리밍 정확도 달성
- KeySEM 사전 훈련 모델이 저데이터 환경에서 기존 방법 대비 7~61%p 향상

**3) 데이터 증강은 필수적**

- SpecAugment: 과적합을 일반화로 전환, test-clean 에서 **2.8%** WER 달성
- 속도/볼륨 증강+대조 학습: 레이블링 데이터 부담 감소, Dev/Eval 에서 **88.5%** 최고 정확도
- 커리큘럼 학습: 저 SNR 환경에서 **3~5.5%** 정확도 추가 향상

#### 3.2 특정 조건에서만 성능이 향상되는 경우

**1) 노이즈 환경**

- ConvMixer: -10dB SNR 에서 **71.88%** 정확도 (AST-Tiny, KWT-3와 동등)
- MLP 믹서 제거 시 저 SNR 환경에서 **약 7%** 정확도 감소
- 커리큘럼 학습: AST-Tiny 에 **3%**, ConvMixer 에 **5.5%** 개선 (저 SNR 에서 더 효과적)

**2) 혼합 음성 환경**

- MT(Mix-Training) 전략이 Clean 대비 **뚜렷한 성능 우위**
- HuBert+MT 조합이 Clean/Mixed 모두에서 강력
- Wav2Vec2.0 은 2-Mix 에서 실패, HuBert는 견고

**3) 오픈-보캐블러리 (사용자 정의 키워드)**

- CMCD: 다른 방법론 대비 **짧은 키워드에서 성능 저하 적음**
- Softtriple + NMH-A: 기준선 대비 **10 배 이상 FRR 감소**
- 텍스트 기반 등록으로 음성 일관성 문제 해결

#### 3.3 논문 간 상충되는 결과

**1) 모델 크기 증가 vs 효율성**

- KWT-3: 5M+ 파라미터로 **97.74%** 정확도 (V2-35)
- ConvMixer: 119K 파라미터로 **98.2%** (SOTA)
- GE2E Conformer: 419K 파라미터로 제로샷 성능 **ASR 대비 23.6%p 개선**
- AdaKWS: 166M 파라미터로 ASR 기반 방법 대비 우수
- **패턴**: 작은 모델이 효율적이지만, 특정 시나리오에서는 큰 모델이 필요

**2) RNN vs CNN/Transformer**

- MHAtt-RNN: **98.0%** 정확도 (Embed+head 대비 10% 오류 감소)
- KWT(Transformer): MHAtt-RNN 상회 **98.56%** (V2-12), **97.74%** (V2-35)
- ConvMixer: 트랜스포머 대비 효율적 (119K vs 5.8M 파라미터)
- **패턴**: 트랜스포머가 성능은 뛰어나지만, RNN 은 스트리밍 지연 시간에서 더 효율적

**3) 학습 전략**

- Few-shot: Prototypical Networks(TD-ResNet7) **94%** 정확도
- Fine-tuning: KeySEM fix 가 86.5%, FT 가 24.3% 급감
- Dark Experience: 버퍼 1500 으로 Joint Training(95.70%) 에 근접
- **패턴**: 메트릭 학습이 온디바이스 개인화에서 우수, Fine-tuning 과는 상충

### 4. 추가 실험 및 검증 패턴

#### 4.1 ablation study

**1) 손실 함수 비교**

- Softtriple Loss: 작은 모델 FRR **8.5%** 이상/큰 모델 **5%** 감소
- NL2+MH-A 정규화: 추가 **2%** 감소
- Triplet vs Cross-Entropy: +25~38% 정확도, +16~57% F1

**2) 모듈 제거 실험**

- MLP 믹서 제거: 저 SNR 환경에서 **약 7%** 정확도 감소
- GCN: CENet-6 에서 **1.1%p**, 모든 단계 삽입 **1.3%p** 향상
- 오디오-오디오 판별 구성 요소 제거: 성능 **크게 저하** (중요성 입증)
- 잔차 연결 제거: 10K 모델에서 특정 크기 이하에서는 이점 감소

**3) 하이퍼파라미터 분석**

- $b=0.33$: 교차 엔트로피 손실과 유사한 성능
- $\alpha=0.5, \beta=1.0$: Dark Experience 최적 하이퍼파라미터
- 패치 크기 **40x1** (시간 도메인): 최적 성능

#### 4.2 민감도 분석

**1) 데이터 양 변화**

- 30K 에포크: 가장 빠른 수렴 (CAB-KWS)
- 25% 훈련 데이터에서 사전 훈련 모델이 100% 새로 초기화 모델과 동등
- 2048 샘플에서 TinyOL/v2 안정화, 배치 기반은 더 많은 데이터 요구

**2) 노이즈 조건 (SNR)**

- -10dB SNR: MatchboxNet-6x2x64 **71.02%** 더 우수
- 0dB→-5dB→-10dB: Gradual 성능 저하
- 커리큘럼 학습이 저 SNR 에서 더 효과적 (**3~5.5%** 추가 향상)

**3) 데이터셋 비율 변화**

- 속도/볼륨 증강: vo+sp 사전 훈련 시 추가 성능 상승
- LibriSpeech-100 사전 훈련: **88.5%** 최고 정확도 (Dev/Eval)

#### 4.3 조건 변화 실험

**1) 키워드 수 증가**

- 키워드 수 증가: 과적합 심화 (최대 4% 증가)
- 유사한 키워드 추가: 정확도 급감 ('Yes','No','Stop' + 'Go' 시 92.6%→82.6%)

**2) 윈도우 스텝 조절**

- 0.03→0.10 초 확대: **90.5%** 정확도 유지 (TM)

**3) 모델 크기 확장**

- 블록 깊이/채널 증가: 정확도 향상 (MatchboxNet)
- 100K 파라미터→250K 파라미터: TC-ResNet14-1.5 대비 TENet12-MTConv 파라미터 1/3 수준 유지

### 5. 실험 설계의 한계 및 비교상의 주의점

#### 5.1 비교 조건의 불일치

| 문제               | 예시                    | 설명                                                                  |
| ------------------ | ----------------------- | --------------------------------------------------------------------- |
| **데이터셋 편향**  | GSCD 중심 실험          | 대부분의 SOTA 모델이 GSCD 에서 평가되어 다른 데이터셋에서 일반화 제한 |
| **평가 지표 차이** | FRR vs AUC vs Accuracy  | 각 논문이 다른 지표를 사용하여 직접 비교 어려움                       |
| **노이즈 조건**    | Clean vs Mixed vs -10dB | 일부 모델은 노이즈 조건에서만 우수, Clean 에서 성능 저하              |
| **정량 방법**      | 프록시 vs 실제 측정     | 곱셈 연산 수 예측과 실제 에너지 소비 간 차이 존재                     |

#### 5.2 데이터셋 의존성

- **GSCD 한정성**: 대부분의 연구가 GSCD 에서 수행되어 실제 배포 환경과의 간극
- **LibriSpeech/Hey-Snips**: 오픈-보캐블러리 벤치마크로서 유용하지만 제한된 어휘
- **Common Voice**: 다국어 평가에 유용하지만 데이터 품질 및 편향 문제
- **VoxPopuli**: 다국어 KWS 평가에 사용되지만 언어별 성능 차이 (카탈루냐어 87.63%, 네덜란드어 72.60%)

#### 5.3 일반화 한계

- **제로샷 가정**: AdaKWS 는 VoxPopuli 에서 우수하지만, Fleurs 저자원 언어에서만 일반화 능력 입증
- **도메인 간극**: GSCD 훈련 모델이 VoxPopuli 테스트셋에서 78.07% (자동 추출 키워드 영향)
- **실시간 조건**: 스트리밍 모델은 이상적 50 배 지연 비율에 미치지 못함 (최대 20 배)

#### 5.4 평가 지표의 한계

| 지표          | 문제점                     | 대안                          |
| ------------- | -------------------------- | ----------------------------- |
| **Accuracy**  | 클래스 불균형 무시         | F1, AUC, DET 곡선             |
| **FRR/FA/hr** | 단일 지점 비교만           | DET 곡선 전체 분석            |
| **AUC**       | 모든 임계점 통합           | EER, Micro Recall             |
| **PARA/MACs** | 실제 에너지 소비 반영 안됨 | 실제 전력 측정 (Watts Up Pro) |

### 7. 종합 정리

제공된 실험 결과들을 종합하면, 키워드 스포팅 (KWS) 성능은 **아키텍처 선택**, **데이터 효율성**, **환경 적응력** 세 가지 요소에 의해 결정된다.

**아키텍처 측면**: 시간 컨볼루션 (TC-ResNet, ConvMixer) 이 2D CNN 보다 효율적이며, 트랜스포머는 정확도는 뛰어나지만 파라미터 대비 효율이 낮음. 스파이킹 뉴런 (SNN) 은 에너지 효율성이 탁월하나 실제 전력 측정 시 프록시 지표와 차이가 있음.

**데이터 측면**: 사전 훈련 임베딩 (다국어/대규모 어휘) 이 저데이터 환경에서 결정적. 합성 데이터는 실제 데이터 대체/보강의 현실적 대안이나, 자연-합성 데이터 혼합이 필요. 데이터 증강 (SpecAugment, 속도/볼륨 증강) 이 일반화 성능에 필수.

**환경 측면**: 노이즈와 혼합 음성 환경에서 성능이 분화. 커리큘럼 학습, MLP 믹서, 오디오 판별이 저 SNR/오탐 문제 해결에 효과적. 스트리밍 지연 시간은 모델 아키텍처 (RNN, CRNN) 와 알고리즘 (자동 스트리밍 변환) 에 의해 결정됨.

**결론적으로**: 엣지 디바이스용 KWS는 **ConvMixer** (119K 파라미터, 98.2%) 또는 **DS-ResNet** (15K~72K 파라미터, 95%+) 가 최상이며, 다국어/제로샷 시나리오에서는 **AdaKWS** (166M 파라미터, 94.6% F1) 가 우수. 오픈-보캐블러리/사용자 정의 키워드는 **Softtriple + NMH-A** 어텐션 조합이 FRR 10 배 감소로 가장 효과적.
