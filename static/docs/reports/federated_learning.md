# Federated Learning

## 서론

연합 학습 (Federated Learning) 은 원시 데이터를 클라이언트에 유지하면서 모델 업데이트만 공유하는 분산형 머신러닝 패러다임이다. 17 편의 핵심 논문을 분석하여 작성된 본 보고서에서는 연합 학습 연구의 체계적 이해를 위해 연구 체계 분류, 방법론 분석, 실험 결과 분석이라는 세 가지 관점에서 문헌을 정리한다.

### 1. 연구 배경

연합 학습 연구는 시스템 구조, 문제 정의, 접근 방식이라는 세 가지 기준으로 체계화될 수 있다. 시스템 구조 관점에서는 중앙 서버 기반, 분산형, 수직 학습, 이질성 지원, 프레임워크 플랫폼 등 물리적 및 논리적 구성에 따른 분류가 이루어졌다. 이는 FL 시스템의 아키텍처 설계 차원에서 다양한 실험 환경과 확장성을 고려한 연구들이 축적되었음을 보여준다.

### 2. 문제의식 및 분석 필요성

연속적 논문들은 프라이버시 보호, 비-IID 데이터 분포 대응, 공격/방어, 클래스 불균형, 전이 학습, 의료 전용 등 다양한 문제 정의에 초점을 맞추고 있다. 이러한 문제들 역시 각각의 해결을 위해 손실 함수 재구성, Fisher 정보 활용, 지식 증류 등 구체적인 방법론적 전략을 채택했다. 그러나 개별 연구들을 단순히 나열할 수 없으며, FL 연구가 아키텍처 설계, 문제 대응, 도구 제공이라는 3 층 구조를 가지며 상호 보완적으로 연계되어 실제 배포 가능한 시스템을 구성하는 양상에서 체계적 분석이 필요하다.

### 3. 보고서의 분석 관점

본 보고서는 (1) 연구 체계 분류를 통해 FL 연구의 주요 범주를 제시하고, (2) 방법론 분석을 통해 6 개의 계열 (압축 효율형, 보안 강화형, 데이터 이질성 완화형, 시스템 최적화형, 공정성/방어형, 전이 학습 통합형) 로 접근 방식을 구분하며, (3) 실험 결과 분석을 통해 데이터셋·환경별 성능과 평가 지표의 경향을 정리한다. 각 관점은 독립적이면서도 서로 연결된 분석을 제공한다.

### 4. 보고서 구성

본문은 총 3 장으로 구성되어 있다. 1 장 연구체계 분류는 중앙 서버 기반, 분산형, 수직 학습 등 시스템 구조와 문제 정의에 따른 체계적 분류를 제시한다. 2 장 방법론 분석은 공통 문제 설정, 계열별 분류, 핵심 설계 패턴, 문제 접근 방식 차이, 기술 진화 경로를 다룬다. 3 장 실험결과 분석은 평가 구조, 데이터셋별 성능 비교, 성능 개선 패턴, 실험 설계의 한계, 결과 해석 경향을 종합한다.

논문들을 분석하여 1 장 "연구체계 분류"를 작성하겠습니다. 먼저 논문들의 공통 주제, 문제 정의, 적용 대상, 접근 방식을 검토한 후 체계적 분류를 설계하겠습니다.

## 1 장. 연구체계 분류

### 1. 연구 분류 체계 수립 기준

본 연구는 제공된 17 편의 연합 학습 (Federated Learning) 논문들을 다음과 같은 기준과 원칙으로 분류하였다.

**분류 기준:**

1. **시스템 구조 관점**: 중앙 서버 아키텍처, 분산형 아키텍처, 수직 학습 구조 등 시스템의 물리적/논리적 구성 방식
2. **문제 정의 관점**: 프라이버시 보호, 비-IID 대응, 통신 효율성, 데이터 이질성, 공격 방어 등 해결하려는 핵심 문제
3. **접근 방식 관점**: 손실 함수 개선, 알고리즘 수정, 모델 분해, 지식 증류 등 방법론적 접근
4. **서베이/인프라 관점**: 기존 연구 종합, 프레임워크 구축, 벤치마크 제공 등 도구적 연구

**분류 원칙:**

- 각 논문은 가장 대표적인 한 범주에만 배치되며, 여러 범주에 해당하더라도 1 회만 분류
- 분류 근거는 반드시 해당 논문의 요약문에서 확인 가능한 내용만을 바탕으로 함
- 연도는 논문 제목에 명시된 대로 표기 (연도 미상인 경우 제외됨)

### 2. 연구 분류 체계

#### 2.1 시스템 구조/아키텍처 기반 분류

**2.1.1 중앙 서버 기반 연합 학습 (Centralized FL)**
중앙 서버에서 글로벌 모델 업데이트를 집계하고 배포하는 전통적 FL 아키텍처

| 분류                              | 논문명                                                                                       | 분류 근거                                                                                  |
| --------------------------------- | -------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| 중앙 서버 기반 FL > 전통적 FL     | Think Locally, Act Globally: Federated Learning with Local and Global Representations (2020) | 지역 모델과 전역 모델을 분리하여 $\alpha$-보간 앙상블로 학습하는 중앙 서버 기반 프레임워크 |
| 중앙 서버 기반 FL > 비동기 최적화 | VAFL: a Method of Vertical Asynchronous Federated Learning (2020)                            | VFL 환경에서 비동기 SGD 와 지연 모델을 도입한 중앙 서버 기반 비동기 FL                     |

**2.1.2 분산형/중앙 서버 자유 FL (Decentralized FL)**
중앙 서버 없이 노드 간 직접 통신으로 학습하는 아키텍처

| 분류                             | 논문명                                                                                | 분류 근거                                                                 |
| -------------------------------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| 분산형 FL > P2P 가십 프로토콜    | Decentralized Federated Learning: A Segmented Gossip Approach (2019)                  | 모델 세그멘테이션과 분산형 가ossip 프로토콜로 중앙 서버 없는 FL 구현      |
| 분산형 FL > 단방향 신뢰 네트워크 | Central Server Free Federated Learning over Single-sided Trust Social Networks (2019) | Online Push-Sum 알고리즘으로 단방향 신뢰 네트워크에서 중앙 서버 없이 학습 |

**2.1.3 수직 연합 학습 (Vertical FL)**
클라이언트가 각기 다른 특성을 공유하며 동일한 사용자 집합을 처리하는 VFL 환경

| 분류                    | 논문명                                                                                                          | 분류 근거                                                                    |
| ----------------------- | --------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| 수직 FL > 비동기 최적화 | VAFL: a Method of Vertical Asynchronous Federated Learning (2020)                                               | VFL 환경에서 교란된 로컬 임베딩과 비동기 SGD 로 클라이언트 간 조율 없이 학습 |
| 수직 FL > 조정자 제거   | Parallel Distributed Logistic Regression for Vertical Federated Learning without Third-Party Coordinator (2019) | AHE 암호화와 분산 통신으로 제3자 조정자 없는 VFL 시스템 구현                 |

**2.1.4 이질성/이종성 지원 FL (Heterogeneous FL)**
모델 아키텍처, 하드웨어, 네트워크 속도 이질성을 지원하는 FL

| 분류                                  | 논문명                                                                         | 분류 근거                                                                       |
| ------------------------------------- | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------- |
| 이질성 지원 FL > 모델 아키텍처 독립성 | FedMD: Heterogenous Federated Learning via Model Distillation (2019)           | FedMD 프레임워크로 각 참여자의 고유 CNN 아키텍처와 데이터 공유 없이 지식 전달   |
| 이질성 지원 FL > 시스템 확장성        | Flower: A Friendly Federated Learning Research Framework (2020)                | 이질적 엣지 디바이스 환경과 최대 1,500 만 클라이언트 규모의 대규모 FL 실험 지원 |
| 이질성 지원 FL > 벤치마크 플랫폼      | FedCV: A Federated Learning Framework for Diverse Computer Vision Tasks (2021) | 이미지 분류/분할/탐지 등 다양한 CV 태스크를 위한 통합 벤치마크 플랫폼 제공      |

**2.1.5 프레임워크/인프라 플랫폼 FL**
FL 연구 및 실험을 위한 프레임워크, SDK, 벤치마크 제공에 중점

| 분류                                   | 논문명                                                                         | 분류 근거                                                                  |
| -------------------------------------- | ------------------------------------------------------------------------------ | -------------------------------------------------------------------------- |
| 프레임워크/인프라 FL > SDK/프레임워크  | Flower: A Friendly Federated Learning Research Framework (2020)                | 최대 1,500 만 클라이언트 실험을 지원하는 확장 가능한 FL 연구용 프레임워크  |
| 프레임워크/인프라 FL > 벤치마크 플랫폼 | FedCV: A Federated Learning Framework for Diverse Computer Vision Tasks (2021) | 이미지 분류, 분할, 탐지 등 다양한 CV 태스크 비교 평가를 위한 통합 벤치마크 |
| 프레임워크/인프라 FL > 시뮬레이션 도구 | NVIDIA FLARE: Federated Learning from Simulation to Real-World (2022)          | FL 시뮬레이터와 컴포넌트 기반 아키텍처로 연구-프로덕션 간 간극 해소        |

#### 2.2 문제 정의 기반 분류

**2.2.1 비-IID 데이터 분포 대응 FL**
클라이언트 간 데이터 분포 이질성 (Non-IID) 문제를 해결하는 FL 기법

| 분류                                   | 논문명                                                                         | 분류 근거                                                                            |
| -------------------------------------- | ------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------ |
| 비-IID 대응 FL > Fisher 정보 기반 적응 | Overcoming Forgetting in Federated Learning on Non-IID Data (2019)             | Fisher 정보 행렬 대각 성분을 활용한 파라미터 강성으로 비-IID 환경에서 모델 이탈 방지 |
| 비-IID 대응 FL > 데이터 합성           | Federated Learning with GAN-based Data Synthesis for Non-IID Clients (2022)    | 차등 프라이버시 GAN 으로 합성 데이터 생성하여 non-IID 문제 해결                      |
| 비-IID 대응 FL > 클라이언트 샘플링     | Fed-Focal Loss for imbalanced data classification in Federated Learning (2020) | 검증 손실 기반 클라이언트 성능 튜닝으로 어려울 클라이언트와 샘플에 집중              |

**2.2.2 프라이버시 보호 FL**
차등 프라이버시, 동형 암호화 등 프라이버시 보존 기술 적용

| 분류                                 | 논문명                                                                                              | 분류 근거                                                               |
| ------------------------------------ | --------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------- |
| 프라이버시 보호 FL > 차등 프라이버시 | Private Federated Learning with Domain Adaptation (2019)                                            | 공용 DP-SGD 모델과 사설 도메인 모델을 MoE 가중 평균으로 결합            |
| 프라이버시 보호 FL > 차등 프라이버시 | FedDPGAN 기반 GAN 합성 (Federated Learning with GAN-based Data Synthesis for Non-IID Clients, 2022) | 클라이언트 측 WGAN-GP 로 차등 프라이버시를 보호하는 합성 데이터 생성    |
| 프라이버시 보호 FL > 동형 암호화     | FedHealth: A Federated Transfer Learning Framework for Wearable Healthcare (2019)                   | 동형 암호화된 파라미터 공유로 프라이버시 보존하는 웨어러블 헬스케어 FL  |
| 프라이버시 보호 FL > 보안 집계       | Secure and Efficient Federated Transfer Learning (2019)                                             | MPC 기반 보안 프로토콜과 테일러 근사로 악의적 참여자에 대한 능동적 보안 |

**2.2.3 공격/방어 관련 FL**
중독 공격, 백도어 공격, 시빌 탐지 등 보안 공격 대응

| 분류                               | 논문명                                                                                                                 | 분류 근거                                                                    |
| ---------------------------------- | ---------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------- |
| 공격/방어 FL > 시빌 기반 중독 공격 | Mitigating Sybils in Federated Learning Poisoning (2018)                                                               | 클라이언트 기여 코사인 유사성 계산을 통한 시빌 기반 중독 공격 탐지 및 방어   |
| 공격/방어 FL > 평판 기반 방어      | A Reputation Mechanism Is All You Need: Collaborative Fairness and Adversarial Robustness in Federated Learning (2020) | 그래디언트 코사인 유사도 기반 평판 메커니즘으로 비기여자 제거 및 차등 보상   |
| 공격/방어 FL > 탐지 기반 방어      | Abnormal Client Behavior Detection in Federated Learning (2019)                                                        | 오토인코더 기반 비정상 클라이언트 탐지로 표적화된 방어 메커니즘              |
| 공격/방어 FL > 백도어 공격         | Can You Really Backdoor Federated Learning? (2019)                                                                     | 비악의적 클라이언트의 표적 데이터 보유 시나리오에서 FL 백도어 공격/방어 연구 |

**2.2.4 클래스 불균형/데이터 불균형 FL**
클래스 불균형 (class imbalance) 및 데이터 불균형 문제 해결

**2.2.5 전이 학습/준지도 학습 FL**
전이 학습, 준지도 학습, 라벨 부족 상황 대응

의료/헬스케어 데이터 및 응용 분야에 특화된 FL 연구

**2.2.7 이론적/기타 FL**
이론적 분석, 서베이, 전망 등 다양한 연구 유형

| 분류                         | 논문명                                                                                       | 분류 근거                                                                 |

### 3. 종합 정리

분석 대상 17 편의 논문은 **시스템 구조/아키텍처**,**문제 정의**,**접근 방식**,**서베이/인프라** 4 가지 관점으로 체계적으로 분류되었습니다.**시스템 구조 관점**에서는 중앙 서버 기반, 분산형, 수직 학습, 이질성 지원, 프레임워크 플랫폼 등 FL 시스템의 물리적/논리적 구성을 기준으로 분류하였으며,**문제 정의 관점**에서는 비-IID 대응, 프라이버시 보호, 공격/방어, 클래스 불균형, 전이 학습, 의료 전용 등 해결하려는 핵심 문제 유형을 기준으로 분류하였습니다.**접근 방식**은 각 논문이 구체적으로 어떤 방법론적 전략 (손실 함수 재구성, Fisher 정보 활용, 지식 증류 등) 을 채택하는지로 구분 가능하나, 연구 범주의 중복을 피하기 위해 주요 범주로 통합하여 분류하였습니다. 또한**서베이/인프라 관점**에서는 기존 연구 종합과 벤치마크 플랫폼 구축 등 도구적 연구 특성을 가진 논문을 별도로 분류하였습니다. 전체 연구 지형을 종합하면, 연합 학습 연구는**아키텍처 설계** (중앙/분산/수직),**문제 대응** (비-IID/프라이버시/보안/불균형),**도구 제공** (프레임워크/벤치마크) 이라는 3 층 구조를 가지며, 각 층은 서로 독립적이지 않고 상호 보완적으로 연계되어 실제 배포 가능한 FL 시스템을 구성하는 데 기여합니다.

### 3. 핵심 설계 패턴 분석

#### 3.1 모듈화된 서버-클라이언트 분리 구조

모든 논문들이 공통적으로 적용하는 패턴:

```text
┌─────────────────────────────────────────────┐
│          Central Server (서버)              │
│  ├── 전역 모델 관리 (broadcast, aggregate)  │
│  ├── 클라이언트 선택/샘플링                 │
│  └── 집계 전략 (FedAvg, 가중 평균, 등)      │
└─────────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────┐
│           Client (클라이언트/장치)          │
│  ├── 로컬 데이터셋 (비-IID)                 │
│  ├── 로컬 학습 (SGD, 최적화)                │
│  └── 모델 업데이트 (가중치, 그래디언트)     │
└─────────────────────────────────────────────┘
```

**적용 사례:**

### 6. 종합 정리

제공된 논문들을 분석한 결과, 연합 학습 방법론은 다음 4 가지 축으로 구조화된다:

```text
┌─────────────────────────────────────────────────────────────────────┐
│                   방법론 지형도 (Methodology Landscape)             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   ┌─────────────┐  ┌──────────────┐  ┌────────────────────────┐     │
│   │  압축 효율형│  │  보안 강화형 │  │  데이터 이질성 완화형  │     │
│   │ (통신 최적) │  │ (프라이버시) │  │ (Non-IID 대응)         │     │
│   └─────────────┘  └──────────────┘  └────────────────────────┘     │
│                            │                                        │
│                            ↓                                        │
│   ┌──────────────────┐  ┌────────────────┐  ┌───────────────────┐   │
│   │  시스템 최적화형 │  │  공정성/방어형 │  │  전이 학습 통합형 │   │
│   │ (확장성)         │  │ (강건성)       │  │ (개인화)          │   │
│   └──────────────────┘  └────────────────┘  └───────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**핵심 발견:**

1. **모든 FL 방법론은 기본 FedAvg 파이프라인 위에 확장된 변형**으로, 통신 효율화, 보안 강화, 문제 특성화 3 가지 축에서 변형됨

2. **보안과 효율성의 Trade-off**는 모든 방법론이 공유하는 핵심 문제이며, HE/DP/압축이 각기 다른 Trade-off 를 제시

3. **Non-IID 문제**는 초기에는 FedProx 등 보정항으로 접근했으나, 최근에는 GAN 합성, 지식 증류 등 보다 적극적 데이터 전략으로 해결되고 있음

4. **시스템 프레임워크** (Flower, FLARE) 는 알고리즘 연구와 실용적 배포 간의 간극을 연결하는 역할을 수행

5. **방어 메커니즘**은 초기에는 모델 크리핑에 의존했으나, 평판 시스템과 오토인코더 기반 탐지로 진화

6. **전이 학습**은 초기 FL 모델 초기화 문제로 제한적이었으나, FedHealth 등을 통해 도메인 적응 도구로 확장됨

**결론적으로**, 연합 학습 방법론은 "통신 효율성 ↔ 보안 ↔ 정확도"라는 3 원 문제 해결을 위해 다양한 기술적 변형을 통해 진화하고 있으며, 각 계열은 상호 보완적으로 적용될 수 있는 구조를 지니고 있다.

### 3. 성능 패턴 및 경향 분석

#### 공통적으로 나타나는 성능 개선 패턴

| 패턴                            | 관찰 조건                        | 대표적 논문                                                    |

### 7. 종합 정리

#### 전체 실험 결과가 시사하는 바

Federated Learning 연구의 실험 결과는 데이터셋·환경·알고리즘 조합에 따라 명확한 성능 격차를 드러낸다. non-IID 환경에서는 FedAvg 대비 FedMeta (UGA), FedCurv, FedMD, LG-FEDAVG 등 기존 알고리즘의 한계를 보완하는 변형들이 필수적이며, 이를 위해서는 UGA (기울기 편향 제거), Fisher 정보 기반 페널티, 지식 증류, 도메인 적응 등 구조적 개입이 요구된다. 프라이버시 보호 측면에서는 동형 암호화 대비 MPC 기반 (ABY/SPDZ) 이 20-30 배 효율성을 달성하면서도 정확도를 유지하며, 노름 클리핑과 소량 가우시안 노이즈만으로 백도어 공격을 방어할 수 있음이 확인되었다. 확장성 측면에서는 VCE 기반 클라이언트 로딩과 $\tau$ cutoff time 전략이 1,500 만 클라이언트 규모에서도 효과적이며, 분산 가십 통신이 노드 대역폭 포화 문제를 해결한다. 반면, 데이터셋 의존성이 강해 클래스 불균형, 스트리밍, 의료 영상 등 도메인에 따라 최적 파라미터가 상이하고, 시뮬레이션 (Dirichlet 파티셔닝) 결과만으로 실제 환경 일반화가 제한된다는 점을 주의해야 한다.
