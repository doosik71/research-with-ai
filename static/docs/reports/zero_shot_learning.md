# Zero-Shot Learning

## 서론

### 1. 연구 배경

제로샷 학습 (Zero-Shot Learning) 은 훈련 시 이전에는 본적 없는 클래스에 대해 일반화 예측을 수행하는 인식 태스크로, 시각적 특징과 의미론적 정보를 공통 공간으로 매핑하거나 생성 모델을 활용한 특징 합성 등을 통해 접근해왔다. 제시된 22 편의 관련 논문은 접근 방식, 방법론적 초점, 시스템 아키텍처 설계 관점을 기준으로 8 대 범주로 분류되는데, 이는 생성 모델 기반 특징 합성, 시맨틱 임베딩 정규화 및 제약, 속성 기반 지식 전이, 분류기 합성 및 앙상블, 시맨틱 분할 및 세그멘테이션, 비전 - 언어 모델 활용, 벤치마크 및 서베이, 교차 모달 및 자기/반지도 학습으로 구분된다. 제로샷 학습 연구는 기본적으로 시맨틱 임베딩을 활용한 지식 전이를 전제로 하며, 시맨틱 정보 활용 방식에 따라 다양한 하위 패러다임으로 분화되고 있다.

### 2. 문제의식 및 분석 필요성

다양한 접근법과 설계 패턴이 존재하는 제로샷 학습 분야에서 방법론적 선택 기준을 마련하기 위해서는 체계적인 분류와 비교 분석이 필요하다. 특히 ZSL 과 GZSL 설정, 본/미본 클래스 편향 문제, 계산 비용 대 성능 트레이드오프, 데이터셋 의존성 등 핵심 이슈를 명확히 정리해야 실제 적용 시나리오에 맞는 방법론 선택이 가능하다. 또한 기존 연구들이 사용한 평가 프로토콜, 지표, 데이터셋 간의 불일치로 인한 결과 해석의 한계를 인지하고, 이를 고려한 비교 분석이 요구된다.

### 3. 보고서의 분석 관점

본 보고서는 제로샷 학습 연구 문헌을 세 가지 축으로 정돈한다: **연구체계 분류**를 통해 방법론적 패러다임과 설계 패턴을 체계적으로 정리하고,**방법론 구조 분석**을 통해 공통 문제 설정, 계열 분류, 핵심 설계 패턴, 손실 함수, 모델 아키텍처 등을 비교 분석하며,**실험결과 분석**을 통해 주요 벤치마크에서의 성능 패턴, 검증 실험 유형, 결과 해석 경향 등을 검토한다.

### 4. 보고서 구성

보고서는 다음과 같이 세 장으로 구성된다:

**1장. 연구체계 분류**: 접근 방식과 설계 관점을 기준으로 22 편의 논문을 8 대 범주로 분류하고, 연구 분류 체계의 수립 기준과 범주별 상세 내용을 정리한다.

**2장. 방법론 구조 분석**: 공통 문제 설정과 접근 구조를 정의한 후, 주요 방법론 계열 (의미론적 매핑 기반, 생성 모델 기반, 반응형 특징 정제 등) 을 상세 비교하며, 핵심 설계 패턴과 손실 함수, 모델 아키텍처 패턴을 분석한다.

**3장. 실험결과 분석**: 평가 구조와 공통 실험 설정을 정리한 후, 주요 벤치마크 데이터셋에서의 실험 결과 정렬과 성능 패턴, 검증 실험 유형을 분석하고, 실험 설계의 한계와 결과 해석 경향을 종합한다.

## 1장. 연구체계 분류

### 1. 연구 분류 체계 수립 기준

제시된 22 편의 제로샷 학습 관련 논문은 접근 방식, 방법론적 초점, 시스템 아키텍처 설계 관점을 중심으로 다음과 같은 8 대 범주로 분류되었다. 각 범주는 제로샷 학습 문제를 해결하기 위해 적용된 핵심 기술과 방법론적 패러다임을 기준으로 정의되며, 논문이 여러 범주에 걸쳐도 가장 적합한 1 개 범주에 배치하였다.

### 2. 연구 분류 체계

#### 2.1 생성 모델 기반 특징 합성

GAN, VAE, Flow 모델 등 생성 모델을 활용하여 훈련되지 않은 (unseen) 클래스의 특징을 합성하거나 생성하는 연구들.

| 분류                                 | 논문명                                                                                    | 분류 근거                                                                                         |
| ------------------------------------ | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| 생성적 ZSL > 특징 합성 네트워크      | Latent Embedding Feedback and Discriminative Features for Zero-Shot Classification (2020) | ZSL 파이프라인 전체에 SED 임베딩 피드백과 판별적 특징 변환을 통합하는 VAE-GAN 하이브리드 아키텍처 |
| 생성적 ZSL > 생성적 ZSL              | Invertible Zero-Shot Recognition Flows (2020)                                             | 학습된 - 미학습 편향 해결을 위해 음수 MMD 손실을 도입한 생성형 플로우 모델                        |
| 생성적 ZSL > 컨텍스트 인지 특징 생성 | Context-aware Feature Generation for Zero-shot Semantic Segmentation (2020)               | 픽셀 단위 컨텍스트 정보를 활용한 GAN 적대적 학습 구조로 모드 붕괴 해소                            |
| 생성적 ZSL > 특징 합성 네트워크      | Feature Generating Networks for Zero-Shot Learning (2017)                                 | 미본 클래스의 CNN 특징 분포를 조건부 생성 모델 (f-WGAN/f-CLSWGAN) 로 합성                         |
| 생성적 ZSL > 생성적 GZSL             | Generalized Zero-Shot Learning via Synthesized Examples (2017)                            | 조건부 VAE 기반 합성 예시 생성과 판별기 피드백으로 seen 편향 완화                                 |
| 생성적 ZSL > 생성적 GZSL             | Alleviating Feature Confusion for Generative Zero-shot Learning (2019)                    | 결정 경계 최적화 및 순환 일관성 손실을 통한 GAN 기반 특징 혼동 해결                               |
| 생성적 ZSL > 특징 합성 네트워크      | Zero-Shot Semantic Segmentation (2019)                                                    | GMMN 생성 모델을 통해 'unseen' 클래스 특징 합성 및 분류 가능하게 하는 접근                        |
| 생성적 ZSL > 특징 합성 네트워크      | Generative Multi-Label Zero-Shot Learning (2021)                                          | 단일 이미지 내 다중 레이블 특징을 multi-headed attention 융합 블록으로 합성                       |
| 생성적 ZSL > 특징 합성 네트워크      | Semantic Autoencoder for Zero-Shot Learning (2017)                                        | 재구성 제약 기반 선형 대칭 오토인코더로 도메인 시프트 완화                                        |

#### 2.2 시맨틱 임베딩 정규화 및 제약

시맨틱 임베딩 공간에서 정규화, 일관성 제약, 거리 기반 판별 등을 적용한 연구들.

| 분류                                    | 논문명                                                                                            | 분류 근거                                                                       |

| 시맨틱 정규화 > 시맨틱 임베딩 볼록 결합 | Zero-Shot Learning by Convex Combination of Semantic Embeddings (2013)                            | N-way 분류기 예측 확률을 클래스 임베딩 벡터의 볼록 결합으로 변환                |
| 시맨틱 정규화 > 시맨틱 임베딩 정규화    | Label-Embedding for Image Classification (2015)                                                   | WSABIE 랭킹 최적화를 통한 이미지 - 레이블 호환성 직접 최적화                    |
| 시맨틱 정규화 > 시맨틱 임베딩 정규화    | Zero-Shot Learning for Semantic Utterance Classification (2014)                                   | 조건부 엔트로피 최소화를 통한 비지도 방식으로 임베딩 판별력 향상                |
| 시맨틱 정규화 > 시맨틱 공간 정렬        | Alternative Semantic Representations for Zero-Shot Human Action Recognition (2017)                | 웹 기반 텍스트/이미지를 사이드 정보로 활용한 이미지 기반 Fisher Feature Vector  |
| 시맨틱 정규화 > 시맨틱 임베딩 정규화    | A Unified approach for Conventional Zero-shot, Generalized Zero-shot and Few-shot Learning (2017) | 시맨틱 공간에서 CAPD 기반 거리 측정과 시맨틱 변환을 통한 통합 연결              |

### 3. 종합 정리

제시된 22 편의 논문은 생성 모델 기반 특징 합성 (9 편), 시맨틱 임베딩 정규화 및 제약 (7 편), 속성 기반 지식 전이 (4 편), 분류기 합성 및 앙상블 (3 편), 시맨틱 분할 및 세그멘테이션 (3 편), 비전 - 언어 모델 활용 (2 편), 벤치마크 및 서베이 (5 편), 교차 모달 및 자기 학습/반지도 학습 (3 편) 등 8 개 주요 범주로 분류될 수 있다. 제로샷 학습 연구는 기본적으로 시맨틱 임베딩을 활용한 지식 전이를 전제로 하며, 이 시맨틱 정보의 활용 방식에 따라 생성 모델 (특징 합성), 정규화/제약, 속성 매개, 분류기 합성 등 다양한 하위 패러다임으로 분화된다. 최근에는 고정된 비전 - 언어 모델 (CLIP 등) 을 활용한 프롬프트 기반 접근과 자기 학습/반지도 전략이 부각되고 있으며, 시맨틱 분할 및 비디오 인식과 같은 밀집 예측 태스크, 3D/신호 인식과 같은 특수 도메인으로의 적용 연구가 활발하다.

### 6. 종합 정리

Zero-Shot Learning 의 방법론은 **3 가지 축**으로 구분될 수 있다:

```text
┌─────────────────────────────────────────────────────────┐
│                     방법론 지형도                       │
├─────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────┐  │
│  │  1. 문제 설정 차원                                │  │
│  │  └─ ZSL (전체 정확도 최적화)                      │  │
│  │  └─ GZSL (학습/미학습 균형)                       │  │
│  │  └─ Few-Shot (소량 데이터 일반화)                 │  │
│  └───────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────┐  │
│  │  2. 접근 방식 차원                                │  │
│  │  └─ 의미론적 매핑 (Mapping-based)                 │  │
│  │  └─ 생성적 합성 (Generative Synthesis)            │  │
│  │  └─ 정규화 기반 (Regularization-based)            │  │
│  └───────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────┐  │
│  │  3. 데이터 활용 차원                              │  │
│  │  └─ 라벨 기반 (Supervised)                        │  │
│  │  └─ 반지도 (Semi-Supervised)                      │  │
│  │  └─ 비지도/생성 (Unsupervised/Generative)         │  │
│  └───────────────────────────────────────────────────┘  │
│  핵심 통찰:                                             │
│  ────────────────────────────────────────────────────── │
│  ┌───────────────────────────────────────────────────┐  │
│  │  제로샷 학습은 "시각 ↔ 의미" 간 지식 전이 문제    │  │
│  │  └─ 매핑: 공간 정렬                               │  │
│  │  └─ 합성: 분포 모델링                             │  │
│  │  └─ 정규화: 편향 제어                             │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

#### 6.1 방법론 지형도 요약

- **방법론 축 1: 지식 전이 전략**
  - 의미론적 매핑 (공간 정렬) vs 생성적 합성 (분포 모델링) vs 정규화 (편향 제어)

- **방법론 축 2: 데이터 의존도**
  - 라벨 기반 (Supervised) → 반지도 (Semi) → 생성/비지도 (Unsupervised)

- **방법론 축 3: 적용 시나리오**
  - 순수 ZSL → GZSL (편향 완화) → Few-Shot (소량 데이터)

이제 2 장. 방법론 분석 문서를 생성합니다.

### 3. 성능 패턴 및 경향 분석

**공통적으로 나타나는 성능 개선 패턴**

**가. 생성적 특징 합성의 효과**

- 여러 논문에서 조건부 GAN/VAE 기반 특징 생성이 GZSL 설정에서 일관된 성능 향상
- f-GAN/VAEGAN 기반 방법: ZSL 정확도 +3.4~8.5%, GZSL H +17~30%p
- 특징 생성 수 증가 (1→100) 시 정확도 개선 경향 (Table 1 보조 결과)
- **패턴**: 생성 모델 → GZSL 설정 최적화 → H 지표 극대화

**나. 손실 함수 조합의 중요성**

- 단일 손실 vs 다중 손실 비교: CENTER loss 제거 시 F1 -6.4% (SR2CNN)
- ablation 연구: 4 가지 손실 항 모두 기여 (CaGNet), RCE 손실 + 인컨텍스트 학습 (ProGen)
- **패턴**: 분류 손실 + 생성 손실 + 정규화 손실 조합 = 최적 성능

**다. 데이터셋 의존성**

- CUB: 작은 클래스 (200 개) → 생성/피드백 모듈 효과 큼
- ImageNet-1K/20K: 대규모 → 모든 방법론 성능 저하
- SUN: 717 클래스/50 속성 → 가장 어려운 설정 (PrEN 63.3% 달성)
- **패턴**: 클래스 수↑/속성 수↑ → 성능 저하 → 데이터셋 편향 존재

**라. 본/미본 편향**

- GZSL 에서 본 클래스 편향 (seen bias) 이 항상 존재
- Calibrated Stacking 등 보정 기법 +10~30% AUSUC 개선
- **패턴**: 생성된 예시 → Seen 편향 완화 → H ↑

**마. 시각적 속성 우위**

- 시각적 속성 > 단어 벡터 > 계층 임베딩 (Score, ZS3Net 등)
- VGSE: 시각적 유사성 + 의미론적 관련성 동시 고려 = 성능↑
- **패턴**: 시각 정보 풍부함 = 지식 전이 용이

**특정 조건에서만 성능이 향상되는 경우**

| 조건              | 최적 방법                 | 성능 향상             |
| ----------------- | ------------------------- | --------------------- |
| 작은 클래스 (CUB) | 생성 모델 (VAE/GAN)       | GZSL H +17~30%p       |
| 데이터 부족       | 자기 훈련 (Self-training) | +9.8~13.4%            |
| 다중 레이블       | CLF 융합                  | ZSL +8.9%, GZSL +5.3% |
| 본/미본 불균형    | Calibrated Stacking       | AUSUC +10~30%         |

**논문 간 상충되는 결과**

- ConSE (2013): ImageNet 제로샷 설정에서 DeViSE 를 능가 (hit@1: 9.4% vs 6.5%)
- 일반 학습 태스크: DeViSE 약간 우위, 제로샷: ConSE 우위
- **해석**: 작업 유형에 최적 방법론 달라

**표현 방식 비교**

| -------------- | ------ | --------------------------------- |

3. **ZSL/GZSL 분기 평가**: conventional + generalized 설정 병렬 실험
4. **정성적 분석**: t-SNE 분포, 시각화, 실패 사례 분석

- **VGSE**: 패치 클러스터링 + 클래스 관계 모듈 실험
- **CLIP**: 13 가지 학습 작업 구성 (작업/클래스/도메인 증분)
- **ProGen**: 인컨텍스트 예제 수 8 개 이하 최적 (8→12 감소)

### 5. 실험 설계의 한계 및 비교상의 주의점

**데이터 유출 문제**

| 논문                      | 유출 방식    | 결과                                              |
| ------------------------- | ------------ | ------------------------------------------------- |
| Zero-Shot Learning (2017) | PS 분할 제안 | AWA 정확도 급감 (기존 분할 vs PS 분할)            |
| ZSL (다양)                | SS 분할 일반 | 테스트 데이터 이미지가 훈련 포함 시 성능 과대평가 |

**평가 지표의 불일치**

- **ZSL**: Top-1 정확도 (본 클래스 편향 허용)
- **GZSL**: H 지표 (본/미본 균형)
- **주의점**: H 지수가 낮을 경우 (예: SUN 26.3%) seen 편향 극심

**데이터셋 의존성**

- CUB: 생성/피드백 모듈 효과 큼 (작은 클래스)
- ImageNet-20K: 모든 방법론 성능 저하 (20K 클래스)
- SUN: 가장 어려운 설정 (717 클래스/50 속성)

**보조 정보 품질 의존**

- 속성 주석 품질 → 지식 전이 성능 직접 영향
- 잘못/불완전한 속성 → 분류기 생성 방해
- Word2Vec/GloVe/FastText 모두 일반화

**계산 비용 및 효율성 논의 부재**

- 다중 데이터셋/모듈 실험 시 계산 리소스 요구
- CLIP 기준선: 미세 조정 없이도 경쟁력 → 효율성 우위
- 2D/3D 분할: 계산 비용과 적응 기간 비교 명시적 부재

**표준화 부재**

- 15 개 이상 데이터셋 (UCF101/HMDB51/Kinetics 등) 사용
- 분할 비율 (90/10, 80/20, 50/50) 차이로 비교 어려움
- Random vs Attributes vs Word2Vec 설정 차이

- 실제 성능 수치보다는 방법론적 혁신 (의미론적 일관성, 지식 전이 등) 강조

**나. 한계는 최소화**

- "추가 연구 필요" 표현 자주 사용
- 실패 사례 (모션 블러, 거울 반사 등) 언급 최소화
- 한계는 "향후 연구 방향"으로 전환

**다. 결과 일반화**

- 단일 데이터셋 (CUB) 결과 → "모든 벤치마크에서" 일반화
- 제한된 실험 조건 → "대규모 데이터셋에서도" 확장 주장

**라. 용어 사용 경향**

| 표현               | 빈도      | 사용 맥락        |
| ------------------ | --------- | ---------------- |
| "SOTA 달성"        | 매우 높음 | 성능 주장        |
| "일반화 능력 향상" | 높음      | GZSL/H 지표      |
| "의미론적 일관성"  | 높음      | 손실 함수/피드백 |
| "지식 전이"        | 높음      | 속성/임베딩      |
| "SOTA 대비 개선"   | 중간      | 비교 결과        |

**결과 vs 해석의 괴리**

- **관찰**: 생성 모델 ZSL/GZSL 모두 향상
- **해석**: "생성적 특징 합성의 일반화 능력 입증"

- **관찰**: CUB 67.2%, SUN 37.6% (ABP)
- **해석**: "지역성 강화가 지식 전이 결정적"

- **관찰**: ImageNet 20K 에서 모든 방법론 성능 저하
- **해석**: "대규모 클래스에서의 도전 과제" (부정적 해석 최소화)

### 7. 종합 정리

전체 실험 결과는 제로샷 학습의 핵심 과제는 "지식 전이"와 "일반화"이며, 이를 위해 **생성적 특징 합성**과**다중 손실 함수 조합**이 필수적임. 생성 모델은 GZSL 설정에서 seen 편향을 완화하며 H 지수를 극대화하지만, 대규모 데이터셋에서는 비선형 모델 대비 상대적으로 낮은 성능. 생성적 방법의 성능은 데이터셋 규모와 클래스 수에 민감하여 CUB (작은 클래스) 에서는 우위이지만 SUN/ImageNet-20K 에서는 일반화 어려움. 손실 함수 조합에서는 분류 손실+생성 손실+정규화 손실이 최적이며, RCE 기반 영향 함수가 노이즈 환경에서 더 정확한 샘플 식별. 실험 설계 측면에서는 평가 프로토콜 (PS 분할) 과 보조 정보 품질이 결과를 크게 좌우하며, 데이터 유출 문제가 성능 과대평가 원인이 됨. GZSL 설정은 현실적 시나리오이지만 H 지표가 낮을 경우 본/미본 편향 극심하므로 보정 기법 필수.
